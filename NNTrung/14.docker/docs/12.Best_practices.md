# Docker Best Practices & Cheat Sheet

Tổng hợp các best practice "đáng tiền" qua từng lớp + cheat sheet đầy đủ.

## 1. Image

### Build
- ✅ **Dùng tag cụ thể**: `node:22-alpine`, không `node:latest`.
- ✅ **Multi-stage build**: tách build và runtime → image production nhỏ, không có compiler/build tools.
- ✅ **Base image nhỏ**: alpine (~5MB) hoặc distroless (gốc Debian slim ~25MB) cho production.
- ✅ **Pin version dependencies**: `npm ci` thay vì `npm install`, `pip install --require-hashes`, `apt install pkg=1.2.3`.
- ✅ **`.dockerignore`**: chặn `.git`, `node_modules`, `.env`, `*.pem` khỏi build context.
- ✅ **BuildKit cache mount**: `--mount=type=cache,target=...` cho npm/pip/apt.
- ✅ **`COPY --link`**: layer độc lập, cache tốt hơn.

### Layer order
```dockerfile
FROM node:22-alpine
WORKDIR /app

# 1. Copy file ÍT thay đổi trước (dependencies)
COPY package*.json ./
RUN npm ci

# 2. Copy source code (HAY thay đổi) sau
COPY . .
RUN npm run build
```

Sửa code → chỉ rebuild từ step `COPY . .` xuống. Không cần npm install lại.

### Layer size
- Mỗi `RUN apt-get install` → cleanup ngay trong cùng layer:
  ```dockerfile
  RUN apt-get update && \
      apt-get install -y --no-install-recommends curl && \
      rm -rf /var/lib/apt/lists/*
  ```
- ❌ KHÔNG dùng `RUN apt-get update` ở layer riêng — cache update bị "trôi" so với install.

### Anti-pattern
| ❌ | ✅ |
|---|---|
| `FROM ubuntu` (~80MB) | `FROM ubuntu:24.04-slim` (~30MB) |
| `RUN apt-get install ...` (không cleanup) | Gộp + cleanup trong cùng `RUN` |
| `COPY . /app` trước cài deps | Copy `package.json` → cài → copy source |
| `ADD url file` để tải | `RUN curl -fsSL url -o file` (có thể verify checksum) |
| `ARG SECRET=...` | `--mount=type=secret,id=...` |
| `RUN chown -R user /app` | `COPY --chown=user:user . /app` (giảm 1 layer) |
| Chạy `root` trong runtime | `USER app` |

## 2. Container Runtime

### Resource limits (luôn đặt)
```bash
docker run \
  --memory=512m \                  # Hard limit RAM
  --memory-swap=512m \             # Tắt swap (=memory)
  --cpus=1.5 \                     # CPU quota
  --pids-limit=200 \               # Chống fork bomb
  --ulimit nofile=1024:2048 \      # File descriptor
  myapp
```

### Restart policy
- ✅ `unless-stopped` cho service production.
- ⚠️ `always` sẽ restart kể cả khi bạn `docker stop` (nếu daemon restart).

### Healthcheck
Bắt buộc cho mọi container production — Compose/Swarm/K8s dựa vào để biết container "thật sự sẵn sàng":
```dockerfile
HEALTHCHECK --interval=30s --timeout=3s --start-period=10s --retries=3 \
  CMD curl -fsS http://localhost:3000/health || exit 1
```

### Log driver
Mặc định `json-file` không có rotation → log đầy disk.

`/etc/docker/daemon.json`:
```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

Hoặc per-container:
```bash
docker run --log-opt max-size=10m --log-opt max-file=3 myapp
```

### One process per container
Mỗi container chạy 1 main process. Nếu cần nhiều process (cron + app + nginx), tách ra nhiều container hoặc dùng `s6-overlay` / `supervisord` (cẩn thận).

### Signal handling
- Container nhận `SIGTERM` từ `docker stop` → có 10s (mặc định) để cleanup → sau đó nhận `SIGKILL`.
- App phải **handle SIGTERM** để shutdown sạch (flush DB, close socket…).
- Dạng exec `CMD ["node", "server.js"]` → signal đến đúng process. Dạng shell `CMD node server.js` → signal kẹt ở `/bin/sh`.

## 3. Network

- ✅ Custom bridge thay default `bridge` (có DNS theo tên).
- ✅ Tách zone: `frontend`, `backend`, `db-net` (`internal: true` cho db).
- ✅ Bind port nhạy cảm về `127.0.0.1`: `-p 127.0.0.1:5432:5432`.
- ✅ Dùng `DOCKER-USER` chain để chặn traffic (Docker bypass ufw/firewalld).
- ❌ Tránh `--network host` trừ khi thật sự cần performance.
- ❌ Đừng publish port DB ra `0.0.0.0` trong production.

---

## 4. Volume / Storage

- ✅ **Named volume cho DB/persistent**: Docker quản lý, portable.
- ✅ **Bind mount cho dev** (sync code) và config file.
- ✅ **Mount `:ro`** cho config — không cho container ghi đè.
- ✅ **Backup volume định kỳ** (cron).
- ✅ **Label volume `--label keep=true`** để tránh `prune` xóa nhầm.
- ❌ Đừng lưu DB vào bind mount trên Docker Desktop (Mac/Win) — chậm vì qua VM.
- ❌ Đừng mount `/` hoặc `/etc` của host vào container.

## 5. Security

(Chi tiết: [10.Security.md](10.Security.md))

### Minimum viable security cho production

```bash
docker run -d \
  --name myapp \
  --user 1000:1000 \
  --cap-drop ALL --cap-add NET_BIND_SERVICE \
  --security-opt no-new-privileges \
  --read-only \
  --tmpfs /tmp \
  --memory=512m --cpus=1.5 --pids-limit=200 \
  --restart unless-stopped \
  --health-cmd="curl -f http://localhost/health" --health-interval=30s \
  -p 127.0.0.1:8080:8080 \
  --network app-net \
  myorg/myapp@sha256:abc123...
```

Compose equivalent:
```yaml
services:
  myapp:
    image: myorg/myapp@sha256:abc123...
    user: "1000:1000"
    cap_drop: [ALL]
    cap_add: [NET_BIND_SERVICE]
    security_opt:
      - no-new-privileges:true
    read_only: true
    tmpfs:
      - /tmp
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: "1.5"
          pids: 200
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost/health"]
      interval: 30s
    ports:
      - "127.0.0.1:8080:8080"
    networks:
      - app-net
```

## 6. Compose

- ✅ Compose V2 (`docker compose`, có space).
- ✅ Tên file `compose.yaml` (V2 chuẩn).
- ✅ Bỏ trường `version:` (deprecated).
- ✅ Dùng `depends_on` + `condition: service_healthy` cho deps có healthcheck.
- ✅ Tách `compose.override.yaml` cho dev, `compose.prod.yaml` cho prod.
- ✅ Dùng `profiles:` để bật/tắt service tùy môi trường.
- ✅ Dùng `extension fields` (`x-*` + `&anchor`) để DRY.
- ✅ Secret qua file, không env: `POSTGRES_PASSWORD_FILE: /run/secrets/db_password`.

## 7. Performance Build

- ✅ Đặt build context nhỏ (`.dockerignore`).
- ✅ Cache mount cho package manager.
- ✅ Multi-stage chỉ copy artifact cần thiết.
- ✅ Build đa nền tảng bằng `buildx`: `--platform linux/amd64,linux/arm64`.
- ✅ Remote cache cho CI:
  ```bash
  docker buildx build \
    --cache-from type=registry,ref=myorg/myapp:cache \
    --cache-to type=registry,ref=myorg/myapp:cache,mode=max \
    -t myorg/myapp:1.0 --push .
  ```
- ✅ `docker system prune -af --filter "until=24h"` định kỳ để dọn cache cũ.

## 8. Daemon Config (`/etc/docker/daemon.json`)

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "live-restore": true,
  "userland-proxy": false,
  "no-new-privileges": true,
  "icc": false,
  "default-runtime": "runc",
  "registry-mirrors": ["https://mirror.example.com"],
  "insecure-registries": [],
  "default-address-pools": [
    { "base": "172.30.0.0/16", "size": 24 }
  ],
  "metrics-addr": "127.0.0.1:9323",
  "experimental": false,
  "storage-driver": "overlay2"
}
```

| Key | Mục đích |
|---|---|
| `live-restore` | Container vẫn chạy khi daemon restart |
| `userland-proxy: false` | Tắt docker-proxy → dùng iptables NAT (nhanh hơn) |
| `icc: false` | Tắt inter-container communication mặc định (an toàn hơn) |
| `default-address-pools` | Tránh xung đột IP với VPN corp (172.17.x.x mặc định hay đụng) |
| `metrics-addr` | Expose metric Prometheus của Engine |

Apply: `sudo systemctl reload docker`.

## 9. Troubleshooting checklist

### Container exit ngay
```bash
docker ps -a                          # Xem exit code
docker logs <name>                    # Đọc log
docker inspect <name> --format '{{.State.ExitCode}}'
docker inspect <name> --format '{{.State.Error}}'
```

### Container chạy nhưng app không nhận request
```bash
docker port <name>                              # Map port đúng chưa?
docker exec <name> netstat -tlnp                # App có bind đúng interface?
docker exec <name> wget -qO- http://localhost   # Test bên trong
docker exec <name> cat /etc/resolv.conf         # DNS đúng?
```

### "Cannot connect to Docker daemon"
```bash
sudo systemctl status docker
sudo systemctl start docker
groups $USER                          # User có trong nhóm `docker`?
sudo usermod -aG docker $USER && newgrp docker
```

### Image build chậm
```bash
docker history --no-trunc myapp:1.0      # Layer nào nặng?
docker build --progress=plain ...        # Xem rõ từng bước
```

### Disk đầy
```bash
docker system df                       # Xem chi tiết
docker system prune -af --volumes      # Xóa sạch (cẩn thận)
docker builder prune                   # Cache build
```

### Network conflict / slow
```bash
docker network ls
docker network inspect bridge | grep Subnet
# Đổi default address pool nếu trùng VPN
```

## 10. Cheat sheet siêu gọn

```bash
# === HỆ THỐNG ===
docker version            # Version client + engine
docker info               # System info
docker system df          # Disk usage
docker system prune -af   # Dọn (trừ volume)

# === IMAGE ===
docker pull I:T
docker build -t N:T .
docker images
docker history I:T
docker rmi I:T
docker image prune -a

# === CONTAINER ===
docker run -d --name N -p H:C --restart unless-stopped I:T
docker ps / ps -a
docker logs -f N
docker exec -it N sh
docker stop N && docker rm N
docker cp src N:dst
docker stats

# === VOLUME ===
docker volume create V
docker run -v V:/data I
docker volume ls / inspect V / rm V

# === NETWORK ===
docker network create N
docker run --network N I
docker network ls / inspect N / rm N

# === COMPOSE (V2) ===
docker compose up -d
docker compose down [-v]
docker compose ps / logs -f / exec S sh
docker compose pull / build / restart

# === REGISTRY ===
echo $T | docker login R -u U --password-stdin
docker tag I R/I:T
docker push R/I:T

# === SWARM ===
docker swarm init --advertise-addr IP
docker swarm join --token T IP:2377
docker node ls / promote / demote
docker service create --name N --replicas R I
docker stack deploy -c f.yaml NAME
docker secret create N - <<< "value"

# === DEBUG ===
docker inspect N
docker top N
docker diff N
docker run --rm -it --network N nicolaka/netshoot

# === SECURITY ===
docker scout cves I:T
docker scout quickview I:T
docker run --user 1000:1000 --cap-drop ALL --read-only --security-opt no-new-privileges I
```
