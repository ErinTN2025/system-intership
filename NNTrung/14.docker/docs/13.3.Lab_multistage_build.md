# Lab 3 — Multi-stage build & tối ưu image

## Mục tiêu

Học cách viết Dockerfile production-grade với multi-stage build:
- So sánh image **single-stage** vs **multi-stage**.
- Dùng BuildKit cache mount để build nhanh.
- Build image đa nền tảng (amd64 + arm64).
- Distroless image cho attack surface tối thiểu.

Áp dụng cho 2 stack: **Node.js** và **Go**.

## Yêu cầu

```bash
docker --version              # 23.0+ (đã có BuildKit mặc định)
docker buildx version
```

### A.2 Single-stage Dockerfile (cách "ngây thơ")

File `Dockerfile.single`:
```dockerfile
FROM node:22-alpine

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

EXPOSE 3000
CMD ["node", "dist/server.js"]
```

Build và đo size:
```bash
docker build -f Dockerfile.single -t nodeapp:single .
docker images nodeapp
# nodeapp   single   ...   ~ 250-300 MB
```

Image chứa: source TypeScript, devDependencies (`typescript`, `@types/*`), npm cache, build tools — toàn bộ thứ không cần lúc runtime.

---

### A.3 Multi-stage Dockerfile (tối ưu)

File `Dockerfile`:
```dockerfile
# syntax=docker/dockerfile:1.7

# ===== Stage 1: Builder =====
FROM node:22-alpine AS builder
WORKDIR /app

COPY package*.json ./
RUN --mount=type=cache,target=/root/.npm \
    npm ci

COPY tsconfig.json .
COPY src/ ./src/
RUN npm run build

# Loại bỏ devDependencies sau khi build
RUN --mount=type=cache,target=/root/.npm \
    npm prune --omit=dev

# ===== Stage 2: Runner =====
FROM node:22-alpine AS runner
WORKDIR /app

ENV NODE_ENV=production

# Non-root user
RUN addgroup -S app && adduser -S app -G app

COPY --from=builder --chown=app:app /app/dist ./dist
COPY --from=builder --chown=app:app /app/node_modules ./node_modules
COPY --from=builder --chown=app:app /app/package.json ./

USER app
EXPOSE 3000

HEALTHCHECK --interval=30s --timeout=3s --start-period=10s \
  CMD wget -qO- http://localhost:3000/health || exit 1

CMD ["node", "dist/server.js"]
```

Build:
```bash
docker build -t nodeapp:multi .
docker images nodeapp
# nodeapp   multi    ...   ~ 180 MB (chỉ runtime + prod deps)
# nodeapp   single   ...   ~ 280 MB
```

Verify hoạt động giống nhau:
```bash
docker run -d --name na -p 3000:3000 nodeapp:multi
curl http://localhost:3000/
# {"message":"Hello from Node.js!","hostname":"...","node":"v22.x.x","platform":"linux/amd64"}
docker rm -f na
```

So sánh nhanh `docker history`:
```bash
docker history nodeapp:single
docker history nodeapp:multi
```

---

### A.4 Tận dụng cache layer

Build lần 2 không đổi gì:
```bash
docker build -t nodeapp:multi .
# Tất cả step: CACHED
```

Sửa `src/server.ts` (thay đổi source) → build lại:
```bash
docker build -t nodeapp:multi .
# Step COPY package*.json + npm ci: CACHED (deps không đổi)
# Step COPY src + npm run build: rebuild
# Step copy artifact sang runner: rebuild
```

Thử sửa `package.json` (thêm dependency):
```bash
docker build -t nodeapp:multi .
# CACHED đến COPY package*.json
# npm ci: rebuild (đã invalidate)
# Tất cả phía sau: rebuild
```

Đây chính là lý do **luôn copy package.json trước, source sau**.

## Phần B — Go app (distroless cực nhỏ)

### B.1 Setup

```bash
mkdir -p ~/docker-lab/multistage/go-app && cd $_
go mod init goapp 2>/dev/null || true
```

File `go.mod`:
```go
module goapp

go 1.23
```

File `main.go`:
```go
package main

import (
	"encoding/json"
	"fmt"
	"log"
	"net/http"
	"os"
	"runtime"
)

func main() {
	hostname, _ := os.Hostname()

	http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		w.Header().Set("Content-Type", "application/json")
		_ = json.NewEncoder(w).Encode(map[string]string{
			"message":  "Hello from Go!",
			"hostname": hostname,
			"go":       runtime.Version(),
			"platform": fmt.Sprintf("%s/%s", runtime.GOOS, runtime.GOARCH),
		})
	})

	http.HandleFunc("/health", func(w http.ResponseWriter, _ *http.Request) {
		fmt.Fprint(w, "OK")
	})

	addr := ":8080"
	log.Printf("listening on %s", addr)
	log.Fatal(http.ListenAndServe(addr, nil))
}
```

---

### B.2 Multi-stage với distroless

File `Dockerfile`:
```dockerfile
# syntax=docker/dockerfile:1.7

# ===== Stage 1: Build =====
FROM golang:1.23-alpine AS builder
WORKDIR /src

COPY go.mod ./
# Nếu có go.sum thì copy nốt; ở đây project chưa cần dep ngoài
# COPY go.sum ./
RUN --mount=type=cache,target=/go/pkg/mod \
    go mod download

COPY . .
# Static binary, strip debug info
RUN --mount=type=cache,target=/go/pkg/mod \
    --mount=type=cache,target=/root/.cache/go-build \
    CGO_ENABLED=0 GOOS=linux \
    go build -ldflags="-w -s" -trimpath \
    -o /out/app .

# ===== Stage 2: Distroless =====
FROM gcr.io/distroless/static-debian12:nonroot
COPY --from=builder /out/app /app

USER nonroot:nonroot
EXPOSE 8080
ENTRYPOINT ["/app"]
```

Build và đo:
```bash
docker build -t goapp:distroless .
docker images goapp
# goapp   distroless   ...   ~ 10-15 MB (!!)
```

> So với Node.js multi-stage 180MB, Go binary tĩnh + distroless chỉ ~10MB.

Test:
```bash
docker run -d --name ga -p 8080:8080 goapp:distroless
curl http://localhost:8080/
# {"go":"go1.23.x","hostname":"...","message":"Hello from Go!","platform":"linux/amd64"}
docker rm -f ga
```

---

### B.3 Khi nào dùng distroless

| Tiêu chí | alpine | distroless |
|---|---|---|
| Có shell `sh` | ✅ | ❌ |
| Có package manager | apk | ❌ |
| Kích thước | ~5MB base | ~2MB base (static) |
| Debug trong container | Dễ | Khó (phải dùng debug image) |
| Attack surface | Trung bình | Rất nhỏ |
| Phù hợp | Hầu hết workload | Binary tĩnh (Go, Rust), critical security |

Debug distroless: dùng `distroless/static-debian12:debug` (có `busybox`).

## Phần C — Build đa nền tảng

### C.1 Setup buildx

```bash
# Tạo builder mới (chỉ làm 1 lần)
docker buildx create --name multi --use --bootstrap
docker buildx ls
# NAME/NODE   DRIVER     PLATFORMS
# multi *     docker-cont linux/amd64, linux/arm64, linux/arm/v7, ...
```

> Nếu chạy trên Linux x86, Docker dùng QEMU để emulate kiến trúc khác (chậm hơn). Trên Apple Silicon, build amd64 cũng cần QEMU.

### C.2 Build cho 2 platform

```bash
cd ~/docker-lab/multistage/go-app

# Chỉ build, không push (chỉ cache cục bộ)
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t goapp:multi-arch \
  --load \                  # --load chỉ hỗ trợ 1 platform
  .
# ERROR: --load không hỗ trợ multi-platform
```

Multi-platform image **bắt buộc push lên registry** vì image local store của Docker không lưu được multi-arch manifest (trừ khi bật containerd image store).

```bash
# Push lên Docker Hub
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t YOUR_USERNAME/goapp:1.0 \
  --push \
  .
```

Verify:
```bash
docker manifest inspect YOUR_USERNAME/goapp:1.0
# Thấy nhiều entry: linux/amd64, linux/arm64
```

Khi user pull, Docker tự chọn entry đúng platform.

## Phần D — Cache hỗn hợp / registry cache

CI cần cache build qua nhiều run, dùng registry cache:

```bash
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --cache-from type=registry,ref=YOUR_USERNAME/goapp:buildcache \
  --cache-to   type=registry,ref=YOUR_USERNAME/goapp:buildcache,mode=max \
  -t YOUR_USERNAME/goapp:1.0 \
  --push \
  .
```

Lần build sau, ngay cả từ máy khác, vẫn dùng được cache.

## Phần F — So sánh tổng kết

| Build | Image size | CVE typical | Build time (cold) | Build time (cache hit) |
|---|---|---|---|---|
| `nodeapp:single` | ~280 MB | Cao | 60s | 60s (không tận dụng được) |
| `nodeapp:multi` | ~180 MB | Trung bình | 45s | <5s (cache deps) |
| `goapp:alpine` (single-stage) | ~350 MB | Trung bình | 50s | 50s |
| `goapp:distroless` (multi-stage) | ~12 MB | Rất thấp | 35s | <5s |

> Multi-stage + distroless cho Go: **30x nhỏ hơn**, **gần như không CVE**, **rebuild gần như instant** khi chỉ sửa code.

## Tổng kết — những thứ cần nhớ

1. **Multi-stage**: stage build có toolchain, stage runtime chỉ có artifact.
2. **COPY deps file trước source**: cache layer dependencies riêng biệt.
3. **`--mount=type=cache`**: cache `~/.npm`, `~/.cache/pip`, `/go/pkg/mod` giữa các build.
4. **Image production**: alpine (general), distroless (Go/Rust binary).
5. **Non-root user**: `USER app` hoặc `USER nonroot:nonroot`.
6. **`--platform linux/amd64,linux/arm64`** + `--push`: deploy đa kiến trúc.
7. **Registry cache** (`--cache-from/to type=registry`): CI tận dụng cache giữa các run.
8. **`docker scout cves`** + `dive`: đo chất lượng image trước khi push.

## Thử thách

1. Viết multi-stage cho project Python (`build` stage cài deps, `runtime` chỉ có `/usr/local/lib/python3.X/site-packages`).

3. Build Go app linked tĩnh + nén bằng `upx` → image còn ~5MB.
