# Docker CLI — Các lệnh thường dùng

Docker CLI giao tiếp với `dockerd` qua REST API. Mọi lệnh đều theo cú pháp:

```bash
docker [OPTIONS] COMMAND [SUBCOMMAND] [ARGS...]
```

Từ Docker 1.13+, các lệnh đã được nhóm lại (ví dụ `docker container ls` thay cho `docker ps`). Cả hai cú pháp đều dùng được.

---

## 1. Thông tin hệ thống

```bash
docker version              # Phiên bản Client + Engine
docker info                 # Trạng thái toàn cục: số container, image, storage driver…
docker system df            # Dung lượng Docker đang dùng (image, container, volume, cache)
docker system df -v         # Chi tiết từng item
docker stats                # Monitor realtime CPU/RAM/IO của container
docker stats --no-stream    # Một snapshot, không stream
docker events               # Stream sự kiện Docker realtime (debug)
```

## 2. Quản lý Image

### 2.1 Pull / Push

```bash
docker pull nginx:1.27-alpine                # Tải image
docker pull --platform linux/amd64 nginx     # Chỉ định platform
docker push myrepo/myapp:1.0                 # Đẩy lên registry (cần login)
```

### 2.2 Liệt kê / Tra cứu

```bash
docker images                                # Liệt kê image local
docker image ls                              # Tương đương
docker images -a                             # Bao gồm cả intermediate layer
docker images --digests                      # Hiện cả digest
docker images --filter "dangling=true"       # Image mồ côi (không tag, không dùng)
docker image inspect nginx:1.27-alpine       # Toàn bộ metadata
docker image history nginx:1.27-alpine       # Layer history
docker search nginx                          # Tìm image trên Docker Hub
```

### 2.3 Tag / Build / Save / Load

```bash
docker tag nginx:1.27-alpine myrepo/nginx:custom    # Đặt tag mới
docker build -t myapp:1.0 .                         # Build từ Dockerfile
docker save myapp:1.0 -o myapp.tar                  # Export image ra file tar
docker load -i myapp.tar                            # Import từ tar
```

### 2.4 Xóa

```bash
docker rmi nginx:1.27-alpine        # Xóa image (alias: docker image rm)
docker rmi -f nginx                 # Force xóa (kể cả khi có container dùng)
docker image prune                  # Xóa image dangling (không tag)
docker image prune -a               # Xóa tất cả image không được dùng bởi container nào
docker image prune -a --filter "until=24h"   # Xóa image cũ hơn 24 giờ
```

## 3. Quản lý Container

### 3.1 Tạo và chạy

```bash
docker run nginx                              # Foreground, gắn STDOUT/STDERR
docker run -d nginx                           # Detached (chạy ngầm)
docker run -d -p 8080:80 nginx                # Map port host:container
docker run -d -p 127.0.0.1:8080:80 nginx      # Chỉ bind localhost
docker run -d --name web nginx                # Đặt tên container
docker run -it ubuntu:24.04 bash              # Interactive + TTY (vào shell)
docker run --rm ubuntu:24.04 echo "hello"     # Tự xóa sau khi exit
docker run -e MY_VAR=value nginx              # Biến môi trường
docker run --env-file .env nginx              # Load env từ file
docker run -v my-vol:/data nginx              # Named volume
docker run -v $(pwd)/src:/app/src nginx       # Bind mount
docker run --network my-net nginx             # Custom network
docker run --restart unless-stopped nginx     # Restart policy
docker run --memory="512m" --cpus="1.5" nginx # Giới hạn tài nguyên
docker run --user 1000:1000 nginx             # Chạy với UID:GID cụ thể
docker run --read-only nginx                  # Filesystem chỉ đọc
docker run --cap-drop ALL --cap-add NET_BIND_SERVICE nginx   # Giới hạn capability
docker run --health-cmd="curl -f http://localhost/" \
           --health-interval=30s nginx        # Healthcheck
```

#### Restart policies

| Policy | Hành vi |
|---|---|
| `no` (mặc định) | Không tự restart |
| `on-failure[:N]` | Restart khi exit code ≠ 0, tối đa N lần |
| `always` | Luôn restart, kể cả khi `docker stop` (rồi daemon restart) |
| `unless-stopped` | Như `always` nhưng tôn trọng `docker stop` thủ công |

### 3.2 Quản lý vòng đời

```bash
docker ps                       # Container đang chạy
docker ps -a                    # Tất cả (cả đã dừng)
docker ps -q                    # Chỉ ID (dùng để pipe)
docker ps --filter "status=exited"
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

docker stop web                 # SIGTERM, chờ 10s, SIGKILL
docker stop -t 30 web           # Tăng timeout lên 30s
docker start web                # Khởi động lại container đã dừng
docker restart web              # Stop rồi start
docker pause web                # Freeze process (SIGSTOP qua cgroups freezer)
docker unpause web              # Resume
docker kill web                 # SIGKILL ngay lập tức
docker kill -s SIGHUP web       # Gửi signal khác

docker rm web                   # Xóa container đã dừng
docker rm -f web                # Force xóa container đang chạy
docker rm $(docker ps -aq)      # Xóa tất cả container
```

### 3.3 Tương tác

```bash
docker exec -it web bash              # Mở shell vào container
docker exec -it web sh                # Khi không có bash (alpine)
docker exec web cat /etc/hosts        # Chạy lệnh đơn
docker exec -u root -it web bash      # Vào với user root
docker exec -w /app web ls            # Đặt working dir

docker logs web                       # Xem log
docker logs -f web                    # Follow realtime
docker logs --tail 100 web            # 100 dòng cuối
docker logs --since 10m web           # Log trong 10 phút gần nhất
docker logs -t web                    # Kèm timestamp

docker attach web                     # Gắn STDIN/STDOUT vào main process (cẩn thận: Ctrl+C sẽ kill container nếu không thoát đúng)
# Thoát attach mà không kill: Ctrl+P, Ctrl+Q
```

### 3.4 Inspect / Top / Stats

```bash
docker inspect web                              # Toàn bộ metadata JSON
docker inspect -f '{{.State.Status}}' web       # Lấy field cụ thể (Go template)
docker inspect -f '{{.NetworkSettings.IPAddress}}' web
docker top web                                  # Process trong container
docker stats web                                # CPU/RAM/IO realtime
docker port web                                 # Xem port mapping
docker diff web                                 # File đã thay đổi so với image
```

### 3.5 Copy file

```bash
docker cp ./config.yaml web:/etc/app/config.yaml     # Host → container
docker cp web:/var/log/app.log ./app.log              # Container → host
docker cp web:/etc/. ./web-etc/                       # Copy nguyên thư mục
```

### 3.6 Commit / Export

```bash
docker commit web my-image:custom        # Tạo image từ container hiện tại (debug only, không dùng cho production)
docker export web -o web.tar             # Export filesystem container ra tar (mất history layer)
docker import web.tar my-image:flat      # Import flat tar thành image
```

### 3.7 Dọn dẹp

```bash
docker container prune                      # Xóa tất cả container đã dừng
docker container prune --filter "until=24h"
docker system prune                         # Xóa container, network, image dangling không dùng
docker system prune -a                      # Xóa tất cả image không dùng
docker system prune -a --volumes            # Xóa sạch (CẢNH BÁO: mất volume data!)
```

## 4. Quản lý Volume

```bash
docker volume create my-data                   # Tạo volume
docker volume create --driver local \
       -o type=nfs -o o=addr=10.0.0.1,rw \
       -o device=:/exports/data my-nfs         # Volume với driver/option

docker volume ls                               # Liệt kê
docker volume ls --filter "dangling=true"      # Volume không gắn container nào
docker volume inspect my-data                  # Xem chi tiết
docker volume rm my-data                       # Xóa
docker volume prune                            # Xóa volume không dùng
docker volume prune --filter "label!=keep"     # Trừ volume có label `keep`
```

Chi tiết: [07.Volume_storage.md](07.Volume_storage.md).

## 5. Quản lý Network

```bash
docker network create my-net                          # Bridge network
docker network create --driver overlay swarm-net      # Overlay (cần swarm)
docker network create --subnet=172.20.0.0/16 \
       --gateway=172.20.0.1 my-net                    # Chỉ định subnet/gateway

docker network ls                                     # Liệt kê
docker network inspect my-net                         # Chi tiết (gồm container đang gắn)

docker network connect my-net web                     # Gắn container vào network
docker network connect --ip 172.20.0.10 my-net web   # Gắn với IP cố định
docker network disconnect my-net web                  # Bỏ gắn

docker network rm my-net                              # Xóa
docker network prune                                  # Xóa network không dùng
```

Chi tiết: [08.Network.md](08.Network.md).

## 6. Build Image

```bash
docker build -t myapp:1.0 .                              # Build từ thư mục hiện tại
docker build -t myapp:1.0 -f Dockerfile.prod .           # Chỉ định Dockerfile
docker build --no-cache -t myapp:1.0 .                   # Không dùng cache
docker build --target=builder -t myapp:dev .             # Chỉ build đến stage `builder` (multi-stage)
docker build --build-arg VERSION=1.0 -t myapp:1.0 .      # Truyền ARG
docker build --platform linux/amd64,linux/arm64 \
       -t myapp:1.0 .                                    # Build đa nền tảng (cần buildx)

# Buildx (BuildKit advanced)
docker buildx create --name multi --use                  # Tạo builder mới
docker buildx build --platform linux/amd64,linux/arm64 \
       -t myorg/myapp:1.0 --push .                       # Build và push đa nền tảng
docker buildx prune                                      # Dọn build cache
```

Chi tiết: [05.Dockerfile.md](05.Dockerfile.md).

## 7. Registry / Auth

```bash
docker login                                  # Login Docker Hub
docker login ghcr.io -u USER -p TOKEN         # Login registry khác
docker logout
docker logout ghcr.io

docker pull myrepo/myapp:1.0
docker push myrepo/myapp:1.0

# Tag để push lên registry private
docker tag myapp:1.0 ghcr.io/myorg/myapp:1.0
docker push ghcr.io/myorg/myapp:1.0
```

## 8. Docker Compose (V2)

Chỉ liệt kê nhanh — chi tiết ở [06.Docker_Compose.md](06.Docker_Compose.md).

```bash
docker compose up -d              # Khởi động stack (detached)
docker compose down               # Dừng và xóa
docker compose ps                 # Trạng thái service
docker compose logs -f            # Xem log tất cả
docker compose exec web bash      # Vào shell service `web`
docker compose restart web
docker compose pull               # Cập nhật image
docker compose build              # Build lại image
```

## 9. Format output (Go template)

Dùng `--format` để lấy đúng thông tin cần:

```bash
# Bảng tùy chỉnh
docker ps --format "table {{.Names}}\t{{.Image}}\t{{.Status}}"

# Lấy IP của container
docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' web

# Liệt kê tên + image
docker ps --format "{{.Names}}: {{.Image}}"

# JSON đầy đủ
docker inspect web | jq '.[0].State'
```

## 10. Cheat sheet siêu gọn

```bash
# Vòng đời thường dùng
docker pull IMAGE:TAG
docker build -t NAME:TAG .
docker run -d -p HOST:CONT --name NAME IMAGE:TAG
docker exec -it NAME sh
docker logs -f NAME
docker stop NAME && docker rm NAME

# Dọn dẹp
docker system df
docker system prune -a            # Trừ volume
docker system prune -a --volumes  # Bao gồm volume

# Debug
docker inspect NAME
docker top NAME
docker stats
docker events
```
