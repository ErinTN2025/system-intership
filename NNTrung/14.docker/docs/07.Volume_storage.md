# Volume & Storage

Container có **writable layer** trên cùng (Union FS), nhưng dữ liệu ở layer này **mất khi container bị xóa**. Để lưu data lâu dài (database, log, uploaded files…), phải dùng **volume** hoặc **bind mount**.

## 1. Ba loại mount

| Loại | Cú pháp ngắn (`-v`) | Cú pháp dài (`--mount`) | Quản lý bởi | Dùng khi |
|---|---|---|---|---|
| **Named volume** | `-v my-vol:/data` | `--mount type=volume,src=my-vol,dst=/data` | Docker | Production, DB |
| **Bind mount** | `-v /host/path:/data` | `--mount type=bind,src=/host/path,dst=/data` | Người dùng (host filesystem) | Dev (sync code), config file |
| **tmpfs** | `--tmpfs /tmp` | `--mount type=tmpfs,dst=/tmp` | Kernel (RAM) | Data tạm, secret runtime |

> **Khuyến nghị**: Dùng `--mount` (verbose nhưng rõ ràng) thay cho `-v` (ngắn nhưng dễ nhầm thứ tự).

### 1.1 Named Volume
Docker tạo ra một vùng lưu trữ riêng
```bash
Host

/var/lib/docker/
└── volumes/
    └── mysql-data/
         └── _data/
```
  - Khi container chạy, container ghi vào `/var/lib/mysql` nhưng thực ra dữ liệu nằm ở: `/var/lib/docker/volumes/mysql-data/_data`
  - Nếu xóa container volume vẫn còn, tạo container mới toàn bộ database sẽ quay lại, db luôn dùng Named Volume.

### 1.2 Bind Mount
Bạn có 1 thư mục ở bất kỳ đâu đó bạn muốn container dùng chính thư mục đó, bạn mount vào thư mục đó. 

Ví dụ:

Host:
```bash
/home/trung/project

index.js
package.json
```
Container:
```bash
/app

index.js
package.json
```
  - Nếu bạn sửa index.js bằng VSCode trên host, Container thấy thay đổi ngay.

### 1.3 tmpfs

Không lưu ở ổ cứng mà lưu trên RAM. Nếu reboot máy là mất. Ứng dụng tạo cache, session, file tạm.

## 2. Named Volume

### 2.1 Tạo và dùng

```bash
docker volume create my-data

docker run -d \
  --name db \
  -v my-data:/var/lib/postgresql/data \
  -e POSTGRES_PASSWORD=secret \
  postgres:16-alpine

# Hoặc cú pháp --mount
docker run -d \
  --name db \
  --mount type=volume,source=my-data,target=/var/lib/postgresql/data \
  -e POSTGRES_PASSWORD=secret \
  postgres:16-alpine
```
- Mount volume có tên là `my-data` vào container `/var/lib/postgresql/data`
- Nếu bạn chỉ định 1 volume đã tồn tại, Docker sử dụng volume đó nếu không tồn tại nó tự tạo.
- Bạn không thể xóa volume nếu nó đang được sử dụng bởi container


### 2.2 Quản lý

```bash
docker volume ls                                # Liệt kê
docker volume inspect my-data                   # Xem chi tiết
docker volume ls -f dangling=true               # Volume không dùng
docker volume rm my-data                        # Xóa
docker volume prune                             # Xóa volume không dùng
```

Volume được lưu tại `/var/lib/docker/volumes/<name>/_data` trên host (chỉ truy cập được với quyền root).

### 2.3 Anonymous volume

Khi `docker run -v /data ...` (không có tên), Docker tạo **anonymous volume** với tên random.

```bash
docker run -d -v /data nginx
docker volume ls
# DRIVER    VOLUME NAME
# local     a8f3b1...
```

Anonymous volume **không tự xóa** khi container bị xóa, trừ khi dùng `docker rm -v` hoặc `docker run --rm`.

### 2.4 Volume driver (nâng cao)

Mặc định driver là `local`. Có thể dùng driver khác cho NFS, CIFS, cloud…

```bash
# NFS share
docker volume create \
  --driver local \
  --opt type=nfs \
  --opt o=addr=10.0.0.5,rw,nfsvers=4 \
  --opt device=:/exports/data \
  nfs-vol

# CIFS/SMB
docker volume create \
  --driver local \
  --opt type=cifs \
  --opt o=username=user,password=pass,vers=3.0 \
  --opt device=//server/share \
  smb-vol

# tmpfs volume
docker volume create \
  --driver local \
  --opt type=tmpfs \
  --opt device=tmpfs \
  --opt o=size=100m,uid=1000 \
  tmp-vol
```

## 3. Bind Mount

Map thẳng một thư mục/file trên host vào container. Container thấy y hệt host (gồm cả permission).

```bash
# Linux/macOS
docker run -d -v $(pwd)/src:/app/src node:22-alpine

# Windows (PowerShell)
docker run -d -v ${PWD}/src:/app/src node:22-alpine

# Read-only
docker run -d -v $(pwd)/config:/config:ro nginx

# --mount form (an toàn hơn — báo lỗi nếu source không tồn tại)
docker run -d --mount type=bind,source=$(pwd)/src,target=/app/src,readonly nginx
```

### Khi nào dùng bind mount

- **Dev**: sync code thay đổi vào container (hot reload).
- **Config file**: mount `nginx.conf`, `prometheus.yml`…
- **Trao đổi dữ liệu** với host (artifact build, log file…).

### Cẩn thận với bind mount

- **Permission**: UID/GID trong container có thể khác với host → file tạo trong container có thể không đọc được trên host (và ngược lại).
- **Mount đè**: Nếu mount lên `/app` mà image cũng có file ở đó, file trong image bị che mất.
- **Performance trên Docker Desktop**: Trên macOS/Windows, bind mount qua VM hypervisor — chậm hơn native Linux. Dùng named volume cho DB nếu thấy chậm.

## 4. tmpfs (RAM)

```bash
# Cú pháp tmpfs đơn giản
docker run -d --tmpfs /tmp:size=100M,uid=1000 nginx

# --mount
docker run -d --mount type=tmpfs,destination=/tmp,tmpfs-size=100m nginx
```

Dùng khi:
- Lưu data tạm không cần persistent.
- Tránh ghi vào disk (chứa secret runtime).
- Tăng tốc workload heavy I/O nhỏ.

Linux only.

## 5. Mount trong Compose

```yaml
services:
  web:
    image: nginx
    volumes:
      # Named volume
      - logs:/var/log/nginx

      # Bind mount (read-only)
      - ./nginx.conf:/etc/nginx/nginx.conf:ro

      # Bind mount (long syntax)
      - type: bind
        source: ./src
        target: /app/src
        read_only: false

      # tmpfs
      - type: tmpfs
        target: /tmp
        tmpfs:
          size: 100M

volumes:
  logs:                          # Khai báo named volume
  shared-data:
    external: true               # Volume đã tồn tại (tạo trước bằng docker volume create)
    name: existing-vol
```

## 6. Backup & Restore Volume

Pattern: chạy container tạm gắn volume + bind mount để dump/restore.

### 6.1 Backup

```bash
# Dump volume `db-data` ra file tar.gz trên host
docker run --rm \
  -v db-data:/source:ro \
  -v $(pwd):/backup \
  alpine \
  tar czf /backup/db-data-$(date +%F).tar.gz -C /source .
```

### 6.2 Restore

```bash
# Tạo volume mới (nếu cần)
docker volume create db-data

# Giải nén vào volume
docker run --rm \
  -v db-data:/target \
  -v $(pwd):/backup \
  alpine \
  sh -c "cd /target && tar xzf /backup/db-data-2026-05-13.tar.gz"
```

### 6.3 Backup PostgreSQL "đúng cách"

Backup volume thô khi DB đang chạy có thể corrupt. Dùng `pg_dump`:

```bash
docker exec -t db-container \
  pg_dump -U user -d mydb \
  | gzip > backup-$(date +%F).sql.gz
```

Restore:
```bash
gunzip -c backup-2026-05-13.sql.gz \
  | docker exec -i db-container psql -U user -d mydb
```

## 7. Permission troubleshooting

Lỗi `Permission denied` khi mount thường do UID mismatch.

### 7.1 Đọc UID/GID

Trong container:
```bash
docker exec web id
# uid=101(nginx) gid=101(nginx) groups=101(nginx)
```

Trên host:
```bash
ls -ln /path/to/mount
# -rw-r--r-- 1 1000 1000 ...
```

### 7.2 Cách xử lý

**Cách 1 — Đổi user của container**:
```bash
docker run -d --user $(id -u):$(id -g) -v $(pwd):/app node:22-alpine
```

**Cách 2 — Đổi owner trên host**:
```bash
sudo chown -R 101:101 /path/to/mount
```

**Cách 3 — Dùng `--chown` khi COPY trong Dockerfile**:
```dockerfile
COPY --chown=app:app . /app
```

**Cách 4 — Init script đổi quyền** (khi UID động):
```dockerfile
COPY entrypoint.sh /entrypoint.sh
ENTRYPOINT ["/entrypoint.sh"]
```
```bash
#!/bin/sh
# entrypoint.sh
chown -R app:app /data
exec gosu app "$@"
```

## 8. Storage driver (host-level)

Docker dùng **storage driver** để quản lý layer của image/container. Mặc định trên Linux hiện đại:
- `overlay2` (khuyến nghị, ổn định nhất).
- `btrfs`, `zfs` (nếu host dùng các filesystem này).
- `fuse-overlayfs` (rootless mode).

Xem driver hiện tại:
```bash
docker info | grep -i "storage driver"
```

> Hiếm khi cần đổi storage driver. Chỉ thay khi có lý do cụ thể (filesystem đặc biệt, rootless, …).

## 9. Best Practices

| Nguyên tắc | Lý do |
|---|---|
| Production: dùng named volume cho DB | Docker quản lý, không phụ thuộc layout host |
| Dev: bind mount source code | Hot reload, sửa code thấy ngay |
| Mount config file dạng `:ro` | Tránh container ghi đè config |
| Backup volume thường xuyên (cron) | Disaster recovery |
| Đặt label volume (`label=keep`) | Tránh `volume prune` xóa nhầm |
| Đừng commit `.env` lên git | Tránh leak secret |
| Đừng mount `/var/run/docker.sock` vào container nếu không cần | = quyền root trên host |
| Dùng `--mount` (verbose) cho production scripts | Báo lỗi sớm, không tự tạo path |
| Bind mount `:ro` cho file nhạy cảm | Defense in depth |

## 10. Cheat sheet

```bash
# Tạo + dùng
docker volume create my-vol
docker run -v my-vol:/data nginx

# Inspect
docker volume ls
docker volume inspect my-vol
docker inspect <container> --format '{{json .Mounts}}' | jq

# Xóa
docker volume rm my-vol
docker volume prune

# Backup
docker run --rm -v my-vol:/src -v $(pwd):/bk alpine \
  tar czf /bk/my-vol.tar.gz -C /src .

# Restore
docker run --rm -v my-vol:/dst -v $(pwd):/bk alpine \
  tar xzf /bk/my-vol.tar.gz -C /dst
```
