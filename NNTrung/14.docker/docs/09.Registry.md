# Docker Registry

**Registry** là nơi lưu trữ và phân phối Docker image. Khi bạn `docker pull nginx`, Docker tải image từ một registry (mặc định là Docker Hub).

## 1. Image Naming

Format đầy đủ: `[REGISTRY[:PORT]/][NAMESPACE/]NAME[:TAG | @DIGEST]`

| Ví dụ | Giải thích |
|---|---|
| `nginx` | `docker.io/library/nginx:latest` (mặc định) |
| `nginx:1.27-alpine` | Tag cụ thể |
| `library/nginx` | Namespace `library` (official image) trên Docker Hub |
| `myuser/myapp:1.0` | User trên Docker Hub |
| `ghcr.io/myorg/myapp:1.0` | GitHub Container Registry |
| `registry.example.com:5000/myapp:1.0` | Self-hosted registry trên port 5000 |
| `nginx@sha256:abc123...` | Digest (bất biến) |

> **Production luôn pin bằng tag cụ thể hoặc digest**, không bao giờ dùng `latest`.

---

## 2. Các Registry phổ biến

| Registry | Domain | Đặc điểm |
|---|---|---|
| **Docker Hub** | `docker.io` (mặc định) | Lớn nhất, free 1 private repo, rate limit cho anonymous pull |
| **GitHub Container Registry** | `ghcr.io` | Free cho public, tích hợp GitHub Actions, dùng GitHub token |
| **Google Artifact Registry** | `*-docker.pkg.dev` | Tích hợp GCP, IAM-based |
| **Amazon ECR** | `*.dkr.ecr.*.amazonaws.com` | Tích hợp AWS, IAM, ECR Public miễn phí |
| **Azure Container Registry** | `*.azurecr.io` | Tích hợp Azure |
| **Quay.io** | `quay.io` | Red Hat, có security scan |
| **Self-hosted** | (tự đặt) | Harbor, Nexus, Gitea, Docker Registry, GitLab Registry |

## 3. Login / Logout

```bash
docker login                                    # Docker Hub (interactive)
docker login -u USERNAME -p PASSWORD            # Inline (KHÔNG khuyến nghị, lưu vào history)
echo $TOKEN | docker login -u USERNAME --password-stdin   # Stdin (an toàn)

docker login ghcr.io -u USERNAME --password-stdin <<< "$GITHUB_TOKEN"
docker login registry.example.com:5000

docker logout
docker logout ghcr.io
```

Credentials được lưu tại:
- Linux: `~/.docker/config.json` (base64, KHÔNG mã hóa — cẩn thận)
- macOS: Keychain
- Windows: Credential Manager

### Credential helper (an toàn hơn)

```json
// ~/.docker/config.json
{
  "credsStore": "secretservice",     // Linux: gnome-keyring
  "credHelpers": {
    "ghcr.io": "pass",
    "123456789.dkr.ecr.us-east-1.amazonaws.com": "ecr-login"
  }
}
```

## 4. Pull / Push

### 4.1 Pull

```bash
docker pull nginx:1.27-alpine                  # Pull tag cụ thể
docker pull nginx@sha256:abc123...             # Pull digest (bất biến)
docker pull --platform linux/arm64 nginx       # Chỉ định platform
docker pull --all-tags nginx                   # Pull TẤT CẢ tag (cẩn thận: nặng!)
docker pull ghcr.io/myorg/myapp:1.0
```

### 4.2 Push

```bash
# Phải tag với registry URL trước khi push
docker tag myapp:1.0 myuser/myapp:1.0                       # Docker Hub
docker tag myapp:1.0 ghcr.io/myorg/myapp:1.0                # GHCR
docker tag myapp:1.0 registry.example.com:5000/myapp:1.0    # Self-hosted

docker push myuser/myapp:1.0
docker push ghcr.io/myorg/myapp:1.0
docker push --all-tags myuser/myapp                         # Push tất cả tag
```

### 4.3 Buildx — build + push một bước

```bash
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t myorg/myapp:1.0 \
  -t myorg/myapp:latest \
  --push \
  .
```

## 5. Tag Strategy

Đừng dùng `latest` cho production. Một số pattern phổ biến:

### 5.1 SemVer + immutable

```
myapp:1.2.3              # Patch cụ thể, immutable
myapp:1.2                # Floating minor (1.2.x mới nhất)
myapp:1                  # Floating major
myapp:latest             # Mới nhất tuyệt đối (chỉ dev/demo)
```

### 5.2 Git-based

```
myapp:git-abc123def      # Short SHA — immutable, dễ trace
myapp:main               # Branch tag — floating
myapp:pr-456             # PR build
```

### 5.3 Multi-tag khi push

```bash
TAG=$(git rev-parse --short HEAD)
docker buildx build \
  -t myorg/myapp:${TAG} \
  -t myorg/myapp:$(git rev-parse --abbrev-ref HEAD) \
  -t myorg/myapp:latest \
  --push .
```

### 5.4 Pin bằng digest cho production

```bash
docker pull nginx:1.27-alpine
docker images --digests nginx
# nginx   1.27-alpine   sha256:abc123...   ...

# Deploy bằng digest — KHÔNG ai đổi được
docker run nginx@sha256:abc123...
```

Compose:
```yaml
services:
  web:
    image: nginx@sha256:abc123def456...
```

## 6. Rate Limit Docker Hub

Docker Hub có rate limit cho anonymous pull (100 pull/6h từ một IP). Hậu quả thường gặp: CI/CD fail vì share IP với hàng nghìn người dùng khác.

### Cách giảm thiểu

1. **Login** trước khi pull (anonymous → authenticated):
   ```bash
   echo $DOCKERHUB_TOKEN | docker login -u USER --password-stdin
   ```
2. **Mirror Docker Hub** qua self-hosted registry hoặc cloud registry.
3. Cấu hình registry mirror trong `/etc/docker/daemon.json`:
   ```json
   {
     "registry-mirrors": ["https://mirror.example.com"]
   }
   ```
4. Cache image trong CI (GitHub Actions: `docker/build-push-action` có cache-from/cache-to).

## 7. Self-hosted Registry

### 7.1 Cách nhanh nhất — `registry:2`

```bash
docker run -d \
  --name registry \
  --restart=unless-stopped \
  -p 5000:5000 \
  -v registry-data:/var/lib/registry \
  registry:2
```

```bash
# Test
docker tag nginx:alpine localhost:5000/nginx:alpine
docker push localhost:5000/nginx:alpine

docker pull localhost:5000/nginx:alpine
curl http://localhost:5000/v2/_catalog
# {"repositories":["nginx"]}
```

> Mặc định registry này **không TLS, không auth** — chỉ dùng cho lab/internal. Docker daemon từ chối push/pull lên registry HTTP trừ khi thêm vào `insecure-registries`:

```json
// /etc/docker/daemon.json
{
  "insecure-registries": ["myhost.local:5000"]
}
```

### 7.2 Có TLS + Basic Auth (production-grade nhẹ)

**1. Tạo cert**:
```bash
mkdir -p certs
openssl req -newkey rsa:4096 -nodes -sha256 \
  -keyout certs/domain.key -x509 -days 365 \
  -subj "/CN=registry.example.com" \
  -addext "subjectAltName=DNS:registry.example.com" \
  -out certs/domain.crt
```

**2. Tạo htpasswd**:
```bash
mkdir -p auth
docker run --rm --entrypoint htpasswd httpd:2 -Bbn admin StrongPassword \
  > auth/htpasswd
```

**3. Chạy registry**:
```bash
docker run -d \
  --name registry \
  --restart=unless-stopped \
  -p 443:5000 \
  -v $(pwd)/certs:/certs \
  -v $(pwd)/auth:/auth \
  -v registry-data:/var/lib/registry \
  -e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/domain.crt \
  -e REGISTRY_HTTP_TLS_KEY=/certs/domain.key \
  -e REGISTRY_AUTH=htpasswd \
  -e "REGISTRY_AUTH_HTPASSWD_REALM=Registry Realm" \
  -e REGISTRY_AUTH_HTPASSWD_PATH=/auth/htpasswd \
  registry:2
```

**4. Trust cert trên client**:
```bash
sudo mkdir -p /etc/docker/certs.d/registry.example.com
sudo cp certs/domain.crt /etc/docker/certs.d/registry.example.com/ca.crt
```

**5. Login + push**:
```bash
docker login registry.example.com
docker push registry.example.com/myapp:1.0
```

### 7.3 Harbor (enterprise-grade)

Harbor là registry mã nguồn mở của CNCF với GUI, RBAC, vulnerability scan, replication, image signing. Cài qua Helm hoặc docker-compose. Phù hợp cho team/công ty.

Trang chủ: https://goharbor.io/

## 8. Useful Registry API

Registry HTTP API V2 dùng được trên cả `registry:2`, Docker Hub, GHCR…

```bash
# List repository (registry:2 hoặc Harbor)
curl https://registry.example.com/v2/_catalog

# List tag của một repo
curl https://registry.example.com/v2/myapp/tags/list

# Lấy manifest
curl -H "Accept: application/vnd.oci.image.manifest.v1+json" \
  https://registry.example.com/v2/myapp/manifests/1.0

# Xóa image bằng digest (cần REGISTRY_STORAGE_DELETE_ENABLED=true)
DIGEST=$(curl -sI -H "Accept: application/vnd.oci.image.manifest.v1+json" \
  https://registry.example.com/v2/myapp/manifests/1.0 \
  | grep -i docker-content-digest | cut -d' ' -f2 | tr -d '\r')
curl -X DELETE https://registry.example.com/v2/myapp/manifests/$DIGEST
```

Sau khi DELETE, vẫn cần `registry garbage-collect` để giải phóng disk:
```bash
docker exec registry bin/registry garbage-collect /etc/docker/registry/config.yml
```

## 9. Image Inspection từ xa

`skopeo` cho phép xem/copy image giữa các registry mà không cần pull về local:

```bash
# Cài
sudo apt install skopeo

# Inspect không pull
skopeo inspect docker://nginx:1.27-alpine

# Liệt kê tag
skopeo list-tags docker://docker.io/library/nginx

# Copy giữa hai registry (mirror)
skopeo copy --all \
  docker://docker.io/library/nginx:1.27-alpine \
  docker://registry.example.com/nginx:1.27-alpine
```

## 10. Best Practices

| Nguyên tắc | Lý do |
|---|---|
| Không dùng tag `latest` cho production | Build "trôi" — không tái lập |
| Pin bằng digest cho release critical | Bất biến — không ai đổi được |
| Login Docker Hub trước khi pull trong CI | Tăng rate limit từ 100/6h lên 200/6h |
| Dùng credential helper (Keychain, pass, gpg) | Tránh lưu password plain trong `config.json` |
| Đặt registry sau TLS (Let's Encrypt) | Tránh MITM |
| Bật image signing (cosign/Notary) cho prod | Verify image gốc chưa bị sửa |
| Quét image bằng Scout/Trivy trước khi push | Bắt CVE sớm |
| Set retention policy (xóa tag cũ tự động) | Tiết kiệm storage |
| Mirror Docker Hub trong corp network | Giảm phụ thuộc internet + rate limit |
| Tách registry cho dev/staging/prod | Tránh push nhầm tag |

```bash
# Login/Logout
echo $TOKEN | docker login REGISTRY -u USER --password-stdin
docker logout REGISTRY

# Pull/Push
docker pull REGISTRY/IMAGE:TAG
docker tag LOCAL_IMAGE REGISTRY/IMAGE:TAG
docker push REGISTRY/IMAGE:TAG

# Self-hosted registry nhanh
docker run -d --name registry -p 5000:5000 -v reg:/var/lib/registry registry:2

# Image inspection từ xa
skopeo inspect docker://nginx:1.27-alpine
skopeo copy docker://src docker://dst

# Liệt kê tag (registry:2)
curl https://REGISTRY/v2/IMAGE/tags/list
```
