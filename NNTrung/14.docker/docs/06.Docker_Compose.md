# Docker Compose

**Docker Compose** dùng để định nghĩa và chạy **ứng dụng đa container** (web + db + cache…) từ một file YAML duy nhất. Thay vì gõ hàng chục lệnh `docker run`, chỉ cần `docker compose up`.

> **Lưu ý 2024+**: Compose V1 (`docker-compose`, có gạch nối) đã **deprecated** từ năm 2023. Luôn dùng **Compose V2**: `docker compose` (có dấu cách). Plugin `docker-compose-plugin` được cài kèm Docker Engine từ v23.0+.

```bash
docker compose version              # Compose V2: 2.x.x
docker-compose version              # Compose V1 (deprecated)
```

## 1. Cấu trúc `compose.yaml`

Từ Compose V2, tên file khuyến nghị là `compose.yaml` (hoặc `compose.yml`). Tên cũ `docker-compose.yml`/`docker-compose.yaml` vẫn hoạt động để tương thích.

> Trường `version:` ở đầu file **đã deprecated** từ Compose V2 — không cần ghi nữa.

```yaml
# compose.yaml
name: myapp                            # Tên project (mặc định = tên folder)

services:                              # Khai báo các container
  web:
    build:
      context: .                       # Thư mục chứa Dockerfile
      dockerfile: Dockerfile.prod      # Tên file (nếu khác mặc định)
      args:
        BUILD_VERSION: "1.0"
    image: myapp:latest                # Tag để cache; tag image khi build
    container_name: myapp-web          # Tên cố định (mặc định: myapp-web-1)
    restart: unless-stopped
    ports:
      - "3000:3000"                    # host:container
      - "127.0.0.1:9229:9229"          # Bind localhost
    environment:
      NODE_ENV: production
      DB_HOST: db                      # Tên service = hostname (DNS tự động)
      DB_PASSWORD_FILE: /run/secrets/db_password
    env_file:
      - .env                           # Load thêm từ file
    depends_on:
      db:
        condition: service_healthy     # Chờ db healthy mới start
      cache:
        condition: service_started     # Chỉ chờ start (mặc định)
    volumes:
      - ./src:/app/src                 # Bind mount (dev)
      - logs:/app/logs                 # Named volume
    networks:
      - app-net
    secrets:
      - db_password
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 10s
    deploy:                            # Chỉ áp dụng cho Swarm
      resources:
        limits:
          cpus: "1.0"
          memory: 512M
        reservations:
          cpus: "0.5"
          memory: 256M
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"

  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: mydb
      POSTGRES_USER: user
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - app-net
    secrets:
      - db_password
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user -d mydb"]
      interval: 10s
      timeout: 5s
      retries: 5

  cache:
    image: redis:7-alpine
    restart: unless-stopped
    networks:
      - app-net
    command: redis-server --appendonly yes
    volumes:
      - cache-data:/data

volumes:
  db-data:                             # Named volume — Docker quản lý
  cache-data:
  logs:

networks:
  app-net:
    driver: bridge
    # ipam:
    #   config:
    #     - subnet: 172.28.0.0/16

secrets:
  db_password:
    file: ./secrets/db_password.txt    # File chứa password (đừng commit!)
```

## 2. Các lệnh Compose

### 2.1 Lifecycle

```bash
docker compose up                       # Khởi động foreground (xem log)
docker compose up -d                    # Detached
docker compose up -d --build            # Build lại image trước khi start
docker compose up -d --force-recreate   # Tạo lại container kể cả khi không đổi
docker compose up -d --no-deps web      # Không khởi động dependency
docker compose up -d web db             # Chỉ khởi động service được chỉ định

docker compose down                     # Dừng và xóa container + network
docker compose down -v                  # Bao gồm cả volume (MẤT DATA)
docker compose down --rmi all           # Xóa cả image

docker compose stop                     # Dừng (giữ container)
docker compose start                    # Khởi động lại container đã stop
docker compose restart web              # Restart 1 service
docker compose pause web                # Freeze process
docker compose unpause web
```

### 2.2 Inspect / Logs / Exec

```bash
docker compose ps                       # Trạng thái service
docker compose ps -a                    # Bao gồm cả stopped
docker compose top                      # Process trong từng service
docker compose stats                    # CPU/RAM realtime

docker compose logs                     # Log tất cả (1 lần)
docker compose logs -f                  # Follow
docker compose logs -f --tail 100 web   # 100 dòng cuối của `web`
docker compose logs --since 10m         # Log 10 phút gần nhất

docker compose exec web bash            # Vào shell service đang chạy
docker compose exec -u root web bash    # Với root
docker compose run --rm web npm test    # Tạo container mới chạy lệnh (rồi xóa)
docker compose run --rm web sh -c "echo $DB_HOST"
```

> `exec` chạy trong container **đang chạy**, `run` tạo container **mới** (có thể có ports/volumes khác).

### 2.3 Build / Pull / Push

```bash
docker compose build                    # Build tất cả service có `build:`
docker compose build --no-cache web
docker compose build --pull             # Pull base image mới nhất
docker compose pull                     # Pull image cho tất cả service
docker compose pull db                  # Pull image cho service cụ thể
docker compose push                     # Push image (cần đã build + login)
```

### 2.4 Config / Validation

```bash
docker compose config                   # In ra config đã merge và validate
docker compose config --services        # Liệt kê tên service
docker compose config --volumes
docker compose config --quiet           # Chỉ kiểm tra syntax (silent)
```

### 2.5 Scale

```bash
docker compose up -d --scale web=3      # Chạy 3 instance `web` (cần dùng `ports` dạng range hoặc bỏ ports)
```

## 3. Các pattern phổ biến

### 3.1 Override file cho dev/prod

Compose mặc định merge `compose.yaml` + `compose.override.yaml`. Có thể chỉ định file khác qua `-f`.

**compose.yaml** (chung):
```yaml
services:
  web:
    image: myapp:latest
    environment:
      NODE_ENV: production
```

**compose.override.yaml** (dev — tự load mặc định):
```yaml
services:
  web:
    build: .
    volumes:
      - ./src:/app/src                  # Hot reload
    environment:
      NODE_ENV: development
    ports:
      - "9229:9229"                     # Debug port
```

**compose.prod.yaml** (prod — chỉ định thủ công):
```yaml
services:
  web:
    image: myorg/myapp:1.2.3
    deploy:
      replicas: 3
    logging:
      driver: gelf
      options:
        gelf-address: udp://graylog:12201
```

Chạy:
```bash
docker compose up -d                                           # = compose.yaml + override.yaml
docker compose -f compose.yaml -f compose.prod.yaml up -d      # Prod
```

### 3.2 Profiles (bật/tắt service)

```yaml
services:
  web:
    image: nginx
  db:
    image: postgres:16-alpine
  debug:
    image: nicolaka/netshoot
    profiles: ["debug"]
    command: sleep infinity

  metrics:
    image: prom/prometheus
    profiles: ["monitoring"]
```

```bash
docker compose up -d                            # Chỉ web + db
docker compose --profile debug up -d            # web + db + debug
docker compose --profile monitoring --profile debug up -d
```

### 3.3 Biến môi trường + `.env`

File `.env` ở cùng thư mục với `compose.yaml` được Compose tự load để **thay biến trong YAML**:

**.env**:
```bash
TAG=1.2.3
DB_USER=app
DB_PASSWORD=supersecret
```

**compose.yaml**:
```yaml
services:
  web:
    image: myapp:${TAG:-latest}            # Mặc định "latest" nếu không có
    environment:
      DB_USER: ${DB_USER}
      DB_PASSWORD: ${DB_PASSWORD:?error}   # Bắt buộc, fail nếu thiếu
```

> **`.env` của Compose** ≠ `env_file:` của service. `env_file:` truyền biến **vào container**; `.env` chỉ thay biến **trong YAML lúc parse**.

### 3.4 Healthcheck + depends_on (chờ DB sẵn sàng)

```yaml
services:
  web:
    depends_on:
      db:
        condition: service_healthy

  db:
    image: postgres:16-alpine
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $$POSTGRES_USER"]
      interval: 5s
      timeout: 5s
      retries: 10
      start_period: 10s
```

Các condition:
- `service_started` (mặc định): chỉ chờ container start.
- `service_healthy`: chờ healthcheck pass.
- `service_completed_successfully`: chờ container exit code 0 (cho init job).

### 3.5 Init / one-off job

```yaml
services:
  migrate:
    build: .
    command: ["npm", "run", "migrate"]
    depends_on:
      db:
        condition: service_healthy

  web:
    build: .
    depends_on:
      migrate:
        condition: service_completed_successfully
```

### 3.6 Secret (file-based)

```yaml
services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password
    secrets:
      - db_password

secrets:
  db_password:
    file: ./secrets/db_password.txt
```

Compose mount file thành `/run/secrets/db_password` trong container — không leak vào env hay image layer.

### 3.7 Extension fields (DRY)

```yaml
x-common-env: &common-env
  TZ: Asia/Ho_Chi_Minh
  LOG_LEVEL: info

x-logging: &default-logging
  driver: json-file
  options:
    max-size: "10m"
    max-file: "3"

services:
  web:
    image: myapp
    environment: *common-env
    logging: *default-logging

  worker:
    image: myapp
    command: ["node", "worker.js"]
    environment: *common-env
    logging: *default-logging
```

### 3.8 Custom network (bridge isolation)

```yaml
services:
  frontend:
    image: nginx
    networks:
      - public
      - internal

  backend:
    image: myapp
    networks:
      - internal
      - db-net

  db:
    image: postgres
    networks:
      - db-net           # Chỉ backend thấy db

networks:
  public:
    driver: bridge
  internal:
    driver: bridge
    internal: true       # Không có internet
  db-net:
    driver: bridge
    internal: true
```

## 5. Compose vs Swarm vs Kubernetes

| | Docker Compose | Docker Swarm | Kubernetes |
|---|---|---|---|
| Phạm vi | Single-host | Multi-host (Docker native) | Multi-host (chuẩn industry) |
| File | `compose.yaml` | `compose.yaml` + `deploy:` | YAML manifest riêng |
| Use case | Dev, demo, single-node prod | Small/medium prod | Production scale lớn |
| Học | Dễ | Trung bình | Khó |

> Compose file V3+ có section `deploy:` chỉ áp dụng khi `docker stack deploy` (Swarm). Khi `docker compose up`, các trường như `replicas`, `placement` sẽ bị bỏ qua.

Swarm chi tiết: [11.Docker_Swarm.md](11.Docker_Swarm.md).
