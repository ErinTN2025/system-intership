# Dockerfile

**Dockerfile** là file text chứa các instruction để build một Docker image. Mỗi instruction tạo ra một **layer** trong image.

## 1. Cấu trúc cơ bản

```dockerfile
# syntax=docker/dockerfile:1.7    # Bật BuildKit syntax mới (khuyến nghị)
FROM ubuntu:24.04                  # Base image (LUÔN là instruction đầu tiên, trừ ARG)

LABEL maintainer="dev@example.com" \
      version="1.0" \
      description="My app"

ENV NODE_ENV=production            # Biến môi trường (có trong cả build và runtime)
ARG BUILD_VERSION=1.0              # Biến chỉ tồn tại lúc build

WORKDIR /app                       # Thư mục làm việc (mkdir + cd)

COPY package*.json ./              # Copy file vào image
RUN apt-get update && \
    apt-get install -y curl && \
    rm -rf /var/lib/apt/lists/*    # Thực thi lệnh khi build

EXPOSE 3000                        # Tài liệu hóa port (KHÔNG tự map ra host)
VOLUME ["/data"]                   # Mount point cho volume

USER appuser                       # Chạy với user không phải root

HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost/ || exit 1

ENTRYPOINT ["node"]                # Lệnh cố định khi container chạy
CMD ["server.js"]                  # Tham số mặc định (có thể override bởi docker run)
```

## 2. Các Instruction chi tiết

### 2.1 FROM

```dockerfile
FROM ubuntu:24.04                  # Image cụ thể
FROM scratch                       # Image rỗng tuyệt đối (dùng cho binary Go, Rust)
FROM node:22-alpine AS builder     # Đặt alias cho multi-stage
FROM --platform=linux/amd64 nginx  # Force platform
```

> **Luôn dùng tag cụ thể** (`node:22-alpine`), tránh `latest` — không tái lập được build.

### 2.2 RUN

Thực thi lệnh trong layer mới. Có 2 dạng:

```dockerfile
RUN apt-get update && apt-get install -y curl     # Shell form (qua /bin/sh -c)
RUN ["apt-get", "install", "-y", "curl"]          # Exec form (không qua shell)
```

**Gộp lệnh để giảm layer**:
```dockerfile
# ❌ Tạo 3 layer, layer 1 vẫn còn cache apt nặng
RUN apt-get update
RUN apt-get install -y curl
RUN rm -rf /var/lib/apt/lists/*

# ✅ 1 layer, không cache thừa
RUN apt-get update && \
    apt-get install -y --no-install-recommends curl ca-certificates && \
    rm -rf /var/lib/apt/lists/*
```

### 2.3 COPY vs ADD

```dockerfile
COPY src/ /app/src/               # Copy đơn giản (dùng đây)
COPY --chown=app:app file /app/   # Đặt owner khi copy
COPY --from=builder /out /app/    # Copy từ stage khác (multi-stage)
COPY --link src/ /app/src/        # BuildKit: layer độc lập, tối ưu cache

ADD archive.tar.gz /app/          # Tự giải nén tar
ADD https://example.com/file /opt/  # Tải URL về (KHÔNG khuyến khích)
```

> **Quy tắc**: Mặc định dùng `COPY`. Chỉ dùng `ADD` khi cần tự giải nén tar (hiếm).

### 2.4 ENV vs ARG

```dockerfile
ARG NODE_VERSION=22               # Chỉ tồn tại lúc build
ENV NODE_ENV=production           # Có cả lúc build và runtime

FROM node:${NODE_VERSION}-alpine  # ARG dùng được ở FROM

ARG BUILD_DATE                    # ARG không default — phải truyền --build-arg
RUN echo "Built at: $BUILD_DATE"
```

Truyền ARG khi build:
```bash
docker build --build-arg NODE_VERSION=20 --build-arg BUILD_DATE=$(date -u +%FT%TZ) .
```

> **Không bao giờ truyền secret qua ARG** — nó sẽ lưu vào image history. Dùng `--mount=type=secret`.

### 2.5 WORKDIR

```dockerfile
WORKDIR /app                      # mkdir -p /app && cd /app
COPY . .                          # Tương đối với WORKDIR
RUN npm install                   # Chạy trong /app
```

### 2.6 EXPOSE

Chỉ là **metadata** — KHÔNG tự map port ra host. Vẫn phải dùng `-p` khi `docker run`.

```dockerfile
EXPOSE 3000
EXPOSE 80/tcp 443/tcp 53/udp
```

### 2.7 VOLUME

Đánh dấu thư mục sẽ được mount thành volume. Khi container chạy không có `-v`, Docker tự tạo anonymous volume.

```dockerfile
VOLUME ["/var/lib/postgresql/data"]
```

> Cẩn thận: instruction này thường gây bất ngờ. Nếu COPY/RUN sửa thư mục **sau** lệnh VOLUME, thay đổi sẽ bị bỏ qua.

### 2.8 USER

```dockerfile
RUN addgroup -S app && adduser -S app -G app
USER app                          # Tất cả lệnh sau chạy với user `app`
USER 1000:1000                    # UID:GID
```

> Production **bắt buộc** chạy non-root.

### 2.9 CMD vs ENTRYPOINT

Đây là điểm hay nhầm nhất. Cả hai đều định nghĩa "lệnh chạy khi container start", nhưng:

| | CMD | ENTRYPOINT |
|---|---|---|
| Mục đích | Tham số mặc định | Lệnh cố định |
| Override bởi `docker run` | ✅ Có | ❌ Không (cần `--entrypoint`) |
| Thường dùng | Khi container có nhiều cách chạy | Khi container hoạt động như executable |

**Dạng exec (khuyến nghị)** — không qua shell, nhận signal đúng:
```dockerfile
CMD ["node", "server.js"]
ENTRYPOINT ["node"]
```

**Dạng shell** — qua `/bin/sh -c`, KHÔNG nhận signal trực tiếp:
```dockerfile
CMD node server.js     # ❌ SIGTERM không tới được Node
```

**Kết hợp ENTRYPOINT + CMD** (pattern phổ biến):
```dockerfile
ENTRYPOINT ["node"]
CMD ["server.js"]
# docker run myapp           → node server.js
# docker run myapp --version → node --version
```

### 2.10 HEALTHCHECK

```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD curl -fsS http://localhost:3000/health || exit 1
```

| Flag | Mặc định | Mô tả |
|---|---|---|
| `--interval` | 30s | Khoảng giữa các lần check |
| `--timeout` | 30s | Quá thời gian này coi như fail |
| `--start-period` | 0s | Thời gian "warm-up" — fail trong giai đoạn này không tính |
| `--retries` | 3 | Số lần fail liên tiếp trước khi đánh dấu `unhealthy` |

Trạng thái: `starting` → `healthy` / `unhealthy`. Xem bằng `docker ps` hoặc `docker inspect`.

### 2.11 ONBUILD, SHELL, STOPSIGNAL

```dockerfile
ONBUILD COPY . /app                # Trigger khi image này được dùng làm base (hiếm dùng)
SHELL ["powershell", "-Command"]   # Đổi shell mặc định (Windows containers)
STOPSIGNAL SIGINT                  # Signal Docker gửi khi `docker stop` (mặc định SIGTERM)
```

## 3. Multi-stage Build (best practice 2026)

Dùng nhiều `FROM` trong một Dockerfile: stage đầu để build, stage cuối chỉ copy artifact → **image production nhỏ gọn, không chứa compiler hay build tool**.

### 3.1 Ví dụ Node.js

```dockerfile
# syntax=docker/dockerfile:1.7

# ----- Stage 1: Build -----
FROM node:22-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci                                # `ci` nhanh hơn `install`, tái lập tốt hơn
COPY . .
RUN npm run build

# ----- Stage 2: Production -----
FROM node:22-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

RUN addgroup -S app && adduser -S app -G app
COPY --from=builder --chown=app:app /app/dist ./dist
COPY --from=builder --chown=app:app /app/node_modules ./node_modules
COPY --from=builder --chown=app:app /app/package.json ./

USER app
EXPOSE 3000
CMD ["node", "dist/server.js"]
```

### 3.2 Ví dụ Go (image siêu nhỏ)

```dockerfile
# ----- Stage 1: Build -----
FROM golang:1.23-alpine AS builder
WORKDIR /src
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-w -s" -o /out/app ./cmd/server

# ----- Stage 2: Distroless -----
FROM gcr.io/distroless/static-debian12:nonroot
COPY --from=builder /out/app /app
USER nonroot:nonroot
EXPOSE 8080
ENTRYPOINT ["/app"]
```

Image cuối ~10-20MB, không có shell, không có package manager → attack surface cực thấp.

### 3.3 Build stage cụ thể

```bash
docker build --target=builder -t myapp:dev .   # Chỉ build đến stage `builder`
```

## 4. BuildKit Mount (tối ưu tốc độ)

Bật BuildKit (mặc định từ Docker 23+). Thêm `# syntax=docker/dockerfile:1.7` ở dòng đầu.

### 4.1 Cache mount

```dockerfile
# Cache thư mục npm — không tải lại mỗi lần build
RUN --mount=type=cache,target=/root/.npm \
    npm ci

# Cache apt
RUN --mount=type=cache,target=/var/cache/apt,sharing=locked \
    --mount=type=cache,target=/var/lib/apt,sharing=locked \
    apt-get update && apt-get install -y curl

# Cache Go module
RUN --mount=type=cache,target=/go/pkg/mod \
    --mount=type=cache,target=/root/.cache/go-build \
    go build -o /out/app ./cmd/server
```

### 4.2 Bind mount (đọc nhanh, không copy)

```dockerfile
RUN --mount=type=bind,source=.,target=/src \
    cd /src && go build -o /out/app ./cmd/server
```

### 4.3 Secret mount (an toàn)

```dockerfile
# Dockerfile
RUN --mount=type=secret,id=npmrc,target=/root/.npmrc \
    npm ci
```

```bash
# Build
docker build --secret id=npmrc,src=$HOME/.npmrc -t myapp:1.0 .
```

Secret KHÔNG bị lưu vào image layer — kiểm tra bằng `docker image history`.

### 4.4 SSH mount (clone private repo)

```dockerfile
RUN --mount=type=ssh \
    git clone git@github.com:myorg/private.git
```

```bash
docker build --ssh default=$SSH_AUTH_SOCK -t myapp:1.0 .
```

## 5. .dockerignore

Tương tự `.gitignore` — loại trừ file khỏi build context, giúp build nhanh hơn và image nhỏ hơn.

```gitignore
# Git
.git
.gitignore

# Node
node_modules
npm-debug.log

# Python
__pycache__
*.pyc
.venv
venv

# Build artifact
dist
build
target
*.exe

# Editor
.vscode
.idea
*.swp

# Secret (rất quan trọng!)
.env
.env.*
*.pem
*.key
id_rsa*

# OS
.DS_Store
Thumbs.db
```

## 6. Best Practices

| Nguyên tắc | Lý do |
|---|---|
| Dùng image tag cụ thể (`node:22-alpine`) | Tránh build "trôi" do tag `latest` đổi |
| Dùng `alpine`/`distroless` cho production | Image nhỏ, ít CVE |
| Gộp `RUN` liên quan thành một lệnh | Giảm số layer |
| Copy `package.json` trước, source sau | Tận dụng cache layer dependencies |
| `apt-get update && install` trong cùng `RUN` | Tránh cache update cũ + install mới (gây lỗi) |
| Xóa `/var/lib/apt/lists/*` ngay trong `RUN` | Không để cache vào layer |
| Dùng `--no-install-recommends` với apt | Bớt package thừa |
| Chạy bằng `USER` không phải root | Bảo mật |
| Dùng `COPY` thay vì `ADD` | `ADD` có hành vi ẩn |
| Dùng multi-stage build | Image production gọn |
| Dùng `.dockerignore` | Build context nhỏ, không leak secret |
| Pin version mọi thứ (`npm ci`, `pip --require-hashes`) | Tái lập build |
| `EXPOSE` đúng port — dùng số nhỏ hơn 1024 chỉ khi cần | Documentation rõ |
| Đặt `HEALTHCHECK` | Compose/Swarm/K8s biết khi nào container sẵn sàng |
| Exec form cho CMD/ENTRYPOINT | Signal (SIGTERM) tới đúng process |

## 7. Ví dụ hoàn chỉnh — Python Flask

```dockerfile
# syntax=docker/dockerfile:1.7
FROM python:3.12-slim AS builder

WORKDIR /app
COPY requirements.txt .
RUN --mount=type=cache,target=/root/.cache/pip \
    pip install --prefix=/install -r requirements.txt

# ----- Production -----
FROM python:3.12-slim

RUN groupadd -r app && useradd -r -g app app

WORKDIR /app
COPY --from=builder /install /usr/local
COPY --chown=app:app . .

USER app
EXPOSE 5000

HEALTHCHECK --interval=30s --timeout=3s --start-period=10s \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:5000/health')" || exit 1

CMD ["gunicorn", "-b", "0.0.0.0:5000", "app:app"]
```
