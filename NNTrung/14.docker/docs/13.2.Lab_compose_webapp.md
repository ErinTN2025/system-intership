# Lab 2 — Web App đa container với Docker Compose

## Mục tiêu

Dựng một stack ứng dụng thật: **Flask web app + PostgreSQL + Redis**, dùng Docker Compose.

Sau lab, bạn sẽ làm được:
- Viết `compose.yaml` đa service.
- Build image từ Dockerfile trong Compose.
- Liên kết container qua network nội bộ + DNS tự động.
- Đặt healthcheck + `depends_on`.
- Dùng named volume cho data persistent.
- Dùng `.env` để cấu hình.
- Init DB schema tự động.
- Dev workflow với bind mount hot-reload.

## Yêu cầu

- Docker Engine + Compose V2:
  ```bash
  docker compose version
  # Docker Compose version v2.x.x
  ```

## Bước 2 — Viết Flask app

File `app/app.py`:
```python
import os
import time
from flask import Flask, jsonify
import psycopg2
import redis

app = Flask(__name__)

DB_HOST = os.getenv("DB_HOST", "db")
DB_NAME = os.getenv("DB_NAME", "mydb")
DB_USER = os.getenv("DB_USER", "app")
DB_PASSWORD = os.getenv("DB_PASSWORD", "secret")
REDIS_HOST = os.getenv("REDIS_HOST", "cache")


def db_conn():
    return psycopg2.connect(
        host=DB_HOST, dbname=DB_NAME, user=DB_USER, password=DB_PASSWORD
    )


def cache_conn():
    return redis.Redis(host=REDIS_HOST, port=6379, decode_responses=True)


@app.get("/health")
def health():
    return jsonify(status="ok"), 200


@app.get("/")
def index():
    r = cache_conn()
    count = r.incr("hits")
    return jsonify(
        message="Hello from Flask in Docker!",
        hits=count,
        hostname=os.uname().nodename,
    )


@app.get("/users")
def users():
    conn = db_conn()
    cur = conn.cursor()
    cur.execute("SELECT id, name, email FROM users ORDER BY id;")
    rows = cur.fetchall()
    cur.close()
    conn.close()
    return jsonify([{"id": r[0], "name": r[1], "email": r[2]} for r in rows])


if __name__ == "__main__":
    # Đợi DB sẵn sàng (depends_on healthcheck thường đủ, đây là backup)
    for _ in range(30):
        try:
            db_conn().close()
            break
        except Exception:
            time.sleep(1)
    app.run(host="0.0.0.0", port=5000)
```

File `app/requirements.txt`:
```
flask==3.0.3
psycopg2-binary==2.9.10
redis==5.2.0
gunicorn==23.0.0
```

File `app/Dockerfile`:
```dockerfile
# syntax=docker/dockerfile:1.7
FROM python:3.12-slim

WORKDIR /app

# Cài deps trước (cache layer)
COPY requirements.txt .
RUN --mount=type=cache,target=/root/.cache/pip \
    pip install -r requirements.txt

# Copy source
COPY . .

# Non-root user
RUN useradd -r -u 1000 app && chown -R app:app /app
USER app

EXPOSE 5000

HEALTHCHECK --interval=15s --timeout=3s --start-period=10s --retries=3 \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:5000/health')" || exit 1

CMD ["gunicorn", "-b", "0.0.0.0:5000", "--workers", "2", "app:app"]
```

File `app/.dockerignore`:
```

*.pyc
.venv
venv
.env
.git
```

## Bước 4 — Viết compose.yaml

File `compose.yaml`:
```yaml
name: composeapp

services:
  web:
    build:
      context: ./app
    image: composeapp-web:dev
    restart: unless-stopped
    ports:
      - "8000:5000"
    environment:
      DB_HOST: db
      DB_NAME: ${POSTGRES_DB}
      DB_USER: ${POSTGRES_USER}
      DB_PASSWORD: ${POSTGRES_PASSWORD}
      REDIS_HOST: cache
    depends_on:
      db:
        condition: service_healthy
      cache:
        condition: service_started
    networks:
      - app-net

  db:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: ${POSTGRES_DB}
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - db-data:/var/lib/postgresql/data
      - ./db-init:/docker-entrypoint-initdb.d:ro
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $${POSTGRES_USER} -d $${POSTGRES_DB}"]
      interval: 5s
      timeout: 5s
      retries: 10
    networks:
      - app-net
    # Không publish port ra host — DB chỉ accessible từ network nội bộ

  cache:
    image: redis:7-alpine
    restart: unless-stopped
    command: redis-server --appendonly yes
    volumes:
      - cache-data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5
    networks:
      - app-net

volumes:
  db-data:
  cache-data:

networks:
  app-net:
    driver: bridge
```

File `.env`:
```bash
POSTGRES_DB=mydb
POSTGRES_USER=app
POSTGRES_PASSWORD=SuperSecret123
```

> Lưu ý `$${POSTGRES_USER}` trong healthcheck: `$$` để Compose không thay biến trước khi gửi vào shell (cho phép shell trong container expand).

## Bước 10 — Dev workflow (hot reload)

Tạo `compose.override.yaml` (Compose tự load — chỉ áp dụng khi `docker compose up` mặc định):

```yaml
# compose.override.yaml
services:
  web:
    volumes:
      - ./app:/app:ro          # Bind mount source
    environment:
      FLASK_ENV: development
    command: ["python", "-u", "app.py"]    # Flask dev server (không phải gunicorn)
```

Sửa `app/app.py` thử thêm endpoint:
```python
@app.get("/version")
def version():
    return jsonify(version="1.0.1")
```

```bash
docker compose up -d
# Hot reload — đổi code → restart container nhanh (không build lại)
docker compose restart web

curl http://localhost:8000/version
# {"version":"1.0.1"}
```

Khi cần chạy như prod (skip override):
```bash
docker compose -f compose.yaml up -d
```

## Bước 11 — Scale service

```bash
# 3 instance web, Compose tự load balance qua DNS round-robin
docker compose up -d --scale web=3

docker compose ps
# composeapp-web-1, -web-2, -web-3
```

> Cần bỏ `ports:` static để scale, hoặc dùng range `"8000-8002:5000"`. Để demo phía trên hoạt động, có thể bỏ port mapping và đặt một service reverse proxy phía trước (nginx upstream — bài tập mở rộng).

## Tổng kết

| Khái niệm | Áp dụng trong lab |
|---|---|
| Multi-service compose | `web` + `db` + `cache` |
| Service-to-service DNS | `DB_HOST=db`, `REDIS_HOST=cache` |
| Healthcheck + `depends_on: service_healthy` | Web chờ DB ready mới start |
| Named volume | `db-data`, `cache-data` persist data |
| Bind mount config | `./db-init` cho schema, `./app` cho hot-reload |
| `.env` + variable substitution | `${POSTGRES_PASSWORD}` |
| Build image trong compose | `build: ./app` |
| Override file cho dev | `compose.override.yaml` |
| Auto schema init | Postgres `/docker-entrypoint-initdb.d/` |
| Internal network | DB không expose port host |

## Thử thách

1. Thêm service `nginx` làm reverse proxy phía trước `web`, scale `web` lên 3 instance, để nginx load balance.
2. Thêm `pgadmin` service (image `dpage/pgadmin4`) để quản lý DB qua GUI.
3. Tạo `compose.prod.yaml` không có bind mount, dùng image đã tag rõ ràng, đặt resource limits.
4. Thay `.env` password bằng **Compose secret** (file `secrets/db_password.txt`, mount thành `/run/secrets/db_password`, app đọc từ file).
5. Bật Postgres `wal_level=logical` + `pg_dump` định kỳ — viết service `backup` chạy mỗi 1 giờ.
