# Lab 1 — Docker cơ bản với Nginx

## Mục tiêu

Sau lab này, bạn sẽ làm được:
- Pull image từ registry, chạy container.
- Map port host ↔ container.
- Mount file/folder vào container.
- Đọc log, exec vào shell container.
- Tạo network riêng cho nhiều container liên lạc với nhau.
- Quản lý vòng đời container.

## Yêu cầu

- Đã cài Docker Engine (xem [03.Docker_install.md](03.Docker_install.md)).
- Kiểm tra:
  ```bash
  docker --version
  docker run --rm hello-world
  ```

## Bước 1 — Chạy Nginx container đầu tiên

```bash
docker run -d --name web1 -p 8080:80 nginx:1.27-alpine
```

Giải thích từng flag:
- `-d`: detached (chạy ngầm).
- `--name web1`: đặt tên container.
- `-p 8080:80`: map port 8080 trên host → port 80 trong container.
- `nginx:1.27-alpine`: image (alpine ~25MB, nhỏ hơn nginx mặc định ~190MB).

Verify:
```bash
docker ps                 # Thấy web1 đang chạy
curl http://localhost:8080
# <html>...Welcome to nginx!...</html>
```

Mở browser http://localhost:8080 — sẽ thấy trang welcome của Nginx.

## Bước 2 — Xem log và inspect

```bash
docker logs web1                          # Log một lần
docker logs -f web1                       # Follow realtime (Ctrl+C để thoát)
docker logs --tail 20 web1                # 20 dòng cuối

# Mở 1 terminal khác và curl vài lần
curl http://localhost:8080
curl http://localhost:8080/notfound
# Quay lại terminal log thấy các request

# Inspect — toàn bộ metadata
docker inspect web1

# Lấy field cụ thể
docker inspect web1 -f '{{.State.Status}}'           # running
docker inspect web1 -f '{{.NetworkSettings.IPAddress}}'   # 172.17.0.2 (default bridge)
```

## Bước 3 — Exec vào container

```bash
docker exec -it web1 sh
```

Trong container:
```sh
ls /etc/nginx/
cat /etc/nginx/nginx.conf
ps aux                            # Process: nginx master + workers
hostname                          # Hostname = container ID
exit
```

> Image alpine không có `bash` — dùng `sh`. Image debian/ubuntu có `bash`.

## Bước 4 — Custom HTML qua bind mount

Tạo file HTML mới:
```bash
mkdir -p ~/docker-lab/site
cat > ~/docker-lab/site/index.html <<'EOF'
<!DOCTYPE html>
<html>
<head><title>My Docker Lab</title></head>
<body style="font-family: sans-serif; padding: 40px;">
  <h1>Hello from inside a Docker container 🐳</h1>
  <p>This file is bind-mounted from the host.</p>
</body>
</html>
EOF
```

Xóa container cũ và chạy lại với bind mount:
```bash
docker stop web1 && docker rm web1

docker run -d \
  --name web1 \
  -p 8080:80 \
  -v ~/docker-lab/site:/usr/share/nginx/html:ro \
  nginx:1.27-alpine
```

Verify:
```bash
curl http://localhost:8080
# Output trang custom
```

Thử sửa file trên host:
```bash
echo '<h2>Edited live!</h2>' >> ~/docker-lab/site/index.html
curl http://localhost:8080
# Thay đổi reflect ngay — không cần restart container
```

> `:ro` = read-only. Container không ghi đè được file host — an toàn hơn.

## Bước 5 — Custom Nginx config

Tạo config riêng:
```bash
cat > ~/docker-lab/nginx.conf <<'EOF'
events {}
http {
  server {
    listen 80;
    server_name _;

    location / {
      root /usr/share/nginx/html;
      index index.html;
    }

    location /health {
      access_log off;
      return 200 "OK\n";
    }

    location /info {
      default_type text/plain;
      return 200 "Container: $hostname\nClient: $remote_addr\nTime: $time_local\n";
    }
  }
}
EOF
```

Chạy lại với config mount thêm:
```bash
docker stop web1 && docker rm web1

docker run -d \
  --name web1 \
  -p 8080:80 \
  -v ~/docker-lab/site:/usr/share/nginx/html:ro \
  -v ~/docker-lab/nginx.conf:/etc/nginx/nginx.conf:ro \
  nginx:1.27-alpine
```

Test:
```bash
curl http://localhost:8080/health        # OK
curl http://localhost:8080/info          # Container: ..., Client: ..., Time: ...
```

Nếu Nginx fail vì config sai, xem log:
```bash
docker logs web1
```

## Bước 6 — Resource limit

Chạy với giới hạn 128MB RAM + 0.5 CPU:
```bash
docker stop web1 && docker rm web1

docker run -d \
  --name web1 \
  -p 8080:80 \
  --memory=128m \
  --cpus=0.5 \
  -v ~/docker-lab/site:/usr/share/nginx/html:ro \
  nginx:1.27-alpine

docker stats web1 --no-stream
# CPU% / MEM USAGE / LIMIT thấy 128MiB
```

## Bước 7 — Custom network + container giao tiếp

Tạo bridge network riêng:
```bash
docker network create lab-net
```

Chạy 2 container Nginx + 1 container debug trong cùng network:
```bash
docker run -d --name web-a --network lab-net nginx:1.27-alpine
docker run -d --name web-b --network lab-net nginx:1.27-alpine
docker run -it --rm --network lab-net nicolaka/netshoot
```

Trong container netshoot:
```sh
# DNS tự động — gọi bằng tên container
curl http://web-a
curl http://web-b

nslookup web-a
# Address: 172.x.x.x

ping -c 3 web-a

# Cùng network → thấy nhau
# Container ở default bridge KHÔNG resolve được tên này
```

Cleanup:
```bash
docker rm -f web-a web-b
docker network rm lab-net
```

## Bước 8 — Healthcheck

Chạy với healthcheck:
```bash
docker run -d \
  --name web1 \
  -p 8080:80 \
  --health-cmd="wget -qO- http://localhost/ > /dev/null || exit 1" \
  --health-interval=10s \
  --health-timeout=3s \
  --health-retries=3 \
  -v ~/docker-lab/site:/usr/share/nginx/html:ro \
  nginx:1.27-alpine

# Chờ ~10s
docker ps
# STATUS: Up 30 seconds (healthy)

docker inspect web1 -f '{{.State.Health.Status}}'
docker inspect web1 -f '{{json .State.Health}}' | jq
```

Mô phỏng fail — đổi port internal cho healthcheck:
```bash
docker exec web1 sh -c "nginx -s stop"
sleep 30
docker ps
# STATUS: Up X (unhealthy)
```

## Bước 9 — Auto restart

```bash
docker run -d \
  --name web1 \
  --restart unless-stopped \
  -p 8080:80 \
  nginx:1.27-alpine

# Mô phỏng crash
docker kill web1
sleep 2
docker ps     # Vẫn thấy Up — Docker đã restart tự động
```

## Bước 10 — Cleanup

```bash
docker stop web1
docker rm web1
docker images
docker rmi nginx:1.27-alpine                    # Tùy chọn — xóa image
```

Hoặc dọn nhanh tất cả:
```bash
docker ps -aq | xargs -r docker rm -f           # Xóa mọi container
docker system prune -a                          # Xóa image, network không dùng
```

## Tổng kết — bạn đã học được

| Khái niệm | Lệnh |
|---|---|
| Pull + run | `docker run -d --name N -p H:C IMAGE` |
| Log | `docker logs [-f] [--tail N] NAME` |
| Exec vào shell | `docker exec -it NAME sh` |
| Inspect | `docker inspect NAME` |
| Bind mount | `-v /host/path:/cont/path[:ro]` |
| Resource limit | `--memory --cpus` |
| Custom network | `docker network create NET` |
| Healthcheck | `--health-cmd --health-interval` |
| Restart policy | `--restart unless-stopped` |

## Thử thách (challenge)

1. Chạy 2 Nginx container trên port 8080 và 8081, mỗi cái phục vụ trang HTML khác nhau.
2. Đổi log driver thành `json-file` với max-size 5MB, max-file 2.
3. Chạy Nginx bind chỉ vào `127.0.0.1` thay vì `0.0.0.0` (gợi ý: `-p 127.0.0.1:8080:80`). Test từ máy khác trong LAN — không truy cập được.
4. Đặt `STOPSIGNAL SIGQUIT` (Nginx graceful shutdown dùng SIGQUIT) — quan sát log khi `docker stop`.
