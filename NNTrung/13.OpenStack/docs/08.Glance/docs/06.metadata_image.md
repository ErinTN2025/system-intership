## 1. Metadata của Glance Image là gì?

### 1.1. Định nghĩa

**Metadata của Glance image** là tập hợp các **thuộc tính mô tả** (key-value pairs) đi kèm với mỗi image, được lưu trong **database** (MariaDB/MySQL), tách biệt hoàn toàn với file dữ liệu thực tế của image.

> **Ví dụ dễ hiểu:** Nếu file `.qcow2` là "quyển sách", thì metadata là "bìa sách + mục lục" — không phải nội dung, nhưng cho biết nội dung là gì, cần bao nhiêu tài nguyên để đọc, và ai được phép đọc.

### 1.2. Tại sao Metadata quan trọng?

| Vai trò | Giải thích |
|---|---|
| **Mô tả image** | Tên, định dạng, dung lượng, hệ điều hành — giúp người dùng biết đang dùng gì |
| **Kiểm soát tài nguyên** | `min_disk`, `min_ram` — Nova từ chối tạo VM nếu flavor không đủ |
| **Hướng dẫn hypervisor** | Các `hw_*` properties báo cho Nova/QEMU biết dùng driver nào |
| **Bảo mật** | Checksum, hash, chữ ký số — đảm bảo tính toàn vẹn dữ liệu |
| **Phân quyền** | Visibility, owner — xác định ai được dùng image |
| **Tự động hoá** | Cloud-init đọc metadata để cấu hình VM sau khi boot |
| **Lập lịch Nova** | ImagePropertiesFilter dùng metadata để chọn compute node phù hợp |

### 1.3. Cây phân loại metadata

```
Metadata của Glance Image
├── Core Properties        — Glance tự quản lý (id, name, status, size...)
├── Hardware Properties    — Nova/QEMU đọc để cấu hình VM (hw_disk_bus, hw_vif_model...)
├── OS Properties          — Mô tả hệ điều hành (os_distro, os_version...)
├── Security Properties    — Mã hóa, ký số, Secure Boot
├── Architecture Properties — Loại CPU, hypervisor (architecture, hypervisor_type...)
├── Resource Traits        — Yêu cầu phần cứng cho scheduler (trait:HW_CPU_X86_AVX2...)
└── Custom Properties      — Do người dùng/admin tự định nghĩa (key=value tùy ý)
```

### 1.4. Prefix quy ước

| Prefix | Nhóm | Ai đọc |
|---|---|---|
| _(không prefix)_ | Core | Glance, Nova, người dùng |
| `hw_` | Hardware | Nova, QEMU/KVM, libvirt |
| `os_` | Operating System | Horizon, Nova, cloud-init |
| `img_` | Image-level flags | Nova, Glance |
| `vmware_` | VMware-specific | VMware API driver |
| `trait:` | Resource traits | Nova Scheduler |
| _(tùy ý)_ | Custom | Người dùng định nghĩa |

> **Lưu ý quan trọng:** Prefix `os_glance` được **Glance giữ riêng** — người dùng và admin **không được phép** đặt property nào bắt đầu bằng `os_glance`. Glance tự dùng prefix này cho metadata nội bộ.

## 2. Các thuộc tính cốt lõi (Core Properties)

Đây là các thuộc tính do **Glance tự quản lý**, không do người dùng nhập tùy ý.

### 2.1. Thuộc tính định danh

| Thuộc tính | Kiểu | Mô tả | Ví dụ |
|---|---|---|---|
| `id` | UUID | ID duy nhất, Glance tự sinh | `abc12345-1234-...` |
| `name` | string | Tên hiển thị, do người dùng đặt | `Ubuntu-22.04-LTS` |
| `owner` | UUID | Project ID sở hữu image | `project-uuid` |

### 2.2. Thuộc tính định dạng

| Thuộc tính | Giá trị hợp lệ | Mô tả |
|---|---|---|
| `disk_format` | `raw`, `qcow2`, `vmdk`, `vhd`, `vdi`, `iso`, `ploop` | Định dạng file image |
| `container_format` | `bare`, `ovf`, `aki`, `ari`, `ami`, `docker`, `compressed` | Container bọc ngoài |

> **Thực tế:** Hầu hết image dùng `disk_format=qcow2` và `container_format=bare`.

### 2.3. Thuộc tính trạng thái và vòng đời

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `status` | `queued` → `saving` → `active` \| `killed` \| `deleted` \| `deactivated` \| `importing` | Trạng thái hiện tại |
| `created_at` | ISO 8601 datetime | Thời điểm tạo |
| `updated_at` | ISO 8601 datetime | Thời điểm cập nhật cuối |

**Vòng đời trạng thái:**

```
queued ──► saving ──► active
                        │
              ┌─────────┼─────────┐
           killed   deactivated  deleted
```

### 2.4. Thuộc tính kích thước và toàn vẹn dữ liệu

| Thuộc tính | Kiểu | Mô tả |
|---|---|---|
| `size` | integer (bytes) | Kích thước file image thực tế |
| `virtual_size` | integer (bytes) | Kích thước ảo của đĩa sau khi giải nén |
| `checksum` | string | MD5 checksum (legacy) |
| `os_hash_algo` | string | Thuật toán hash mới, mặc định `sha512` |
| `os_hash_value` | string | Giá trị hash theo `os_hash_algo` |

> **Nên dùng** `os_hash_value` + `os_hash_algo` thay vì `checksum` (MD5 đã lỗi thời về bảo mật). Từ Glance Train trở đi, cặp "multihash" này là chuẩn xác minh toàn vẹn.

### 2.5. Thuộc tính phân quyền và chia sẻ

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `visibility` | `public` / `private` / `shared` / `community` | Mức độ hiển thị |
| `protected` | `true` / `false` | Khi `true`: không ai xóa được image này |
| `member_status` | `accepted` / `rejected` / `pending` | Trạng thái nhận shared image (phía member) |

**Bảng so sánh visibility:**

| Visibility | Ai thấy | Ai dùng được |
|---|---|---|
| `public` | Mọi project | Mọi project |
| `community` | Mọi project | Mọi project (không cần accept) |
| `shared` | Owner + members được mời | Owner + members đã `accepted` |
| `private` | Chỉ owner project | Chỉ owner project |

### 2.6. Thuộc tính giới hạn tài nguyên

| Thuộc tính | Đơn vị | Mô tả |
|---|---|---|
| `min_disk` | GB | Dung lượng đĩa tối thiểu — Nova từ chối nếu flavor disk nhỏ hơn |
| `min_ram` | MB | RAM tối thiểu — Nova từ chối nếu flavor RAM nhỏ hơn |

```bash
# Ví dụ: Windows cần tối thiểu 40GB disk và 2048MB RAM
openstack image set \
    --min-disk 40 \
    --min-ram 2048 \
    "Windows-Server-2022"
```

### 2.7. Tags

Tags là danh sách nhãn tự do, không phải key-value, dùng để phân loại và lọc image.

```bash
# Thêm tag
openstack image set --tag ubuntu --tag lts --tag production <image-id>

# Tìm image theo tag
openstack image list --tag ubuntu
```

## 3. Các thuộc tính phần cứng (Hardware Properties)

Nhóm `hw_*` — Nova đọc và truyền xuống hypervisor (QEMU/KVM/libvirt) để cấu hình phần cứng ảo. **Hầu hết chỉ hoạt động với driver libvirt.**

### 3.1. CPU

| Thuộc tính | Kiểu | Giá trị hợp lệ | Mô tả |
|---|---|---|---|
| `hw_cpu_sockets` | int | ≥ 1 | Số socket CPU ảo |
| `hw_cpu_cores` | int | ≥ 1 | Số core trên mỗi socket |
| `hw_cpu_threads` | int | ≥ 1 | Số thread trên mỗi core |
| `hw_cpu_policy` | str | `shared` (default), `dedicated` | `dedicated` = vCPU pinning vào pCPU cụ thể |
| `hw_cpu_thread_policy` | str | `prefer` (default), `isolate`, `require` | Chính sách sibling thread (SMT/hyperthreading) |
| `hw_pmu` | bool | `true` / `false` | Bật/tắt vPMU (tắt để giảm latency cho workload realtime) |

**Giải thích `hw_cpu_thread_policy`:**

| Giá trị | Hành vi |
|---|---|
| `prefer` | Ưu tiên dùng sibling thread nếu có, không bắt buộc |
| `isolate` | Mỗi vCPU chạy trên một core riêng, không dùng sibling thread |
| `require` | Bắt buộc phải có sibling thread — lỗi nếu host không hỗ trợ SMT |

```bash
# CPU pinning cho workload HPC
openstack image set \
    --property hw_cpu_policy=dedicated \
    --property hw_cpu_thread_policy=isolate \
    "HPC-Image"
```

### 3.2. Bộ nhớ (Memory)

| Thuộc tính | Kiểu | Giá trị | Mô tả |
|---|---|---|---|
| `hw_mem_page_size` | str | `small`, `large`, `any` | Kích thước memory page. `large` = HugePages (giảm TLB miss) |
| `hw_numa_nodes` | int | ≥ 1 | Số NUMA node ảo |
| `hw_mem_encryption` | bool | `true` / `false` | Bật mã hóa RAM ở mức phần cứng (AMD SEV) |
| `hw_mem_encryption_model` | str | `amd-sev`, `amd-sev-es` | Model mã hóa RAM phần cứng cụ thể |

> `hw_mem_encryption` yêu cầu host hỗ trợ AMD SEV và libvirt driver với KVM/QEMU. Xem thêm tài liệu Nova về cấu hình AMD SEV.

### 3.3. Disk & Storage

| Thuộc tính | Giá trị hợp lệ | Mô tả |
|---|---|---|
| `hw_disk_bus` | `virtio` (KVM tốt nhất), `scsi`, `ide`, `sata`, `usb`, `xen`, `uml`, `lxc` | Bus đĩa ảo |
| `hw_scsi_model` | `virtio-scsi`, `lsilogic` | Model SCSI controller (khi `hw_disk_bus=scsi`) |
| `hw_cdrom_bus` | `ide`, `scsi` | Bus cho ổ CD/DVD ảo |
| `hw_serial_port_count` | int | Số cổng serial ảo |
| `hw_rescue_bus` | `scsi`, `virtio`, `ide`, `usb` | Bus dùng khi boot rescue mode |
| `hw_rescue_device` | `cdrom`, `disk`, `floppy` | Loại thiết bị khi boot rescue mode |

> **Best practice:** Luôn dùng `hw_disk_bus=virtio` cho image Linux trên KVM để đạt hiệu năng I/O tốt nhất.

### 3.4. Mạng (Network)

| Thuộc tính | Giá trị hợp lệ | Mô tả |
|---|---|---|
| `hw_vif_model` | `virtio` (Linux tốt nhất), `e1000`, `e1000e`, `rtl8139`, `ne2k_pci`, `pcnet`, `vmxnet3`, `igb` | Model card mạng ảo |
| `hw_vif_multiqueue_enabled` | `true` / `false` | Bật virtio-net multiqueue (tăng throughput mạng) |
| `hw_virtio_packed_ring` | `true` / `false` | Bật Packed VIRT-IO Queue (cải thiện hiệu năng gói tin nhỏ) |

**Gợi ý lựa chọn NIC model:**

| Model | Dùng khi nào |
|---|---|
| `virtio` | Linux với driver virtio (hiệu năng cao nhất trên KVM) |
| `e1000` | Windows hoặc OS cũ không có driver virtio |
| `vmxnet3` | VMware guest tools |

### 3.5. Hiển thị (Video & Display)

| Thuộc tính | Giá trị hợp lệ | Mô tả |
|---|---|---|
| `hw_video_model` | `vga`, `cirrus`, `vmvga`, `xen`, `qxl`, `virtio`, `gop`, `bochs`, `none` | Card đồ họa ảo. `none` = tắt GPU ảo (dùng với GPU passthrough) |
| `hw_video_ram` | int (MB) | VRAM tối đa của card đồ họa ảo |
| `hw_pointer_model` | `usbtablet` | Thiết bị con trỏ cho framebuffer đồ họa |

> `none` cho `hw_video_model` thường dùng khi cắm GPU vật lý (passthrough) vào VM — không cần GPU ảo nữa.

### 3.6. Boot & Firmware

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `hw_firmware_type` | `bios` (default), `uefi` | Loại firmware. `uefi` bắt buộc cho Secure Boot và Windows 11 |
| `hw_firmware_stateless` | `true` / `false` | Boot stateless firmware — config không lưu giữa các lần reboot. Yêu cầu UEFI |
| `hw_machine_type` | `pc`, `q35` (KVM); `hyperv-gen1`, `hyperv-gen2` (Hyper-V) | Chipset ảo. `q35` hỗ trợ PCIe đầy đủ |
| `hw_boot_menu` | `true` / `false` | Hiện menu boot BIOS khi khởi động |

**Lựa chọn machine type:**

| Machine type | Ưu điểm |
|---|---|
| `pc` | Tương thích rộng, chipset i440fx cũ |
| `q35` | PCIe, cần cho một số driver SCSI/NVMe hiện đại, khuyên dùng cho VM mới |

### 3.7. Watchdog, RNG & Guest Agent

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `hw_watchdog_action` | `disabled` (default), `reset`, `poweroff`, `pause`, `none` | Hành động khi VM bị treo (thiết bị i6300ESB) |
| `hw_rng_model` | `virtio` | Thêm thiết bị RNG ảo. Yêu cầu flavor bật `hw_rng:allowed=True` |
| `hw_qemu_guest_agent` | `yes` / `no` | Bật QEMU Guest Agent trong VM |
| `hw_time_hpet` | `true` / `false` | Thêm HPET timer (chỉ áp dụng cho x86/x86_64 với QEMU) |

**Ví dụ cấu hình đầy đủ cho Ubuntu production trên KVM:**

```bash
openstack image set \
    --property hw_disk_bus=virtio \
    --property hw_scsi_model=virtio-scsi \
    --property hw_vif_model=virtio \
    --property hw_vif_multiqueue_enabled=true \
    --property hw_qemu_guest_agent=yes \
    --property hw_firmware_type=bios \
    --property hw_machine_type=q35 \
    --property hw_watchdog_action=reset \
    --property hw_rng_model=virtio \
    "Ubuntu-22.04-LTS"
```

## 4. Các thuộc tính hệ điều hành (OS Properties)

Nhóm `os_*` mô tả hệ điều hành bên trong image — dùng cho Horizon Dashboard, Nova scheduling, và cloud-init.

### 4.1. Thông tin hệ điều hành

| Thuộc tính | Giá trị ví dụ | Mô tả |
|---|---|---|
| `os_distro` | `ubuntu`, `centos`, `rhel`, `windows`, `debian`, `fedora`, `rocky`, `opensuse`... | Tên OS theo chuẩn [libosinfo](https://libosinfo.org/) — viết thường |
| `os_version` | `22.04`, `9`, `2022` | Phiên bản OS |
| `os_type` | `linux`, `windows` | Loại OS ảnh hưởng đến hành vi libvirt driver |
| `os_admin_user` | `ubuntu`, `cloud-user`, `Administrator` | Username admin mặc định khi SSH lần đầu |
| `os_command_line` | string | Kernel command line (libvirt) hoặc LXC arguments |

**Danh sách `os_distro` chuẩn (một số phổ biến):**

```
arch, centos, debian, fedora, freebsd, gentoo, mandrake, mandriva,
mes, msdos, netbsd, netware, openbsd, opensolaris, opensuse, rocky,
rhel, sled, ubuntu, windows
```

### 4.2. Hành vi OS

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `os_require_quiesce` | `yes` / `no` | Yêu cầu quiesce filesystem trước khi snapshot (cần Guest Agent) |
| `os_shutdown_timeout` | int (giây) | Thời gian chờ graceful shutdown. `0` = tắt ngay lập tức. Mặc định: 60 |

### 4.3. Config Drive

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `img_config_drive` | `mandatory`, `optional` (default) | Image có cần config drive không |

> Config drive là một ISO image nhỏ Nova đính kèm vào VM, chứa metadata và user-data. Dùng khi không có metadata service hoặc cloud-init cần dữ liệu ngay lúc boot.

**Ví dụ:**

```bash
openstack image set \
    --property os_distro=ubuntu \
    --property os_version=22.04 \
    --property os_type=linux \
    --property os_admin_user=ubuntu \
    --property os_shutdown_timeout=30 \
    "Ubuntu-22.04-LTS"
```

## 5. Thuộc tính bảo mật và xác thực (Security Properties)

### 5.1. Ký số image (Image Signing)

Glance tích hợp với **Barbican** (Key Manager) để ký số image. Nova có thể xác minh chữ ký trước khi dùng image, đảm bảo image không bị giả mạo hoặc thay đổi.

| Thuộc tính | Mô tả |
|---|---|
| `img_signature` | Chữ ký số của dữ liệu image (base64-encoded) |
| `img_signature_hash_method` | Thuật toán hash dùng để ký: `SHA-256`, `SHA-3-256` |
| `img_signature_key_type` | Loại khóa: `RSA-PSS`, `DSA`, `ECC-SECT571K1`, `ECC-SECT571R1`... |
| `img_signature_certificate_uuid` | UUID của certificate lưu trong Barbican |

**Luồng xác minh:**

```
Upload image → Glance lưu img_signature + certificate UUID
                    ↓
Nova download image → Lấy certificate từ Barbican
                    ↓
Nova verify chữ ký → OK thì tạo VM, FAIL thì từ chối
```

### 5.2. Secure Boot (UEFI)

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `os_secure_boot` | `required`, `optional`, `disabled` (default) | Yêu cầu Secure Boot. Bắt buộc với Windows 11 |

> Khi `os_secure_boot=required`:
> - `hw_firmware_type` **phải** là `uefi`
> - Host compute phải hỗ trợ Secure Boot

### 5.3. Ẩn hypervisor ID

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `img_hide_hypervisor_id` | `true` / `false` | Ẩn chữ ký hypervisor. Dùng khi một số driver (ví dụ NVIDIA) từ chối chạy trong VM |

### 5.4. Mã hóa RAM (AMD SEV)

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `hw_mem_encryption` | `true` / `false` | Bật mã hóa RAM phần cứng |
| `hw_mem_encryption_model` | `amd-sev`, `amd-sev-es` | Model cụ thể (SEV hoặc SEV-ES với register encryption) |

**Cấu hình cho Windows 11 với Secure Boot:**

```bash
openstack image set \
    --property hw_firmware_type=uefi \
    --property os_secure_boot=required \
    --property hw_machine_type=q35 \
    "Windows-11"
```

## 6. Thuộc tính kiến trúc và ảo hóa

Nhóm này mô tả kiến trúc phần cứng và loại hypervisor mà image hỗ trợ — Nova dùng để lọc compute node phù hợp khi scheduling.

### 6.1. Kiến trúc CPU

| Thuộc tính | Giá trị phổ biến | Mô tả |
|---|---|---|
| `architecture` | `x86_64`, `aarch64`, `i686`, `ppc64`, `ppc64le`, `s390x`, `armv7l`... | Kiến trúc CPU mà image hỗ trợ |

**Danh sách đầy đủ:** `aarch64`, `alpha`, `armv7l`, `cris`, `i686`, `ia64`, `lm32`, `m68k`, `microblaze`, `mips`, `mips64`, `openrisc`, `parisc`, `ppc`, `ppc64`, `s390`, `s390x`, `sparc`, `sparc64`, `x86_64`, `xtensa`...

### 6.2. Loại hypervisor

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `hypervisor_type` | `qemu`, `kvm`, `xen`, `hyperv`, `vmware`, `ironic`, `lxc`, `uml` | Loại hypervisor image tương thích. `qemu` dùng cho cả QEMU lẫn KVM |

### 6.3. Chế độ VM

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `vm_mode` | `hvm`, `xen`, `uml`, `exe` | ABI của guest. `hvm` = fully virtualized (phổ biến nhất); `xen` = paravirtualized |

### 6.4. Thuộc tính dành cho Ironic (Bare Metal)

| Thuộc tính | Mô tả |
|---|---|
| `img_type` | `whole-disk` (có partition table) hoặc `partition` (chỉ root partition). Ironic dùng để triển khai bare metal |
| `rootfs_uuid` | UUID của root partition — Ironic dùng khi cấu hình software RAID |
| `kernel_id` | Image ID dùng làm kernel khi boot AMI-style image |
| `ramdisk_id` | Image ID dùng làm ramdisk khi boot AMI-style image |

### 6.5. Thuộc tính VMware

| Thuộc tính | Giá trị | Mô tả |
|---|---|---|
| `vmware_adaptertype` | `lsiLogic`, `lsiLogicsas`, `busLogic`, `ide`, `paraVirtual` | SCSI/IDE controller trong VMware |
| `vmware_ostype` | string (VMware GuestID) | Mô tả OS theo chuẩn VMware. Mặc định: `otherGuest` |

## 7. Thuộc tính Resource Traits

**Resource Traits** cho phép yêu cầu tính năng phần cứng cụ thể trên compute node khi scheduling. Nova Scheduler sẽ chỉ đặt VM lên host có trait đó.

### 7.1. Cú pháp

```
trait:<TRAIT_NAME>=required
```

> Giá trị hợp lệ duy nhất hiện tại là `required`.

### 7.2. Ví dụ trait phổ biến

| Trait | Mô tả |
|---|---|
| `trait:HW_CPU_X86_AVX2=required` | Host phải hỗ trợ AVX2 (cho ML/AI workloads) |
| `trait:HW_CPU_X86_AVX512F=required` | Host phải hỗ trợ AVX-512 |
| `trait:STORAGE_DISK_SSD=required` | Host phải có SSD |
| `trait:HW_GPU_API_VULKAN=required` | Host có GPU hỗ trợ Vulkan |
| `trait:COMPUTE_NODE=required` | Tổng quát — đặt lên compute node |

### 7.3. Cách dùng

```bash
# Image yêu cầu host hỗ trợ AVX2 và SSD
openstack image set \
    --property "trait:HW_CPU_X86_AVX2=required" \
    --property "trait:STORAGE_DISK_SSD=required" \
    "ML-Training-Image"
```

> **Lưu ý:** Image traits được Nova Scheduler áp dụng ngay cả khi tạo VM từ volume (volume-backed instance), nếu volume source là image có traits.
> Tính năng này được thêm từ bản **Rocky**.

## 8. Thuộc tính tùy chỉnh (Custom Properties)

Admin và người dùng có thể thêm bất kỳ thuộc tính nào dưới dạng key-value tùy ý.

### 8.1. Thêm và xem custom property

```bash
# Thêm property khi tạo image
openstack image create \
    --property environment=production \
    --property team=devops \
    --property backup_policy=daily \
    "My-Custom-Image"

# Thêm property vào image đã tồn tại
openstack image set \
    --property cost_center=CC-001 \
    --property approved_by=admin \
    <image-id>

# Xem tất cả properties
openstack image show <image-id>

# Lọc image theo property
openstack image list --property team=devops
```

### 8.2. Xóa custom property

```bash
# Xóa một property
openstack image unset --property team <image-id>

# Xóa nhiều property cùng lúc
openstack image unset --property team --property cost_center <image-id>
```

### 8.3. Bảo vệ property (Property Protection)

Admin có thể giới hạn ai được phép đọc/ghi/xóa property nào bằng file `property-protections.conf`.

```ini
# /etc/glance/property-protections.conf

# Chỉ admin mới được đặt os_secure_boot
[os_secure_boot]
create = admin
read = @
update = admin
delete = admin

# hw_* — mọi người đọc được, chỉ admin được sửa/xóa
[hw_.*]
create = admin
read = @
update = admin
delete = admin

# Mọi người đọc/ghi custom property
[.*]
create = @
read = @
update = @
delete = @
```

> `@` = tất cả người dùng đã xác thực.

## 9. Metadata Definitions (Metadef)

**Metadef** là hệ thống cho phép admin **định nghĩa trước** schema metadata chuẩn (namespace, object, property) để áp dụng nhất quán lên nhiều loại tài nguyên: image, volume, flavor, aggregate...

### 9.1. Kiến trúc Metadef

```
Metadef Catalog
└── Namespace  (ví dụ: "OS::Compute::Hypervisor")
    │  ├── display_name, description
    │  ├── visibility: public/private
    │  └── resource_type_associations: [image, flavor, volume, aggregate...]
    │
    ├── Object  (ví dụ: "CPUPinning")
    │   ├── Mô tả nhóm properties liên quan
    │   └── Property (ví dụ: "cpu_policy": shared|dedicated)
    │
    ├── Property trực tiếp  (không qua Object)
    └── Tag  (nhãn gắn với namespace)
```

### 9.2. Điểm quan trọng về Metadef

- **Catalog vs Storage:** Metadef chỉ định nghĩa **schema** — không lưu giá trị thực. Khi user gán property cho image, giá trị thực được lưu trong Glance DB (hoặc Nova DB cho flavor).
- **Prefix khác nhau theo resource:** Cùng một khái niệm (CPU topology) nhưng prefix khác nhau:
  - Trên image: `hw_cpu_policy=dedicated`
  - Trên flavor: `hw:cpu_policy=dedicated`

### 9.3. Lệnh CLI quản lý Metadef

```bash
# Liệt kê tất cả namespace
openstack metadef namespace list

# Xem chi tiết namespace
openstack metadef namespace show "OS::Compute::Hypervisor"

# Tạo namespace tùy chỉnh
openstack metadef namespace create \
    --public \
    --display-name "Thông tin nội bộ" \
    --description "Metadata cho quản lý nội bộ" \
    "org.example.internal"

# Load namespace mặc định của OpenStack (chạy sau cài đặt Glance)
glance-manage db_load_metadefs

# Xóa toàn bộ metadefs
glance-manage db_clear_metadefs
```

## 10. Glance Metadata vs Cloud-init

Hai cơ chế **bổ sung** cho nhau, không thay thế nhau.

### 10.1. So sánh tổng quan

| Tiêu chí | Glance Metadata | Cloud-init |
|---|---|---|
| **Lưu ở đâu** | Database của Glance | Script/config bên trong image |
| **Ai đọc** | Nova, Glance API, người dùng | Cloud-init daemon bên trong VM |
| **Khi nào chạy** | **Trước** khi VM được tạo | **Sau** khi VM boot lần đầu |
| **Mục đích chính** | Mô tả image, cấu hình hypervisor | Cấu hình OS bên trong VM |
| **Ví dụ** | `hw_disk_bus=virtio`, `min_ram=2048` | Đặt hostname, SSH key, cài package |
| **Thay đổi được không** | Có (dùng `openstack image set`) | Không (đã baked vào image) |

### 10.2. Luồng phối hợp hoàn chỉnh

```
[User] openstack server create
         │
         ▼
[Nova] Đọc flavor + Glance metadata
         │  ─ hw_disk_bus → chọn controller
         │  ─ hw_vif_model → chọn NIC model
         │  ─ min_ram → kiểm tra flavor đủ RAM chưa?
         │  ─ hw_firmware_type → BIOS hay UEFI?
         ▼
[QEMU/KVM] Khởi tạo VM với phần cứng ảo đã được cấu hình
         │
         ▼
[VM Boot] Cloud-init chạy
         │  ─ Đặt hostname
         │  ─ Inject SSH key
         │  ─ Cài thêm package
         │  ─ Chạy user-data script
         ▼
[VM sẵn sàng]
```

## 11. Flow xử lý Metadata khi tạo máy ảo

![Full VM creation flow](./images/Glance_8.png)

**Giải thích từng bước:**

```
1. User: openstack server create --image <id> --flavor <id>
         │
2. Nova API: nhận request, validate
         │
3. Nova Scheduler:
         │  ─ Đọc image metadata từ Glance
         │  ─ ImagePropertiesFilter: lọc host theo architecture, hypervisor_type, vm_mode
         │  ─ Xét trait:* requirements
         │  ─ Kiểm tra min_disk, min_ram vs flavor
         ▼
4. Nova Compute (trên host được chọn):
         │  ─ Download image từ Glance
         │  ─ Xác minh img_signature nếu có
         │  ─ Áp dụng hw_* properties → tạo libvirt XML
         ▼
5. libvirt/QEMU:
         │  ─ Tạo VM với disk bus, NIC model, firmware theo metadata
         │  ─ Boot VM
         ▼
6. Cloud-init trong VM:
         ─ Cấu hình OS
         ─ VM sẵn sàng phục vụ
```

## 13. Giới hạn & Lưu ý quan trọng

### 13.1. Giới hạn số lượng properties

| Cấu hình | Mặc định | File cấu hình |
|---|---|---|
| `image_property_quota` | **128 properties** mỗi image | `glance-api.conf` |

> Admin có thể tăng giới hạn này nhưng nên cân nhắc vì mỗi property là một row trong DB.

### 13.2. Prefix bị Glance bảo vệ

- **`os_glance`**: Glance giữ riêng cho metadata nội bộ — người dùng và admin không được đặt property nào bắt đầu bằng prefix này. Glance sẽ từ chối request nếu cố đặt.

### 13.3. Non-inheritable properties

Khi tạo snapshot từ VM, một số properties từ image gốc **không được kế thừa** sang snapshot. Danh sách này được cấu hình bởi `non_inheritable_image_properties` trong Nova.

### 13.4. Tương thích driver

| Property group | Driver hỗ trợ |
|---|---|
| `hw_*` (hầu hết) | **libvirt only** |
| `hw_vif_model` | libvirt + VMware API |
| `hw_machine_type` (Hyper-V values) | Hyper-V driver |
| `vmware_*` | VMware API only |
| `img_type` | Ironic only |
| `trait:*` | Nova Scheduler (tất cả driver) |

### 13.5. ImagePropertiesFilter

Nova bật sẵn **ImagePropertiesFilter** trong scheduler. Filter này:
- Đọc `architecture`, `hypervisor_type`, `vm_mode` từ image metadata
- Chỉ cho phép schedule lên các compute node có capabilities khớp
- Ảnh hưởng đến cả volume-backed instances nếu volume source từ image có traits

## Tổng kết nhanh

| Nhóm | Prefix | Ví dụ key | Ai đọc |
|---|---|---|---|
| **Core** | _(không)_ | `id`, `name`, `status`, `min_disk`, `visibility` | Glance, Nova, người dùng |
| **Hardware** | `hw_` | `hw_disk_bus`, `hw_vif_model`, `hw_cpu_policy` | Nova, QEMU/KVM, libvirt |
| **OS** | `os_` | `os_distro`, `os_version`, `os_secure_boot` | Horizon, Nova, cloud-init |
| **Security** | `img_` | `img_signature`, `img_hide_hypervisor_id` | Nova, Barbican, Glance |
| **Architecture** | _(không)_ | `architecture`, `hypervisor_type`, `vm_mode` | Nova Scheduler |
| **Traits** | `trait:` | `trait:HW_CPU_X86_AVX2=required` | Nova Scheduler |
| **VMware** | `vmware_` | `vmware_adaptertype`, `vmware_ostype` | VMware API driver |
| **Custom** | _(tùy ý)_ | `team=devops`, `env=prod` | Người dùng/ứng dụng |
