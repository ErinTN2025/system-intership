# File log Neutron & Troubleshooting

## 1. Vị trí log

Neutron dùng `oslo.log` như Nova/Cinder.

- **Non-container**: `/var/log/neutron/`
- **Containerized** (Kolla, TripleO): `/var/log/containers/neutron/`
- **OVN-specific**: `/var/log/ovn/`
- **OVS**: `/var/log/openvswitch/`

## 2. Danh sách file log

### 2.1 Trên Controller node

| File | Service | Nội dung |
|---|---|---|
| `neutron-server.log` | `neutron-server` | Tất cả API request, ML2 plugin, RPC tới agents. **Quan trọng nhất khi debug.** |

### 2.2 Trên Network node (legacy ML2/OVS)

| File | Service | Nội dung |
|---|---|---|
| `l3-agent.log` | `neutron-l3-agent` | Tạo/xóa router, namespace `qrouter`, NAT/FIP rules |
| `dhcp-agent.log` | `neutron-dhcp-agent` | dnsmasq lifecycle, namespace `qdhcp` |
| `metadata-agent.log` | `neutron-metadata-agent` | Proxy metadata request về Nova |
| `openvswitch-agent.log` | `neutron-openvswitch-agent` | OVS port binding, flow setup |

### 2.3 Trên Compute node (Legacy)

| File | Service | Nội dung |
|---|---|---|
| `openvswitch-agent.log` | `neutron-openvswitch-agent` | Port binding cho VM, security group iptables |
| `metadata-agent.log` | `neutron-metadata-agent` (đôi khi) | Nếu cấu hình metadata trên compute |

### 2.4 Trên Compute/Network node (ML2/OVN)

| File | Service | Nội dung |
|---|---|---|
| `/var/log/ovn/ovn-controller.log` | `ovn-controller` | Tất cả flow translation, chassis binding |
| `/var/log/ovn/ovn-northd.log` | `ovn-northd` | NB → SB translation |
| `/var/log/openvswitch/ovs-vswitchd.log` | OVS datapath | Flow execution, datapath errors |
| `/var/log/neutron/ovn-metadata-agent.log` | OVN metadata proxy | Metadata service trên compute |

### 2.5 Log phụ trợ

| File | Mục đích |
|---|---|
| `/var/log/nova/nova-compute.log` | VIF plug/unplug — debug attach |
| `/var/log/libvirt/libvirtd.log` | libvirt error khi bind tap |
| `/var/log/syslog` (Ubuntu) hoặc `/var/log/messages` (RHEL) | Kernel-level network (iptables, conntrack overflow) |
| `journalctl -u <service>` | Systemd unit log |
| `/var/lib/neutron/dhcp/<net-id>/host` | dnsmasq host file (DHCP binding) |
| `/var/lib/neutron/dhcp/<net-id>/leases` | DHCP leases hiện tại |

## 3. Cấu hình logging

`/etc/neutron/neutron.conf` phần `[DEFAULT]`:

```ini
[DEFAULT]
log_dir = /var/log/neutron
debug = False                              # True khi debug
use_json = False                           # True nếu dùng ELK/Loki
use_journal = False
log_rotation_type = size
max_logfile_size_mb = 100
max_logfile_count = 10
default_log_levels = neutron=INFO,oslo=WARNING,sqlalchemy=WARNING,ovsdbapp=INFO

logging_context_format_string = %(asctime)s.%(msecs)03d %(process)d %(levelname)s %(name)s [%(request_id)s %(user_identity)s] %(message)s
```

**Bật debug tạm thời**:

```bash
# Service cần debug
sudo crudini --set /etc/neutron/neutron.conf DEFAULT debug True
sudo systemctl restart neutron-server

# Trên agent
sudo crudini --set /etc/neutron/l3_agent.ini DEFAULT debug True
sudo systemctl restart neutron-l3-agent

# Sau khi xong, tắt lại
sudo crudini --set /etc/neutron/neutron.conf DEFAULT debug False
sudo systemctl restart neutron-server
```

## 4. Trace 1 request từ đầu đến cuối

### 4.1 Lấy request-id

```bash
# Bật debug ở client
openstack --debug network create demo 2>&1 | grep -i "x-openstack-request-id"

# Hoặc nhìn trong API log gần nhất
sudo tail -100 /var/log/neutron/neutron-server.log | grep "POST /v2"
```

### 4.2 Grep qua toàn bộ stack

```bash
REQ="req-abc12345-..."

# Controller
grep "$REQ" /var/log/neutron/neutron-server.log

# Network node (legacy)
ssh network01 "grep '$REQ' /var/log/neutron/*.log"

# Compute (legacy)
ssh compute01 "grep '$REQ' /var/log/neutron/*.log"

# OVN flow
ssh controller "grep '$REQ' /var/log/ovn/ovn-northd.log"
```

### 4.3 Trace theo Port UUID

Khi debug 1 port cụ thể không cần biết request-id:

```bash
PORT=12345678-1234-1234-1234-123456789012

# Backend Neutron
grep "$PORT" /var/log/neutron/*.log

# Trên compute (OVS)
ssh compute01 "sudo ovs-vsctl find Interface external_ids:iface-id=$PORT"

# Trên compute (OVN)
sudo ovn-sbctl find Port_Binding logical_port=$PORT
```

## 5. Lỗi thường gặp và Fix

### 5.1 VM stuck ở BUILD / không lên ACTIVE

**Nguyên nhân thường gặp**:
- Port không bind được vào host.
- Agent down.
- Bridge mapping sai.

**Debug**:
```bash
# 1. Status VM
openstack server show <vm> -c status -c fault

# 2. Port status
openstack port list --server <vm>
openstack port show <port-id> -c status -c binding_vif_type -c binding_host_id

# binding_vif_type:
#   ovs / ovn       : OK
#   binding_failed  : LỖI - check 4

# 3. Agent
openstack network agent list

# 4. Check log
ssh compute01 "sudo tail -100 /var/log/neutron/openvswitch-agent.log"
# Hoặc OVN:
ssh compute01 "sudo tail -100 /var/log/ovn/ovn-controller.log"

# 5. Check bridge mapping
ssh compute01 sudo ovs-vsctl get open . external-ids:ovn-bridge-mappings   # OVN
# Hoặc
grep bridge_mappings /etc/neutron/plugins/ml2/openvswitch_agent.ini       # OVS
```

**Fix phổ biến**:
```bash
# Restart agent
ssh compute01 sudo systemctl restart neutron-openvswitch-agent
# Hoặc OVN
ssh compute01 sudo systemctl restart ovn-controller

# Force rebind (xóa và tạo lại port)
openstack server remove port <vm> <port>
openstack server add port <vm> <port>
```

### 5.2 VM không lấy được IP DHCP

**Debug**:
```bash
# 1. DHCP agent (legacy)
openstack network agent list --network <net> --agent-type dhcp

# 2. Kiểm tra dnsmasq trên network node
ssh network01 sudo ip netns exec qdhcp-<net-id> ps aux | grep dnsmasq

# 3. Host file binding
cat /var/lib/neutron/dhcp/<net-id>/host
# Phải thấy: <MAC>,<IP>,<hostname>

# 4. Leases
cat /var/lib/neutron/dhcp/<net-id>/leases

# 5. Bắt gói DHCP
ssh compute01 sudo tcpdump -i tap<xxx> -nn 'port 67 or port 68'

# OVN native DHCP — check NB DHCP_Options
sudo ovn-nbctl list DHCP_Options
sudo ovn-nbctl lsp-get-dhcpv4-options <port>
```

**Fix**:
```bash
# Legacy - restart dhcp-agent
ssh network01 sudo systemctl restart neutron-dhcp-agent

# OVN - rebuild DHCP option
sudo ovn-nbctl --wait=sb sync
```

### 5.3 VM ping không được nhau (cùng subnet)

**Debug**:
```bash
# 1. Security Group
openstack port show <port> -c security_group_ids
openstack security group rule list <sg-id>
# Phải có rule allow ICMP intra-group hoặc cùng SG

# 2. Port security
openstack port show <port> -c port_security_enabled

# 3. Trên compute, xem flow
ssh compute01 sudo ovs-ofctl dump-flows br-int | grep <vm-mac>

# 4. OVN ACL
sudo ovn-nbctl acl-list <port-group>

# 5. Bắt gói trên 2 tap
ssh compute01 sudo tcpdump -i tap<vm1> -nn icmp
ssh compute02 sudo tcpdump -i tap<vm2> -nn icmp
```

### 5.4 Floating IP không ping được

```bash
# 1. NAT rule
ssh network01 "sudo ip netns exec qrouter-<router-id> iptables -t nat -L -n -v"
# Phải thấy DNAT + SNAT cho FIP

# OVN
sudo ovn-nbctl lr-nat-list <router>

# 2. FIP có trên router namespace?
ssh network01 sudo ip netns exec qrouter-<router-id> ip addr | grep <fip>

# 3. ARP từ ngoài có resolve được FIP không?
ip neigh | grep <fip>
# Nếu không → router không respond ARP, kiểm tra L3 agent / gateway chassis (OVN)

# 4. Trace
sudo ovn-trace --detailed neutron-<ext-net-uuid> \
  'inport=="provnet-..." \
   && eth.src==<external-router-mac> \
   && eth.dst==<router-mac> \
   && ip4.src==<external-test-ip> \
   && ip4.dst==<fip> \
   && ip.ttl==64 && icmp4.type==8'
```

### 5.5 VM không ra Internet (SNAT fail)

```bash
# 1. Router có gateway?
openstack router show <r> -c external_gateway_info
# Phải có "enable_snat": true

# 2. SNAT rule
ssh network01 "sudo ip netns exec qrouter-<id> iptables -t nat -L POSTROUTING -n -v"

# 3. Router namespace ping được gateway không?
ssh network01 "sudo ip netns exec qrouter-<id> ping -c 2 <gateway>"

# 4. MTU vấn đề (rất phổ biến với Geneve/VXLAN)
ssh network01 "sudo ip netns exec qrouter-<id> ping -c 2 -s 1450 -M do 8.8.8.8"
# Nếu ping size lớn fail → MTU
```

**Fix MTU**:
```bash
# Set MTU phù hợp cho overlay (Geneve = 1450, VXLAN = 1450)
openstack network set --mtu 1450 <network>

# Hoặc trong dhcp_agent.ini
[DEFAULT]
advertise_mtu = True
```

### 5.6 Agent stuck DOWN

```bash
# 1. Check trên controller
openstack network agent list

# 2. Agent có chạy không?
ssh <node> sudo systemctl status neutron-<agent>

# 3. Check log
ssh <node> sudo tail -50 /var/log/neutron/<agent>.log

# 4. Heartbeat issue (RabbitMQ down)
ssh <node> "sudo rabbitmqctl status | head -20"

# 5. Reset agent
openstack network agent delete <agent-id>      # Sẽ tự register lại khi restart
ssh <node> sudo systemctl restart neutron-<agent>
```

### 5.7 OVN: ovn-controller không sync

```bash
# 1. Connection
ssh compute01 sudo ovn-appctl -t ovn-controller connection-status
# Phải thấy: "connected"

# 2. SB DB reachable?
ssh compute01 sudo ovs-vsctl get open . external-ids:ovn-remote

# 3. Force recompute
ssh compute01 sudo ovn-appctl -t ovn-controller recompute

# 4. Inactivity probe (nếu connection bị drop định kỳ)
sudo ovn-nbctl set NB_Global . options:northd_probe_interval=180000
```

### 5.8 Loop network / Storm broadcast

```bash
# Triệu chứng: CPU OVS 100%, traffic broadcast cao
sudo ovs-ofctl dump-ports br-int

# Tìm port có traffic bất thường
sudo tcpdump -i br-int -nn 'broadcast or multicast' | head -100

# Disable port tạm thời
sudo ovs-ofctl mod-port br-int <port> NO_FLOOD
```

### 5.9 Conntrack table full

```bash
# Trên compute
sudo cat /proc/sys/net/netfilter/nf_conntrack_count
sudo cat /proc/sys/net/netfilter/nf_conntrack_max

# Nếu count gần max → tăng
echo 1048576 | sudo tee /proc/sys/net/netfilter/nf_conntrack_max

# Persistent
echo 'net.netfilter.nf_conntrack_max=1048576' | sudo tee -a /etc/sysctl.conf
```

## 6. Bật trace toàn bộ flow (sâu nhất)

Khi cần debug khó:

```bash
# Bật DEBUG ở mọi service
for n in controller network01 compute01 compute02; do
  ssh $n "sudo crudini --set /etc/neutron/neutron.conf DEFAULT debug True"
done

# Restart
ssh controller sudo systemctl restart neutron-server
ssh network01 "sudo systemctl restart neutron-l3-agent neutron-dhcp-agent neutron-metadata-agent neutron-openvswitch-agent"
ssh compute01 "sudo systemctl restart neutron-openvswitch-agent"
# Hoặc OVN:
ssh compute01 sudo systemctl restart ovn-controller
ssh controller sudo systemctl restart ovn-northd

# Reproduce
openstack --debug network create test-debug

# Trace
REQ=$(openstack network show test-debug -f json | jq -r '.request_id // empty')
for n in controller network01 compute01; do
  echo "=== $n ===" && ssh $n "grep '$REQ' /var/log/neutron/*.log 2>/dev/null"
done

# QUAN TRỌNG: TẮT debug sau khi xong (log debug rất nặng)
for n in controller network01 compute01 compute02; do
  ssh $n "sudo crudini --set /etc/neutron/neutron.conf DEFAULT debug False"
  ssh $n "sudo systemctl restart neutron-*"
done
```

## 7. Giám sát Production

### 7.1 Alert quan trọng

| Event | Severity | Action |
|---|---|---|
| `neutron-server` down | Critical | Page oncall |
| Bất kỳ agent DOWN > 5min | Critical | Page oncall |
| `ovn-northd` down | Critical | Failover RAFT, page |
| ERROR/Traceback trong log | Warning | Investigate trong 1h |
| Conntrack > 80% max | Warning | Tăng max hoặc check abuse |
| RabbitMQ disconnect | Critical | Restart agent + rabbitmq |
| FIP allocation > 80% pool | Warning | Mở rộng pool hoặc audit unused FIP |

### 7.2 Metrics nên monitor

```bash
# Network count
openstack network list -f value | wc -l

# FIP usage
openstack floating ip list -f json | jq '[.[] | {status:.Status}] | group_by(.status) | map({status:.[0].status, count:length})'

# Port count theo status
openstack port list --all -f value -c Status | sort | uniq -c

# Agent status
openstack network agent list -f json | jq '[.[] | {alive:.Alive, binary:.Binary}] | group_by(.binary) | map({binary:.[0].binary, alive_count:[.[].alive]|map(select(.==true))|length, total:length})'
```

Tools đề xuất:
- **Prometheus + node_exporter**: metrics hệ thống
- **prometheus-openstack-exporter**: metrics Neutron
- **Grafana**: dashboard
- **Loki / ELK**: aggregate logs
- **OVN Monitor**: https://github.com/ovn-org/ovsdb-monitor (cho OVN)

## 8. Quick reference – Lệnh debug hay dùng

```bash
# Top ERROR mới nhất
sudo grep ERROR /var/log/neutron/neutron-server.log | tail -10

# Đếm error theo loại
sudo grep ERROR /var/log/neutron/*.log | awk -F'ERROR' '{print $2}' | \
  awk '{print $2}' | sort | uniq -c | sort -rn | head

# Tail tất cả log
sudo tail -F /var/log/neutron/*.log /var/log/ovn/*.log

# Filter warning + error
sudo tail -F /var/log/neutron/*.log | grep -E "WARNING|ERROR"

# Trace port
PORT=xxxx
grep $PORT /var/log/neutron/*.log
sudo ovn-sbctl find Port_Binding logical_port=$PORT
sudo ovs-vsctl find Interface external_ids:iface-id=$PORT

# Đếm packet bị drop trên iptables
ssh compute01 sudo iptables -L -n -v | grep DROP

# OVS port stats
sudo ovs-ofctl dump-ports br-int

# Status đầy đủ
openstack network agent list
sudo ovn-nbctl show
sudo ovn-sbctl show
```
