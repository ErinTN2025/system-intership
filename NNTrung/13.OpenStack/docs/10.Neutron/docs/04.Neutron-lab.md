# Lab thực hành Neutron

## Mục tiêu

1. Build full network topology: external network + tenant network + router.
2. Tạo VM, attach vào tenant network.
3. Cấp Floating IP, ping/SSH từ external.
4. Cấu hình Security Group cho phép SSH/HTTP/ICMP.
5. Liên thông 2 tenant network qua 1 router.
6. Cô lập 2 tenant (đảm bảo VM của project A không thấy được project B).
7. Debug bằng `ip netns`, `ovs-vsctl`, `ovn-nbctl`.

## Mô hình lab

```
External (provider) network: 192.168.1.0/24  (gateway 192.168.1.1)
                                |
                          [Router R1]
                                |
Tenant network A: 192.168.10.0/24
   ├── vm-a1 (192.168.10.x)
   └── vm-a2 (192.168.10.y)

Tenant network B: 192.168.20.0/24
   └── vm-b1 (192.168.20.x)
```

> Tất cả lệnh chạy trên **controller** trừ khi có `[compute01]` hoặc `[network01]`.

## Phần 0 – Chuẩn bị

```bash
# ==== Trên CONTROLLER ====
source /root/admin-openrc

# 1. Kiểm tra agent
openstack network agent list

# 2. Kiểm tra image + flavor
openstack image list | grep cirros
openstack flavor list | grep m1.tiny

# 3. Nếu chưa có
wget https://download.cirros-cloud.net/0.6.2/cirros-0.6.2-x86_64-disk.img -O /tmp/cirros.img
openstack image create cirros --disk-format qcow2 --container-format bare \
  --public --file /tmp/cirros.img

openstack flavor create --ram 512 --disk 1 --vcpus 1 m1.tiny

# 4. Tạo keypair để SSH
openstack keypair create --public-key ~/.ssh/id_rsa.pub mykey || \
  (ssh-keygen -t rsa -N "" -f ~/.ssh/id_rsa && \
   openstack keypair create --public-key ~/.ssh/id_rsa.pub mykey)
```

## Phần 1 – Tạo External Network

```bash
# ==== Trên CONTROLLER (admin) ====
# 1. Tạo provider network (flat, share=True)
openstack network create \
  --share --external \
  --provider-physical-network physnet1 \
  --provider-network-type flat \
  public

# 2. Tạo subnet cho external (KHÔNG DHCP)
openstack subnet create public-subnet \
  --network public \
  --no-dhcp \
  --subnet-range 192.168.1.0/24 \
  --gateway 192.168.1.1 \
  --allocation-pool start=192.168.1.100,end=192.168.1.200

# 3. Verify
openstack network show public
openstack subnet show public-subnet
```

> **Lưu ý**: `physnet1` phải khớp với `bridge_mappings`/`ovn-bridge-mappings` trên compute/network node (xem [02.Neutron_conf.md](./02.Neutron_conf.md)).

## Phần 3 – Tạo Router & Kết nối

```bash
# ==== Trên CONTROLLER ====
# 1. Tạo router R1
openstack router create R1

# 2. Set external gateway (cho phép VM ra Internet qua NAT)
openstack router set --external-gateway public R1

# 3. Thêm interface vào subnet A và B
openstack router add subnet R1 subnet-a
openstack router add subnet R1 subnet-b

# 4. Verify
openstack router show R1
openstack port list --router R1

# Trên Network node hoặc OVN — verify namespace/logical-router
# Legacy ML2/OVS:
ssh network01 sudo ip netns | grep qrouter

# OVN:
sudo ovn-nbctl lr-list
sudo ovn-nbctl show
```

## Phần 4 – Security Group

```bash
# ==== Trên CONTROLLER ====
# 1. Tạo security group "web"
openstack security group create web \
  --description "Allow ICMP, SSH, HTTP"

# 2. Thêm rule
openstack security group rule create --proto icmp --remote-ip 0.0.0.0/0 web
openstack security group rule create --proto tcp --dst-port 22 --remote-ip 0.0.0.0/0 web
openstack security group rule create --proto tcp --dst-port 80 --remote-ip 0.0.0.0/0 web

# 3. Verify
openstack security group rule list web

# 4. Group cho phép giao tiếp giữa các VM cùng group (intra-group)
openstack security group create internal
openstack security group rule create --proto tcp --dst-port 1:65535 \
  --remote-group internal internal
openstack security group rule create --proto udp --dst-port 1:65535 \
  --remote-group internal internal
openstack security group rule create --proto icmp \
  --remote-group internal internal
```

## Phần 5 – Tạo VM và Verify Connectivity

```bash
# ==== Trên CONTROLLER ====
# 1. Tạo 2 VM trong net-a
openstack server create vm-a1 \
  --image cirros --flavor m1.tiny \
  --network net-a \
  --security-group web --security-group internal \
  --key-name mykey \
  --wait

openstack server create vm-a2 \
  --image cirros --flavor m1.tiny \
  --network net-a \
  --security-group web --security-group internal \
  --key-name mykey \
  --wait

# 2. Tạo VM trong net-b
openstack server create vm-b1 \
  --image cirros --flavor m1.tiny \
  --network net-b \
  --security-group web --security-group internal \
  --key-name mykey \
  --wait

# 3. Lấy IP của các VM
openstack server list

VM_A1=$(openstack server show vm-a1 -f value -c addresses | awk -F'=' '{print $2}')
VM_A2=$(openstack server show vm-a2 -f value -c addresses | awk -F'=' '{print $2}')
VM_B1=$(openstack server show vm-b1 -f value -c addresses | awk -F'=' '{print $2}')
echo "vm-a1=$VM_A1  vm-a2=$VM_A2  vm-b1=$VM_B1"

# 4. Verify từ trong router namespace (legacy ML2/OVS)
ROUTER_ID=$(openstack router show R1 -f value -c id)
ssh network01 "sudo ip netns exec qrouter-$ROUTER_ID ping -c 3 $VM_A1"
ssh network01 "sudo ip netns exec qrouter-$ROUTER_ID ping -c 3 $VM_B1"

# 5. Console vào vm-a1 và ping vm-a2 / vm-b1
openstack console url show vm-a1
# Mở URL ở browser, login cirros/gocubsgo

# Trong VM:
# ping 192.168.10.x       # vm-a2 (cùng subnet → OK)
# ping 192.168.20.x       # vm-b1 (khác subnet → OK qua router R1)
# ping 8.8.8.8            # External → OK qua SNAT
```

## Phần 6 – Floating IP (truy cập VM từ ngoài)

```bash
# ==== Trên CONTROLLER ====
# 1. Cấp Floating IP
FIP=$(openstack floating ip create public -f value -c floating_ip_address)
echo "FIP=$FIP"

# 2. Associate vào vm-a1
openstack server add floating ip vm-a1 $FIP

# 3. Verify
openstack server show vm-a1 -f value -c addresses
# Sẽ thấy 2 IP: private (192.168.10.x) + floating (192.168.1.x)

# 4. Ping từ controller (giả sử controller trong cùng dải 192.168.1.0/24)
ping -c 3 $FIP

# 5. SSH vào VM bằng floating IP
ssh -i ~/.ssh/id_rsa cirros@$FIP
# Mặc định Cirros: user cirros, password gocubsgo (nếu key fail)

# 6. Disassociate
openstack server remove floating ip vm-a1 $FIP

# 7. Release FIP
openstack floating ip delete $FIP
```

## Phần 7 – Multi-tenant Isolation

Mục tiêu: chứng minh 2 project khác nhau **không thấy** network của nhau.

```bash
# ==== Trên CONTROLLER (admin) ====
# 1. Tạo 2 project
openstack project create proj-a --description "Project A"
openstack project create proj-b --description "Project B"

# 2. Tạo 2 user
openstack user create --password Welcome123 --project proj-a user-a
openstack user create --password Welcome123 --project proj-b user-b

openstack role add --project proj-a --user user-a member
openstack role add --project proj-b --user user-b member

# 3. Tạo openrc cho mỗi user
cat > /tmp/proj-a-openrc <<EOF
export OS_AUTH_URL=http://10.10.34.161:5000/v3
export OS_PROJECT_NAME=proj-a
export OS_USERNAME=user-a
export OS_PASSWORD=Welcome123
export OS_USER_DOMAIN_NAME=Default
export OS_PROJECT_DOMAIN_NAME=Default
export OS_IDENTITY_API_VERSION=3
EOF

cat > /tmp/proj-b-openrc <<EOF
export OS_AUTH_URL=http://10.10.34.161:5000/v3
export OS_PROJECT_NAME=proj-b
export OS_USERNAME=user-b
export OS_PASSWORD=Welcome123
export OS_USER_DOMAIN_NAME=Default
export OS_PROJECT_DOMAIN_NAME=Default
export OS_IDENTITY_API_VERSION=3
EOF

# 4. User A tạo network trùng dải IP với project khác
source /tmp/proj-a-openrc
openstack network create net-x
openstack subnet create subnet-x --network net-x \
  --subnet-range 10.0.0.0/24

openstack server create vm-x --image cirros --flavor m1.tiny \
  --network net-x --wait

# 5. User B cũng tạo network trùng IP 10.0.0.0/24 → vẫn được vì namespace tách biệt
source /tmp/proj-b-openrc
openstack network create net-y
openstack subnet create subnet-y --network net-y \
  --subnet-range 10.0.0.0/24

openstack server create vm-y --image cirros --flavor m1.tiny \
  --network net-y --wait

# 6. Verify user B không thấy network của user A
openstack network list
# Chỉ thấy net-y, KHÔNG thấy net-x

# 7. Quay về admin
source /root/admin-openrc
openstack network list --long
# Admin thấy cả 2

# Trên network node, có 2 namespace khác nhau cho 2 router/dhcp của 2 project
ssh network01 sudo ip netns | grep qdhcp
```

## Phần 8 – Debug bằng namespace + OVS/OVN

### 8.1 Legacy ML2/OVS

```bash
# ==== Trên Network node (legacy) ====
# Xem các namespace
sudo ip netns
# qdhcp-<net-uuid>   (DHCP namespace mỗi network)
# qrouter-<router-uuid>  (Router namespace)

# Vào router namespace
sudo ip netns exec qrouter-<id> ip addr
sudo ip netns exec qrouter-<id> ip route
sudo ip netns exec qrouter-<id> iptables -t nat -L -n -v

# Vào DHCP namespace
sudo ip netns exec qdhcp-<id> ip addr
sudo ip netns exec qdhcp-<id> ps aux | grep dnsmasq

# Trace ping từ namespace router ra VM
sudo ip netns exec qrouter-<id> tcpdump -i any -nn icmp

# OVS bridges
sudo ovs-vsctl show
sudo ovs-vsctl list-ports br-int

# Flow table
sudo ovs-ofctl dump-flows br-int | head -50
```

### 8.2 OVN

```bash
# ==== Trên OVN host (thường là controller) ====
# Logical view
sudo ovn-nbctl show
sudo ovn-nbctl ls-list           # Logical switches (= Neutron networks)
sudo ovn-nbctl lr-list           # Logical routers (= Neutron routers)
sudo ovn-nbctl lsp-list <ls>     # Logical switch ports

# Physical view (binding)
sudo ovn-sbctl show
sudo ovn-sbctl list chassis
sudo ovn-sbctl list Port_Binding | grep <port-uuid>

# Trace 1 microflow (rất mạnh!)
sudo ovn-trace --detailed <datapath> \
  'inport=="<port-name>" && eth.src==<MAC> && ip4.src==<src> && ip4.dst==<dst>'

# Ví dụ ping từ vm-a1 đến vm-b1
sudo ovn-trace --detailed net-a \
  'inport=="<vm-a1-port>" && eth.src==fa:16:3e:11:11:11 \
   && eth.dst==fa:16:3e:99:99:99 \
   && ip4.src==192.168.10.10 && ip4.dst==192.168.20.10 \
   && ip.ttl==64 && icmp4.type==8'

# Trên compute node — OVS local
sudo ovs-vsctl show
sudo ovs-ofctl dump-flows br-int
sudo ovn-appctl -t ovn-controller list-commands
```

### 8.3 Bắt gói tin trên tap interface của VM

```bash
# ==== Trên Compute node có VM chạy ====
# Tìm tap interface
sudo virsh domiflist <instance-name>
# Hoặc
sudo ovs-vsctl show | grep tap

# Bắt gói
sudo tcpdump -i tap<xxxx> -nn

# Trên bridge integration
sudo tcpdump -i br-int -nn 'host 192.168.10.10'
```

## Phần 9 – Test HA Router (chỉ Legacy ML2/OVS)

```bash
# ==== Trên CONTROLLER ====
# 1. Bật l3_ha = True trong neutron.conf
sudo crudini --set /etc/neutron/neutron.conf DEFAULT l3_ha True
sudo systemctl restart neutron-server

# 2. Tạo HA router
openstack router create --ha R-HA
openstack router set --external-gateway public R-HA
openstack router add subnet R-HA subnet-a

# 3. Xem 2 L3 agent host router này
openstack network agent list --router R-HA
# Phải thấy 2 agent: 1 active, 1 standby

# 4. Verify VRRP trong namespace
ssh network01 sudo ip netns exec qrouter-<id> ip addr show
# Active node có IP, standby không

# 5. Failover test - tắt active node
ssh network01 sudo systemctl stop neutron-l3-agent

# Trong vòng vài giây, standby sẽ thành active
ssh network02 sudo ip netns exec qrouter-<id> ip addr show

# 6. Restore
ssh network01 sudo systemctl start neutron-l3-agent
```

## Phần 10 – Cleanup

```bash
# ==== Trên CONTROLLER ====
source /root/admin-openrc

# Xóa VM
openstack server delete vm-a1 vm-a2 vm-b1 vm-x vm-y --wait 2>/dev/null

# Disassociate + delete FIP
for fip in $(openstack floating ip list -f value -c ID); do
  openstack floating ip delete $fip
done

# Xóa router (phải remove interface + clear gateway trước)
openstack router unset --external-gateway R1 2>/dev/null
openstack router remove subnet R1 subnet-a 2>/dev/null
openstack router remove subnet R1 subnet-b 2>/dev/null
openstack router delete R1 R-HA 2>/dev/null

# Xóa subnet + network
openstack subnet delete subnet-a subnet-b 2>/dev/null
openstack network delete net-a net-b 2>/dev/null

# Xóa security group
openstack security group delete web internal 2>/dev/null

# Xóa user/project test
source /tmp/proj-a-openrc 2>/dev/null
openstack network delete net-x 2>/dev/null
source /tmp/proj-b-openrc 2>/dev/null
openstack network delete net-y 2>/dev/null
source /root/admin-openrc
openstack user delete user-a user-b 2>/dev/null
openstack project delete proj-a proj-b 2>/dev/null

# Xóa external (admin)
openstack subnet delete public-subnet 2>/dev/null
openstack network delete public 2>/dev/null
```

## Phần 11 – Troubleshooting checklist

| Triệu chứng | Kiểm tra | Fix |
|---|---|---|
| VM không lấy được IP DHCP | `qdhcp namespace` hoặc OVN native DHCP | Restart `neutron-dhcp-agent` hoặc check `ovn-nbctl show` |
| VM ping được nhau cùng network nhưng không qua router | Router gateway chưa set | `openstack router set --external-gateway` |
| Không ping được FIP từ ngoài | NAT rule trên router / Security Group | `ip netns exec qrouter iptables -t nat -L` + check SG |
| VM không ra Internet | SNAT rule trên router + DNS | Check `openstack router show` có external-gateway, thử ping IP raw |
| Port stuck ở `DOWN` / `BUILD` | Agent down | `openstack network agent list`, restart agent tương ứng |
| 2 VM cùng subnet không ping được nhau | Security Group không cho phép | `openstack security group rule list <sg>` |
| Trunk port không hoạt động | Backend không hỗ trợ hoặc thiếu trunk plugin | `service_plugins` phải có `trunk` |
| OVN: `ovn-nbctl: connection failed` | OVN DB down | `systemctl status ovn-northd ovn-ovsdb-server-nb` |

**Log nên xem khi debug**:
```bash
# Controller
sudo tail -f /var/log/neutron/neutron-server.log

# Network node
sudo tail -f /var/log/neutron/{l3,dhcp,metadata,openvswitch}-agent.log

# Compute node (OVN)
sudo tail -f /var/log/ovn/ovn-controller.log

# Compute node (legacy)
sudo tail -f /var/log/neutron/openvswitch-agent.log
```

Trace 1 request:
```bash
REQ_ID=req-xxxx
ssh controller "grep $REQ_ID /var/log/neutron/*.log"
ssh network01 "grep $REQ_ID /var/log/neutron/*.log"
ssh compute01 "grep $REQ_ID /var/log/neutron/*.log"
```
