# File cấu hình Neutron
## 1. File `/etc/neutron/neutron.conf` (Cốt lõi)
Section `[DEFAULT]` tập trung vào việc điều phối và tích hợp OVN.

```conf
[DEFAULT]
# Plugin cốt lõi và các dịch vụ đi kèm
core_plugin = ml2
service_plugins = ovn-router,trunk,segments,port_forwarding

# Tối ưu hóa API cho hệ thống lớn
api_workers = auto
rpc_state_report_workers = 1

# Xác thực và Message Queue
auth_strategy = keystone
transport_url = rabbit://openstack:password@controller:5672/

# Thông báo cho Nova (Cực kỳ quan trọng để VM lên mạng nhanh)
notify_nova_on_port_status_changes = True
notify_nova_on_port_data_changes = True
```

- **Cập nhật:** `ovn-router` thay thế cho L3 Agent. `port_forwarding` hiện đã ổn định và nên được bật mặc định.

## 2. File `/etc/neutron/plugins/ml2/ml2_conf.ini` (Cấu trúc mạng)
Đây là nơi bạn định nghĩa "luật chơi" cho các gói tin.

```conf
[ml2]
# Geneve là 'vua' thay thế cho VXLAN trong OVN
type_drivers = flat,vlan,geneve
tenant_network_types = geneve
mechanism_drivers = ovn
extension_drivers = port_security,qos

[ml2_type_geneve]
# Geneve cung cấp định danh mạng ảo lên tới 24-bit
vni_ranges = 1:1000
max_header_size = 38

[securitygroup]
# Sử dụng OVN native để thực thi luật firewall ngay trên switch ảo
enable_security_group = True
firewall_driver = ovn
```

- **Tại sao dùng Geneve?** So với VXLAN, Geneve linh hoạt hơn, cho phép truyền tải các metadata của OVN giữa các node, giúp xử lý các tính năng nâng cao mà VXLAN cũ không làm được.

## 3. File `/etc/neutron/plugins/ml2/ml2_conf.ini` (Kết nối OVN DB)
Thay vì cấu hình Agent, bạn cấu hình cách Neutron nói chuyện với cơ sở dữ liệu OVN.

```conf
[ovn]
ovn_nb_connection = tcp:10.10.31.166:6641
ovn_sb_connection = tcp:10.10.31.166:6642
ovn_l3_mode = True
ovn_metadata_enabled = True
```
* **6641 (Northbound):** Nơi Neutron đẩy cấu hình mạng ảo xuống.
* **6642 (Southbound):** Nơi các Compute Node lấy dữ liệu về để thực thi trên switch ảo.

## 4. File `/etc/neutron/ovn_metadata_agent.ini` (Thay thế Metadata Agent cũ)
Trong năm 2026, chúng ta không dùng `metadata_agent.ini` đơn thuần nữa mà dùng bản dành riêng cho OVN.

```conf
[DEFAULT]
nova_metadata_host = 10.10.31.166
metadata_proxy_shared_secret = Welcome123

[ovn]
ovn_sb_connection = tcp:10.10.31.166:6642
```
* **Điểm mới:** Agent này "soi" trực tiếp vào Southbound DB của OVN để biết VM nào đang yêu cầu Metadata, thay vì phải dựa vào các qdhcp namespace như trước.

## 5. Những thành phần đã Deprecated
Để học và làm hiệu quả vào năm 2026, bạn nên biết những thứ sau đang dần biến mất:
1.  **Linux Bridge Agent:** Rất ít dự án mới dùng vì hiệu suất kém và khó quản lý Overlay network lớn.
2.  **dhcp_agent.ini:** OVN đã tự có "Native DHCP" chạy bên trong switch ảo. Bạn không cần chạy thêm tiến trình `dnsmasq` nữa.
3.  **l3_agent.ini:** Tất cả việc định tuyến đã được "phần mềm hóa" hoàn toàn bởi OVN.
