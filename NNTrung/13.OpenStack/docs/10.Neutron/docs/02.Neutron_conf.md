# File cấu hình Neutron

Tài liệu này trình bày cấu hình Neutron cho **2 kiến trúc**:
- **Phần A**: ML2/OVN (modern, khuyến nghị từ 2024+)
- **Phần B**: ML2/OVS + agents (legacy, vẫn còn nhiều môi trường dùng)

Mỗi phần tách rõ theo **node**: Controller / Network / Compute.

---

# Phần A — Kiến trúc ML2/OVN (Modern)

OVN gộp L3/DHCP/Metadata vào trong switch ảo, không cần các agent riêng. Chỉ còn 2 thành phần chính trên data plane:
- `ovn-controller` (mỗi compute/network node)
- `ovn-metadata-agent` (cho metadata service)

## A.1 Controller node

### A.1.1 `/etc/neutron/neutron.conf`

```ini
[DEFAULT]
core_plugin = ml2
service_plugins = ovn-router,trunk,segments,port_forwarding,qos
auth_strategy = keystone
transport_url = rabbit://openstack:Welcome123@10.10.34.161:5672/
state_path = /var/lib/neutron

api_workers = 4
rpc_state_report_workers = 1

# Quan trọng để VM lên mạng nhanh - báo Nova khi port UP/DOWN
notify_nova_on_port_status_changes = True
notify_nova_on_port_data_changes = True

[database]
connection = mysql+pymysql://neutron:Welcome123@10.10.34.161/neutron

[keystone_authtoken]
www_authenticate_uri = http://10.10.34.161:5000
auth_url = http://10.10.34.161:5000
memcached_servers = 10.10.34.161:11211
auth_type = password
project_domain_name = Default
user_domain_name = Default
project_name = service
username = neutron
password = Welcome123

[nova]
auth_url = http://10.10.34.161:5000
auth_type = password
project_domain_name = Default
user_domain_name = Default
region_name = RegionOne
project_name = service
username = nova
password = Welcome123

[oslo_concurrency]
lock_path = /var/lib/neutron/tmp
```

### A.1.2 `/etc/neutron/plugins/ml2/ml2_conf.ini`

```ini
[ml2]
# Geneve khuyến nghị cho OVN, có thể giữ flat/vlan cho provider network
type_drivers = flat,vlan,geneve
tenant_network_types = geneve
mechanism_drivers = ovn
extension_drivers = port_security,qos

[ml2_type_flat]
flat_networks = physnet1

[ml2_type_vlan]
network_vlan_ranges = physnet1:100:200

[ml2_type_geneve]
vni_ranges = 1:65536
max_header_size = 38

[securitygroup]
enable_security_group = True
firewall_driver = ovn

[ovn]
ovn_nb_connection = tcp:10.10.34.161:6641
ovn_sb_connection = tcp:10.10.34.161:6642
ovn_l3_mode = True
ovn_metadata_enabled = True
ovn_emit_need_to_frag = True
```

> **Port**: 6641 = Northbound (Neutron ghi xuống), 6642 = Southbound (Compute đọc lên).

## A.2 Compute node (chạy `ovn-controller`)

### A.2.1 Cấu hình OVS bridge

```bash
# Tạo bridge tích hợp
sudo ovs-vsctl add-br br-int

# Tạo bridge provider (kết nối ra NIC vật lý)
sudo ovs-vsctl add-br br-provider
sudo ovs-vsctl add-port br-provider eth1   # NIC kết nối provider net

# Bridge mapping
sudo ovs-vsctl set open . external-ids:ovn-bridge-mappings=physnet1:br-provider

# Encap type + IP cho overlay (Geneve)
sudo ovs-vsctl set open . external-ids:ovn-encap-type=geneve
sudo ovs-vsctl set open . external-ids:ovn-encap-ip=10.10.35.162   # IP tunnel

# Trỏ về SB DB
sudo ovs-vsctl set open . external-ids:ovn-remote=tcp:10.10.34.161:6642

# Bật chassis là gateway nếu node này là network node
sudo ovs-vsctl set open . external-ids:ovn-cms-options=enable-chassis-as-gw
```

### A.2.2 `/etc/neutron/neutron.conf` (compute)

```ini
[DEFAULT]
core_plugin = ml2
auth_strategy = keystone
transport_url = rabbit://openstack:Welcome123@10.10.34.161:5672/

[keystone_authtoken]
# ... (giống controller)

[oslo_concurrency]
lock_path = /var/lib/neutron/tmp
```

### A.2.3 `/etc/neutron/ovn_metadata_agent.ini` (Metadata service trên compute)

```ini
[DEFAULT]
nova_metadata_host = 10.10.34.161
metadata_proxy_shared_secret = Welcome123
auth_url = http://10.10.34.161:5000

[ovn]
ovn_sb_connection = tcp:10.10.34.161:6642
```

### A.2.4 Service cần chạy trên compute

```bash
sudo systemctl enable --now openvswitch-switch
sudo systemctl enable --now ovn-controller
sudo systemctl enable --now neutron-ovn-metadata-agent
```

## A.3 Verify OVN

```bash
# Trên controller
openstack network agent list
# Phải thấy: OVN Controller agent (1 per chassis) + OVN Metadata agent

# Trên OVN DB host
sudo ovn-nbctl show           # Logical Network
sudo ovn-sbctl show           # Chassis + binding
sudo ovn-sbctl list chassis
```

# Phần B — Kiến trúc ML2/OVS Legacy

Mô hình truyền thống: Controller (server) → Network node (L3/DHCP/Metadata agent) → Compute node (OVS agent).

## B.1 Controller node

### B.1.1 `/etc/neutron/neutron.conf`

```ini
[DEFAULT]
core_plugin = ml2
service_plugins = router,trunk,qos
allow_overlapping_ips = True
auth_strategy = keystone
transport_url = rabbit://openstack:Welcome123@10.10.34.161:5672/
api_workers = 4

notify_nova_on_port_status_changes = True
notify_nova_on_port_data_changes = True

[database]
connection = mysql+pymysql://neutron:Welcome123@10.10.34.161/neutron

[keystone_authtoken]
# ... (như A.1.1)

[nova]
# ... (như A.1.1)

[oslo_concurrency]
lock_path = /var/lib/neutron/tmp
```

### B.1.2 `/etc/neutron/plugins/ml2/ml2_conf.ini`

```ini
[ml2]
type_drivers = flat,vlan,vxlan
tenant_network_types = vxlan
mechanism_drivers = openvswitch,l2population
extension_drivers = port_security,qos

[ml2_type_flat]
flat_networks = physnet1

[ml2_type_vlan]
network_vlan_ranges = physnet1:100:200

[ml2_type_vxlan]
vni_ranges = 1:1000

[securitygroup]
enable_security_group = True
firewall_driver = openvswitch
enable_ipset = True
```

## B.2 Network node (chạy L3 / DHCP / Metadata agent)

### B.2.1 Cài đặt

```bash
sudo apt install -y neutron-l3-agent neutron-dhcp-agent neutron-metadata-agent \
                    neutron-openvswitch-agent openvswitch-switch
```

### B.2.2 `/etc/neutron/l3_agent.ini`

```ini
[DEFAULT]
interface_driver = openvswitch
agent_mode = legacy                      # hoặc dvr / dvr_snat
external_network_bridge =                # Để trống ở phiên bản mới
```

### B.2.3 `/etc/neutron/dhcp_agent.ini`

```ini
[DEFAULT]
interface_driver = openvswitch
dhcp_driver = neutron.agent.linux.dhcp.Dnsmasq
enable_isolated_metadata = True
force_metadata = True
```

### B.2.4 `/etc/neutron/metadata_agent.ini`

```ini
[DEFAULT]
nova_metadata_host = 10.10.34.161
metadata_proxy_shared_secret = Welcome123
```

### B.2.5 `/etc/neutron/plugins/ml2/openvswitch_agent.ini`

```ini
[ovs]
local_ip = 10.10.35.171                   # IP overlay (VXLAN tunnel) của network node
bridge_mappings = physnet1:br-provider

[agent]
tunnel_types = vxlan
l2_population = True

[securitygroup]
firewall_driver = openvswitch
enable_security_group = True
```

### B.2.6 Tạo OVS bridge trên network node

```bash
sudo ovs-vsctl add-br br-int
sudo ovs-vsctl add-br br-provider
sudo ovs-vsctl add-port br-provider eth1     # NIC ra provider

# br-ex (nếu tách external network bridge riêng)
sudo ovs-vsctl add-br br-ex
sudo ovs-vsctl add-port br-ex eth2
```

### B.2.7 Service

```bash
sudo systemctl restart \
  openvswitch-switch \
  neutron-openvswitch-agent \
  neutron-l3-agent \
  neutron-dhcp-agent \
  neutron-metadata-agent
```


## B.4 DVR (Distributed Virtual Router)

Để bật DVR — router phân tán xuống compute node, giảm tải network node:

### B.4.1 Controller `ml2_conf.ini`

```ini
[ml2]
mechanism_drivers = openvswitch,l2population

[agent]
enable_distributed_routing = True
l2_population = True
```

### B.4.2 Network node `l3_agent.ini`

```ini
[DEFAULT]
agent_mode = dvr_snat
```

### B.4.3 Compute node — cài thêm L3 agent

```bash
sudo apt install -y neutron-l3-agent
```

`/etc/neutron/l3_agent.ini`:
```ini
[DEFAULT]
agent_mode = dvr
interface_driver = openvswitch
```

### B.4.4 Bật DVR cho router cụ thể

```bash
openstack router set --distributed <router>
```

## B.5 HA Router (VRRP)

Để bật HA router — failover giữa nhiều L3 agent:

### B.5.1 Controller `neutron.conf`

```ini
[DEFAULT]
l3_ha = True
max_l3_agents_per_router = 3
min_l3_agents_per_router = 2
l3_ha_net_cidr = 169.254.192.0/18
```

### B.5.2 Tạo HA router

```bash
openstack router create --ha my-ha-router
```

# Phần C — Verify & Common Commands

## C.1 Kiểm tra agent

```bash
openstack network agent list

# Kết quả OVN:
# - OVN Controller agent (mỗi compute/network node)
# - OVN Metadata agent

# Kết quả Legacy OVS:
# - Open vSwitch agent (mỗi node)
# - L3 agent
# - DHCP agent
# - Metadata agent
```

## C.2 Tạo external network test

```bash
# Provider network (flat) cho external
openstack network create --share --external \
  --provider-physical-network physnet1 --provider-network-type flat \
  ext-net

openstack subnet create --network ext-net --no-dhcp \
  --allocation-pool start=192.168.1.100,end=192.168.1.200 \
  --gateway 192.168.1.1 --subnet-range 192.168.1.0/24 \
  ext-subnet
```

## C.3 Restart đầy đủ service

```bash
# Controller
sudo systemctl restart neutron-server

# Network node (legacy)
sudo systemctl restart neutron-l3-agent neutron-dhcp-agent neutron-metadata-agent neutron-openvswitch-agent

# Compute (OVN)
sudo systemctl restart ovn-controller neutron-ovn-metadata-agent

# Compute (legacy OVS)
sudo systemctl restart neutron-openvswitch-agent
```

# Phần D — Những thành phần đã Deprecated

| Thành phần | Trạng thái | Thay bằng |
|---|---|---|
| Linux Bridge Agent | Deprecated | OVS hoặc OVN |
| `dhcp_agent.ini` + dnsmasq | Vẫn dùng trong ML2/OVS | OVN Native DHCP |
| `l3_agent.ini` | Vẫn dùng trong ML2/OVS | OVN logical router |
| `metadata_agent.ini` (cũ) | Vẫn dùng trong ML2/OVS | `ovn_metadata_agent.ini` |
| nova-network | Bỏ hoàn toàn | Neutron |

> Tóm lại: **Deployment mới → chọn OVN**. Deployment cũ vẫn được hỗ trợ cho ML2/OVS, nhưng nên có lộ trình migrate.
