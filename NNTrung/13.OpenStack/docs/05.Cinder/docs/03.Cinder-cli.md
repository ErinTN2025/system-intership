# Các lệnh CLI thường dùng trong Cinder

## 1. Chuẩn bị môi trường

```bash
# Cài CLI nếu chưa có
pip install python-openstackclient python-cinderclient

# Source admin credentials
source /root/admin-openrc      # hoặc user-openrc.sh

# Kiểm tra
openstack --version
openstack volume service list
```

> **Lưu ý**: Từ OpenStack Pike trở đi, ưu tiên dùng `openstack volume` thay vì `cinder` (legacy). Cả hai vẫn dùng được.

## 2. Quản lý Volume

### 2.1 Các thao tác cơ bản

```bash
# Liệt kê volume
openstack volume list [--all-projects] [--long] [--status available|in-use|error]
openstack volume show <tên-hoặc-uuid>

# Tạo volume rỗng 10GB
openstack volume create --size 10 my-volume

# Tạo volume từ image (boot-from-volume sau này)
openstack volume create --size 20 --image cirros vol-from-image

# Tạo volume từ snapshot
openstack volume create --size 10 --snapshot <snapshot-id> vol-from-snap

# Tạo volume bằng cách clone volume khác
openstack volume create --size 10 --source <source-volume-id> vol-clone

# Tạo volume với type cụ thể
openstack volume create --size 10 --type ssd --description "DB disk" db-vol

# Tạo volume trong availability zone
openstack volume create --size 10 --availability-zone nova vol-az

# Xóa volume (phải ở status available, không attached)
openstack volume delete <volume>

# Xóa volume kèm snapshot/backup phụ thuộc
openstack volume delete --force <volume>
```

### 2.2 Thao tác nâng cao

```bash
# Đổi tên / mô tả
openstack volume set --name new-name --description "New desc" <volume>

# Resize (extend - chỉ tăng được, không giảm)
openstack volume set --size 30 <volume>

# Đặt read-only
openstack volume set --read-only <volume>
openstack volume set --read-write <volume>

# Đặt bootable (dùng cho boot-from-volume)
openstack volume set --bootable <volume>
openstack volume set --non-bootable <volume>

# Đặt image metadata
openstack volume set --image-property hw_scsi_model=virtio-scsi <volume>

# Reset trạng thái (khi bị stuck ở error/creating)
openstack volume set --state available <volume>      # admin only
openstack volume set --state error <volume>
```

### 2.3 Attach / Detach Volume

```bash
# Attach volume vào VM (thực hiện qua Nova)
openstack server add volume <vm-name> <volume-name>

# Attach với device path cụ thể
openstack server add volume <vm> <volume> --device /dev/vdc

# Detach
openstack server remove volume <vm> <volume>

# Xem tất cả attachment của 1 volume
openstack volume show <volume> -c attachments

# Multi-attach (Volume Type phải hỗ trợ)
openstack volume create --size 10 --type multiattach-type shared-vol
openstack server add volume vm1 shared-vol
openstack server add volume vm2 shared-vol
```

## 3. Snapshot

```bash
# Liệt kê snapshot
openstack volume snapshot list [--volume <volume>]

# Tạo snapshot
openstack volume snapshot create --volume <volume> snap-name

# Tạo snapshot khi volume in-use (force)
openstack volume snapshot create --volume <volume> --force snap-inuse

# Xem chi tiết
openstack volume snapshot show <snap-name>

# Xóa
openstack volume snapshot delete <snap-name>

# Tạo volume mới từ snapshot
openstack volume create --size 10 --snapshot <snap-name> new-vol

# Đổi tên snapshot
openstack volume snapshot set --name new-name <snap>
```

## 4. Backup

```bash
# Tạo backup (volume phải available, hoặc dùng --force nếu in-use)
openstack volume backup create --name <backup-name> <volume>

# Backup volume in-use (cần --force)
openstack volume backup create --name <backup-name> --force <volume>

# Backup incremental (chỉ lưu phần thay đổi từ backup đầy đủ trước đó)
openstack volume backup create --name <backup-name> --incremental <volume>

# Liệt kê
openstack volume backup list [--all-projects]
openstack volume backup show <backup>

# Restore vào volume mới
openstack volume backup restore <backup> <new-volume-name>

# Restore vào volume hiện có (volume phải available và đủ dung lượng)
openstack volume backup restore <backup> <existing-volume-id>

# Xóa backup
openstack volume backup delete <backup>

# Reset state (admin only - khi backup bị stuck)
openstack volume backup set --state error <backup>
```

## 5. Volume Type & QoS

```bash
# Liệt kê volume types
openstack volume type list [--long]
openstack volume type show <type>

# Tạo volume type
openstack volume type create ssd
openstack volume type create --public ssd
openstack volume type create --private vip-only

# Gán extra-specs (backend assignment)
openstack volume type set ssd --property volume_backend_name=CEPH_SSD
openstack volume type set ssd --property volume_backend_name=LVM_iSCSI

# Gán multiattach
openstack volume type set multiattach-type --property multiattach="<is> True"

# Cho phép project private dùng type
openstack volume type set --project <project-id> vip-only

# Xóa
openstack volume type delete <type>
openstack volume type unset ssd --property volume_backend_name

# QoS specs (giới hạn IOPS, bandwidth)
openstack volume qos create high-iops --consumer front-end \
  --property total_iops_sec=2000 \
  --property total_bytes_sec=104857600

# Gắn QoS vào volume type
openstack volume qos associate high-iops ssd

# Bỏ liên kết
openstack volume qos disassociate high-iops --volume-type ssd

# Xem QoS
openstack volume qos list
openstack volume qos show high-iops
```

## 6. Quota & Limits

```bash
# Xem quota của project hiện tại
openstack quota show <project>

# Xem limits effective
openstack limits show --absolute

# Đổi quota (admin only)
openstack quota set --volumes 50 --snapshots 100 --gigabytes 1000 <project>

# Reset về default
openstack quota set --reset --volumes <project>

# Xem usage thực tế
openstack quota show --usage <project>
```

## 7. Availability Zone & Pool

```bash
# Liệt kê AZ cho volume
openstack availability zone list --volume

# Xem các pool và capacity còn lại (admin)
openstack volume backend pool list --long
```

## 8. Service & Health

```bash
# Liệt kê các service Cinder
openstack volume service list

# Disable / enable một service
openstack volume service set --disable <host> cinder-volume \
  --disable-reason "maintenance"
openstack volume service set --enable <host> cinder-volume

# Liệt kê volume host
openstack volume service list --service cinder-volume

# Cluster (Active-Active HA)
openstack volume cluster list
openstack volume cluster show <name> <binary>
```

## 9. Migration

```bash
# Migrate volume sang backend khác (admin only)
openstack volume migrate <volume> --host <target-host>

# Migrate qua nhiều bước (force)
openstack volume migrate <volume> --host <target-host> --force-host-copy

# Retype (đổi volume type, có thể trigger migration)
openstack volume set --type <new-type> --retype-policy on-demand <volume>
```

## 10. Encryption

```bash
# Tạo encrypted volume type
openstack volume type create encrypted-type
openstack volume type set encrypted-type \
  --encryption-provider luks \
  --encryption-cipher aes-xts-plain64 \
  --encryption-key-size 256 \
  --encryption-control-location front-end

# Xem encryption settings
openstack volume type show encrypted-type

# Tạo volume với type đã encrypt
openstack volume create --size 10 --type encrypted-type secret-vol
```

## 11. Transfer (chuyển volume giữa các project)

```bash
# Owner cũ tạo request
openstack volume transfer request create <volume>
# Lưu lại: id + auth_key

# Owner mới (trong project khác) accept
openstack volume transfer request accept --auth-key <key> <transfer-id>

# Xem danh sách
openstack volume transfer request list

# Hủy request
openstack volume transfer request delete <transfer-id>
```

## 12. Debug & Trace

```bash
# Bật debug toàn bộ lệnh
openstack --debug volume create --size 1 debug-vol

# Trace 1 request-id trong log
grep "req-xxxxxxxx" /var/log/cinder/*.log

# Xem fault message khi volume ở status error
openstack volume show <volume> -f value -c fault

# Xem trạng thái chi tiết qua cinder client (legacy)
cinder show <volume>
cinder list --filters 'status=error'
```

## 13. Lệnh `cinder-manage` (Admin trên Controller)

```bash
# Xem version DB
cinder-manage db version

# Sync schema
cinder-manage db sync

# Online data migration sau upgrade
cinder-manage db online_data_migrations

# Dọn dẹp bản ghi đã xóa
cinder-manage db purge <age-in-days>

# Quản lý service
cinder-manage service list
cinder-manage service remove cinder-volume <hostname>

# Quản lý cluster (HA)
cinder-manage cluster list
cinder-manage cluster remove <binary> <cluster-name>

# Reset state cho volume bị stuck
cinder-manage volume update_host --currenthost <old> --newhost <new>
```

## 14. Quick reference – Lệnh hay dùng nhất

```bash
# Tạo + attach + xem 1 dòng
openstack volume create --size 5 vol1 && \
openstack server add volume my-vm vol1 && \
openstack volume show vol1

# Snapshot + tạo volume mới + verify
openstack volume snapshot create --volume vol1 snap1 && \
openstack volume create --size 5 --snapshot snap1 vol1-restored

# Cleanup
openstack server remove volume my-vm vol1
openstack volume snapshot delete snap1
openstack volume delete vol1 vol1-restored

# Check tổng quan
openstack volume service list
openstack volume backend pool list --long
openstack quota show --usage admin
```
