# Cinder Backends – LVM, Ceph RBD, NFS

## 1. Tổng quan

Cinder không tự lưu data — nó là **abstraction layer** trên các hệ thống lưu trữ thật phía dưới gọi là **Backend**. Mỗi backend được điều khiển qua một **Driver** Python.

Một deployment Cinder có thể có **một** hoặc **nhiều** backend chạy song song (multi-backend). Volume Type sẽ quyết định volume được tạo ở backend nào (qua `volume_backend_name`).

### So sánh nhanh các backend phổ biến

| Backend | Use case | Ưu | Nhược |
|---|---|---|---|
| **LVM + iSCSI** | Lab, single-node, test | Đơn giản, không phụ thuộc | Không HA, không scale, single-point-of-failure |
| **Ceph RBD** | Production phổ biến nhất | HA, multi-attach, snapshot/clone instant, unified storage (Glance+Nova+Cinder) | Phức tạp, cần ≥3 OSD nodes |
| **NFS** | Đơn giản, share file | Dễ cài, multi-attach ngon | Latency cao, không thật là block storage |
| **iSCSI SAN** (NetApp, EMC, Pure) | Enterprise | Performance tốt, vendor support | Đắt, vendor lock-in |
| **GlusterFS** | Distributed file | Scale-out đơn giản | Performance kém Ceph |

## 2. Backend 1: LVM + iSCSI (Lab/Dev)

### 2.1 Khi nào dùng

- Lab học OpenStack lần đầu.
- Single-node deployment.
- KHÔNG dùng production (không HA, mất 1 storage node = mất volume).

### 2.2 Cài đặt trên Storage Node

```bash
# ==== Trên storage01 ====
# 1. Cài LVM2 + iSCSI target
sudo apt install -y lvm2 thin-provisioning-tools targetcli-fb open-iscsi
# CentOS/RHEL:
# sudo dnf install -y lvm2 targetcli iscsi-initiator-utils

# 2. Chuẩn bị disk (giả sử /dev/sdb là disk trống dành cho Cinder)
sudo pvcreate /dev/sdb
sudo vgcreate cinder-volumes /dev/sdb

# 3. Verify
sudo vgs cinder-volumes
sudo pvs

# 4. Filter LVM để không scan ổ của VM (quan trọng!)
sudo sed -i 's|filter = \[ "a/.\*/" \]|filter = [ "a|/dev/sdb|", "r|.\*|" ]|' /etc/lvm/lvm.conf
# Đảm bảo: chỉ ổ /dev/sdb được LVM scan

# 5. Cài cinder-volume
sudo apt install -y cinder-volume tgt
# CentOS: sudo dnf install openstack-cinder-volume targetcli

# 6. Cấu hình /etc/cinder/cinder.conf
sudo tee /etc/cinder/cinder.conf <<'EOF'
[DEFAULT]
transport_url = rabbit://openstack:Welcome123@10.10.34.161
auth_strategy = keystone
my_ip = 10.10.34.171
enabled_backends = lvm
glance_api_servers = http://10.10.34.161:9292

[database]
connection = mysql+pymysql://cinder:Welcome123@10.10.34.161/cinder

[keystone_authtoken]
www_authenticate_uri = http://10.10.34.161:5000
auth_url = http://10.10.34.161:5000
memcached_servers = 10.10.34.161:11211
auth_type = password
project_domain_name = Default
user_domain_name = Default
project_name = service
username = cinder
password = Welcome123

[lvm]
volume_driver = cinder.volume.drivers.lvm.LVMVolumeDriver
volume_group = cinder-volumes
target_protocol = iscsi
target_helper = lioadm
volume_backend_name = LVM_iSCSI

[oslo_concurrency]
lock_path = /var/lib/cinder/tmp
EOF

# 7. Restart
sudo systemctl restart openstack-cinder-volume tgt
# Ubuntu: sudo systemctl restart cinder-volume tgt
```

### 2.3 Verify

```bash
# ==== Trên CONTROLLER ====
openstack volume service list
# storage01@lvm cinder-volume   nova  enabled  up

openstack volume backend pool list --long
# Phải thấy storage01@lvm#LVM_iSCSI

# Tạo Volume Type gắn vào backend
openstack volume type create lvm
openstack volume type set lvm --property volume_backend_name=LVM_iSCSI

# Test
openstack volume create --size 1 --type lvm test-lvm
openstack volume show test-lvm
```

### 2.4 Lưu ý production

- **Không scale**: 1 storage node → mất là mất tất cả.
- **Không HA**: nếu cần HA cho LVM, phải dùng DRBD hoặc Pacemaker (phức tạp).
- **iSCSI bottleneck**: traffic tập trung qua mạng → cần 10G/25G mạng.

## 3. Backend 2: Ceph RBD (Khuyến nghị Production)

### 3.1 Tại sao Ceph?

- **HA built-in**: replicate data sang ≥3 OSD nodes, mất 1 node vẫn chạy.
- **Multi-attach native**: hỗ trợ RBD shared.
- **Snapshot/Clone instant**: copy-on-write trên Ceph layer.
- **Unified storage**: 1 cụm Ceph chạy được Glance (image) + Nova (ephemeral) + Cinder (volume) + Manila (file).
- **Scale-out**: thêm OSD node để tăng dung lượng.

### 3.2 Giả định cluster Ceph đã có

Lab này giả định bạn đã có Ceph cluster với:
- MON: `ceph-mon01`, `ceph-mon02`, `ceph-mon03`
- 3+ OSD nodes
- Pool `volumes`, `images`, `vms` đã tạo

### 3.3 Tạo Pool + User Cinder trên Ceph

```bash
# ==== Trên ceph-mon01 ====
# 1. Tạo pool
ceph osd pool create volumes 128 128
ceph osd pool create images 64 64
ceph osd pool create vms 128 128

# Bật RBD application
ceph osd pool application enable volumes rbd
ceph osd pool application enable images rbd
ceph osd pool application enable vms rbd

# 2. Tạo user "cinder" với quyền
ceph auth get-or-create client.cinder mon 'profile rbd' \
  osd 'profile rbd pool=volumes, profile rbd pool=vms, profile rbd-read-only pool=images' \
  > /etc/ceph/ceph.client.cinder.keyring

# 3. Tạo user "glance" (cho image-volume cache)
ceph auth get-or-create client.glance mon 'profile rbd' \
  osd 'profile rbd pool=images' \
  > /etc/ceph/ceph.client.glance.keyring

# 4. Copy keyring + ceph.conf sang storage node
scp /etc/ceph/ceph.conf storage01:/etc/ceph/
scp /etc/ceph/ceph.client.cinder.keyring storage01:/etc/ceph/

# 5. Tương tự copy sang compute nodes
scp /etc/ceph/ceph.conf compute01:/etc/ceph/
scp /etc/ceph/ceph.client.cinder.keyring compute01:/etc/ceph/
```

### 3.4 Cấu hình libvirt secret trên compute nodes

```bash
# ==== Trên mỗi compute node ====
# 1. Cài ceph-common
sudo apt install -y ceph-common

# 2. Tạo UUID cho libvirt secret
UUID=$(uuidgen)
echo $UUID > /tmp/cinder-secret-uuid
# Ghi nhớ UUID này — phải GIỐNG NHAU trên TẤT CẢ compute và đưa vào cinder.conf

# 3. Tạo secret XML
cat > /tmp/secret.xml <<EOF
<secret ephemeral='no' private='no'>
  <uuid>$UUID</uuid>
  <usage type='ceph'>
    <name>client.cinder secret</name>
  </usage>
</secret>
EOF

# 4. Define secret trong libvirt
sudo virsh secret-define --file /tmp/secret.xml

# 5. Set value cho secret (key của user cinder)
KEY=$(ceph auth get-key client.cinder)
sudo virsh secret-set-value --secret $UUID --base64 $KEY

# 6. Verify
sudo virsh secret-list
```

### 3.5 Cấu hình Cinder volume node

```bash
# ==== Trên storage01 (hoặc controller nếu chạy chung) ====
# 1. Cài ceph-common
sudo apt install -y ceph-common

# 2. Set quyền keyring
sudo chgrp cinder /etc/ceph/ceph.client.cinder.keyring
sudo chmod 0640 /etc/ceph/ceph.client.cinder.keyring

# 3. Sửa /etc/cinder/cinder.conf — chỉ thêm backend ceph + đổi enabled_backends
sudo tee -a /etc/cinder/cinder.conf <<EOF

[ceph]
volume_driver = cinder.volume.drivers.rbd.RBDDriver
volume_backend_name = CEPH_RBD
rbd_pool = volumes
rbd_user = cinder
rbd_secret_uuid = <UUID-từ-bước-3.4>
rbd_ceph_conf = /etc/ceph/ceph.conf
rbd_flatten_volume_from_snapshot = false
rbd_max_clone_depth = 5
rbd_store_chunk_size = 4
rados_connect_timeout = -1
report_discard_supported = True
EOF

# 4. Đổi enabled_backends
sudo crudini --set /etc/cinder/cinder.conf DEFAULT enabled_backends ceph
# Hoặc multi-backend:
# sudo crudini --set /etc/cinder/cinder.conf DEFAULT enabled_backends lvm,ceph

# 5. Restart
sudo systemctl restart openstack-cinder-volume
```

### 3.6 Verify

```bash
# ==== Trên CONTROLLER ====
openstack volume service list
# storage01@ceph cinder-volume nova enabled up

openstack volume backend pool list --long
# Phải thấy storage01@ceph#CEPH_RBD

# Tạo type
openstack volume type create ceph
openstack volume type set ceph --property volume_backend_name=CEPH_RBD

# Test tạo volume
openstack volume create --size 1 --type ceph test-ceph

# ==== Trên ceph-mon01 - verify RBD image đã được tạo ====
rbd ls volumes
# Phải thấy: volume-<uuid>
```

### 3.7 Best practices Ceph + Cinder

- **`rbd_flatten_volume_from_snapshot = false`**: giữ snapshot dạng COW (nhanh). Set `true` chỉ khi muốn xóa snapshot gốc.
- **`rbd_max_clone_depth = 5`**: tránh chain clone quá dài làm chậm IO.
- **Replication count**: ít nhất 3 (`ceph osd pool set volumes size 3`).
- **Erasure coding**: cho cold data (backup) — tiết kiệm 30-50% dung lượng.
- **Pool placement**: tách pool SSD (cho volumes nóng) và HDD (cho backup), dùng `crush rule`.

## 4. Backend 3: NFS

### 4.1 Khi nào dùng

- Đã có sẵn NAS/NFS server.
- Cần multi-attach đơn giản.
- Lab nhanh không muốn cài Ceph.

### 4.2 Hạn chế

- Performance kém hơn iSCSI/RBD (qua filesystem layer).
- Single-point-of-failure ở NFS server (trừ khi dùng NFS HA).
- Không thật là "block storage" — Cinder tạo file `.raw` trên NFS rồi expose như block.

### 4.3 Cài NFS Server

```bash
# ==== Trên NFS server (10.10.34.180) ====
sudo apt install -y nfs-kernel-server
sudo mkdir -p /srv/nfs/cinder
sudo chown nobody:nogroup /srv/nfs/cinder
sudo chmod 777 /srv/nfs/cinder

# Export
echo "/srv/nfs/cinder  10.10.34.0/24(rw,sync,no_subtree_check,no_root_squash)" | \
  sudo tee -a /etc/exports
sudo exportfs -ra
sudo systemctl restart nfs-kernel-server
```

### 4.4 Cấu hình Cinder

```bash
# ==== Trên storage01 ====
sudo apt install -y nfs-common

# Tạo file shares
echo "10.10.34.180:/srv/nfs/cinder" | sudo tee /etc/cinder/nfs_shares
sudo chown root:cinder /etc/cinder/nfs_shares
sudo chmod 0640 /etc/cinder/nfs_shares

# Thêm backend vào cinder.conf
sudo tee -a /etc/cinder/cinder.conf <<'EOF'

[nfs]
volume_driver = cinder.volume.drivers.nfs.NfsDriver
nfs_shares_config = /etc/cinder/nfs_shares
nfs_mount_point_base = /var/lib/cinder/mnt
volume_backend_name = NFS
nas_secure_file_operations = false
nas_secure_file_permissions = false
EOF

sudo crudini --set /etc/cinder/cinder.conf DEFAULT enabled_backends nfs
sudo systemctl restart openstack-cinder-volume
```

### 4.5 Verify

```bash
# ==== Trên CONTROLLER ====
openstack volume type create nfs
openstack volume type set nfs --property volume_backend_name=NFS

openstack volume create --size 1 --type nfs test-nfs
openstack volume show test-nfs

# Trên NFS server - kiểm tra file đã được tạo
ls -lh /srv/nfs/cinder/
# Phải thấy file volume-<uuid> dạng sparse file
```

## 5. Multi-backend (chạy nhiều backend song song)

### 5.1 Cấu hình

```ini
# cinder.conf
[DEFAULT]
enabled_backends = lvm,ceph,nfs
default_volume_type = ceph              # Type mặc định khi user không chỉ định

[lvm]
volume_driver = cinder.volume.drivers.lvm.LVMVolumeDriver
volume_group = cinder-volumes
volume_backend_name = LVM_iSCSI
target_protocol = iscsi
target_helper = lioadm

[ceph]
volume_driver = cinder.volume.drivers.rbd.RBDDriver
volume_backend_name = CEPH_RBD
rbd_pool = volumes
rbd_user = cinder
rbd_secret_uuid = <UUID>
rbd_ceph_conf = /etc/ceph/ceph.conf

[nfs]
volume_driver = cinder.volume.drivers.nfs.NfsDriver
nfs_shares_config = /etc/cinder/nfs_shares
volume_backend_name = NFS
```

### 5.2 Verify

```bash
openstack volume service list
# Sẽ thấy 3 entries: storage01@lvm, storage01@ceph, storage01@nfs

openstack volume backend pool list --long
# 3 pools tương ứng

openstack volume type list
openstack volume type create lvm-tier  --property volume_backend_name=LVM_iSCSI
openstack volume type create ceph-tier --property volume_backend_name=CEPH_RBD
openstack volume type create nfs-tier  --property volume_backend_name=NFS
```

User chọn type → scheduler tự định tuyến đúng backend.

## 6. Cinder Scheduler – Filter & Weigher

Khi có nhiều backend, scheduler chọn backend nào tạo volume?

### 6.1 Filter (loại backend không đủ điều kiện)

| Filter | Chức năng |
|---|---|
| `AvailabilityZoneFilter` | Loại backend khác AZ user yêu cầu |
| `CapacityFilter` | Loại backend không đủ free space |
| `CapabilitiesFilter` | Match extra_specs ↔ backend capabilities (`volume_backend_name`, `multiattach`…) |
| `AffinityFilter` | Volume cần ở cùng/khác host volume khác |
| `DriverFilter` | Custom filter dùng Python code |

### 6.2 Weigher (chấm điểm)

| Weigher | Ý nghĩa | Default |
|---|---|---|
| `CapacityWeigher` | Ưu tiên backend còn nhiều dung lượng (spread) | `1.0` |
| `AllocatedCapacityWeigher` | Ưu tiên backend ít allocated (stack) | `-1.0` |
| `ChanceWeigher` | Random | — |
| `VolumeNumberWeigher` | Ưu tiên backend có ít volume nhất | `1.0` |
| `GoodnessWeigher` | Custom rules | — |

### 6.3 Cấu hình

```ini
# cinder.conf
[DEFAULT]
scheduler_default_filters = AvailabilityZoneFilter,CapacityFilter,CapabilitiesFilter
scheduler_default_weighers = CapacityWeigher

# Multiplier
capacity_weight_multiplier = 1.0           # Spread (dương) vs Stack (âm)
allocated_capacity_weight_multiplier = -1.0
```

## 7. Maintenance & Troubleshooting

### 7.1 Backend hết dung lượng

```bash
# Xem capacity còn lại
openstack volume backend pool list --long

# Disable backend tạm thời (không cho tạo volume mới)
openstack volume service set --disable --disable-reason "full" \
  storage01@ceph cinder-volume

# Sau khi giải quyết
openstack volume service set --enable storage01@ceph cinder-volume
```

### 7.2 Migrate volume sang backend khác

```bash
# Migrate (admin only)
openstack volume migrate <volume> --host storage02@ceph#CEPH_RBD

# Hoặc retype (đổi type → trigger migration nếu cần)
openstack volume set --type ceph --retype-policy on-demand <volume>
```

### 7.3 Backend bị down

```bash
# Xem status
openstack volume service list

# Volume bị stuck do backend down — chuyển volume sang backend khác
cinder-manage volume update_host \
  --currenthost storage01@lvm \
  --newhost storage02@lvm

# Reset state nếu volume stuck
openstack volume set --state available <volume>
```

### 7.4 Lỗi thường gặp

| Lỗi | Nguyên nhân | Cách fix |
|---|---|---|
| `No valid backend found` | Không backend nào pass filter | Check capacity, capabilities, AZ |
| LVM: `Insufficient free space` | VG hết dung lượng | `vgextend cinder-volumes /dev/sdc` |
| Ceph: `error connecting to the cluster` | Keyring sai / network | `ceph -s` với keyring cinder |
| NFS: `Operation not permitted` | NFS export thiếu `no_root_squash` | Sửa `/etc/exports` |
| Multi-attach fail | Backend không support hoặc type chưa set | Check matrix + type spec |
