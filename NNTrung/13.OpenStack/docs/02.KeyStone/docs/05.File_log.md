## File log của Keystone

> **Tổng quan nhanh:** Keystone ghi log tại `/var/log/keystone/keystone.log`. Cấu hình logging qua `logging.conf` (Python logging format). Kích hoạt bằng `log_config_append` trong `keystone.conf`.

---

### 1. File cấu hình logging

Cấu hình vị trí log qua file:
```
/etc/keystone/logging.conf
```
- Sử dụng định dạng **Python logging module** (không phải định dạng thông thường của `keystone.conf`).
- Có file mẫu tham khảo: `etc/logging.conf.sample` trong source code Keystone.

**Cách kích hoạt:** Thêm dòng sau vào phần `[DEFAULT]` của `/etc/keystone/keystone.conf`:
```ini
log_config_append = /etc/keystone/logging.conf
```
> **Lưu ý quan trọng:** Khi dùng `log_config_append`, toàn bộ cấu hình logging sẽ lấy hoàn toàn từ `logging.conf`. Các tùy chọn khác như `log_dir`, `log_file`, `debug` trong `keystone.conf` sẽ **bị bỏ qua**.

---

### 2. File log mặc định

```
/var/log/keystone/keystone.log
```
- Ghi hầu hết các log: request, lỗi, thông tin hoạt động.
- Thư mục `/var/log/keystone/` thường thuộc sở hữu của user `keystone`.

**Các tùy chọn hữu ích:**

| Tùy chọn | Mô tả | Lưu ý |
|---|---|---|
| `insecure_debug = true` | Log hiển thị thông tin nhạy cảm (lý do xác thực thất bại...) | Chỉ dùng khi debug, **không dùng production** |
| `--debug` | Chạy Keystone ở chế độ debug | Dùng khi không thấy request nào trong log |
| `use_syslog = true` | Gửi log qua syslog | Đặt trong phần `[DEFAULT]` |

---

### 3. Ba kiểu Formatter (định dạng log)

Formatter quyết định **mỗi dòng log hiển thị những thông tin gì**.

#### Bảng so sánh nhanh

| Formatter | Thông tin hiển thị | Khi nào dùng |
|---|---|---|
| `minimal` | Chỉ nội dung message | Log rất gọn, ít dùng |
| `normal` | Tên logger + Thời gian + Level + Message | Chạy production (INFO/WARNING) |
| `debug` | normal + Tên file + Tên hàm | Troubleshoot sâu (DEBUG) |

---

#### 3.1 formatter_minimal
```ini
format=%(message)s
```
- Chỉ ghi nội dung thông báo thuần túy.
- Không có thời gian, không có level, không có tên module.

---

#### 3.2 formatter_normal
```ini
format=(%(name)s): %(asctime)s %(levelname)s %(message)s
```
Giải thích từng trường:

| Trường | Ý nghĩa | Ví dụ |
|---|---|---|
| `%(name)s` | Tên logger | `keystone`, `keystone.api` |
| `%(asctime)s` | Thời gian xảy ra | `2026-04-15 15:30:45,123` |
| `%(levelname)s` | Mức độ log | `DEBUG`, `INFO`, `WARNING`, `ERROR` |
| `%(message)s` | Nội dung thông báo | `Token validated successfully` |

---

#### 3.3 formatter_debug
```ini
format=(%(name)s): %(asctime)s %(levelname)s %(module)s %(funcName)s %(message)s
```
Thêm 2 trường so với `normal`:

| Trường | Ý nghĩa | Ví dụ |
|---|---|---|
| `%(module)s` | Tên file Python | `controllers.py`, `core.py` |
| `%(funcName)s` | Tên hàm đang chạy | `authenticate`, `validate_token` |

> Khi bật `level=DEBUG`, nên chuyển sang `formatter_debug` để biết chính xác lỗi ở đâu trong code.

---

### 4. Cấu trúc tổng quát của `/etc/keystone/logging.conf`

```
[formatters]   → Khai báo tên các formatter
[handlers]     → Khai báo nơi ghi log (file, console, syslog...)
[loggers]      → Cấu hình mức log cho từng logger cụ thể
[logger_root]  → Logger gốc – mức log mặc định cho toàn bộ Keystone
```

**Handler phổ biến:**
- `handler_file` → Ghi vào `/var/log/keystone/keystone.log` (dùng `WatchedFileHandler` hoặc `RotatingFileHandler`)
- `handler_console` → Ghi ra màn hình

**Ví dụ cấu hình điển hình:**
```ini
[formatters]
keys=minimal, normal, debug

[formatter_minimal]
format=%(message)s

[formatter_normal]
format=(%(name)s): %(asctime)s %(levelname)s %(message)s

[formatter_debug]
format=(%(name)s): %(asctime)s %(levelname)s %(module)s %(funcName)s %(message)s

[handlers]
keys=console, file

[handler_file]
class=handlers.WatchedFileHandler
level=INFO
formatter=normal
args=('/var/log/keystone/keystone.log', 'a')

[logger_root]
level=INFO
handlers=file
```

### 5. Lưu ý
Khi cài bằng devstack, hầu hết các log đều chạy dưới journalctl, nếu muốn xem log bạn phải xem qua journalctl
