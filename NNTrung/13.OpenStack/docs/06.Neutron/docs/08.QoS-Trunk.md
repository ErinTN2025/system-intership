# QoS Policy & Trunk Port

## Phần 1 — QoS Policy

### 1.1 Khái niệm

**QoS (Quality of Service)** trong Neutron cho phép admin/user kiểm soát hiệu năng mạng của VM thông qua các rule:

| Loại rule | Tác dụng |
|---|---|
| **Bandwidth Limit** | Giới hạn băng thông tối đa (egress hoặc ingress) |
| **Minimum Bandwidth** | Đảm bảo băng thông tối thiểu (best-effort hoặc strict) |
| **Minimum Packet Rate** | Đảm bảo packet/s tối thiểu (cho NFV) |
| **DSCP Marking** | Đánh DSCP code lên gói tin để mạng vật lý ưu tiên |
| **Maximum Packet Rate** | (Mới từ Yoga) Giới hạn pps |

QoS Policy là **container** chứa nhiều rule, được gán vào:
- **Network** → áp cho tất cả port trong network
- **Port** → áp riêng cho 1 port (override policy của network)
- **Floating IP** → giới hạn FIP

### 1.2 Backend hỗ trợ

| Backend | Bandwidth Limit | Min BW | DSCP | Min Packet Rate |
|---|---|---|---|---|
| OVS (linuxbridge driver) | ✅ | ❌ | ✅ | ❌ |
| OVS (native driver) | ✅ | ❌ | ✅ | ❌ |
| OVN | ✅ | ✅ (egress) | ✅ | ❌ |
| SR-IOV | ✅ (vendor-specific) | ✅ | ❌ | ✅ |

> **Lưu ý**: Min Bandwidth dùng **Placement service** để guarantee — scheduler chỉ đặt VM trên host còn đủ BW.

### 1.3 Bật QoS

```ini
# /etc/neutron/neutron.conf
[DEFAULT]
service_plugins = ovn-router,trunk,qos

# /etc/neutron/plugins/ml2/ml2_conf.ini
[ml2]
extension_drivers = port_security,qos
```

Trên OVS agent (legacy):
```ini
# /etc/neutron/plugins/ml2/openvswitch_agent.ini
[agent]
extensions = qos
```

Restart service:
```bash
sudo systemctl restart neutron-server
sudo systemctl restart neutron-openvswitch-agent  # Hoặc ovn-controller
```

### 1.4 CLI thao tác

```bash
# Tạo policy
openstack network qos policy create web-policy --description "Bandwidth limit for web"

# Liệt kê
openstack network qos policy list
openstack network qos policy show web-policy

# Xem các loại rule có sẵn
openstack network qos rule type list

# === Bandwidth Limit ===
# 10 Mbps egress, burst 1 Mbit
openstack network qos rule create --type bandwidth-limit \
  --max-kbps 10000 --max-burst-kbits 1000 --egress \
  web-policy

# Ingress (chỉ một số backend support)
openstack network qos rule create --type bandwidth-limit \
  --max-kbps 5000 --max-burst-kbits 500 --ingress \
  web-policy

# === DSCP Marking ===
# DSCP 26 = AF31 (cho voice/video priority)
openstack network qos rule create --type dscp-marking \
  --dscp-mark 26 \
  web-policy

# === Minimum Bandwidth (cần Placement) ===
openstack network qos rule create --type minimum-bandwidth \
  --min-kbps 5000 --egress \
  web-policy

# === Liệt kê rule trong policy ===
openstack network qos rule list web-policy

# === Sửa rule ===
openstack network qos rule set --max-kbps 20000 web-policy <rule-id>

# === Xóa rule ===
openstack network qos rule delete web-policy <rule-id>
```

### 1.5 Apply Policy

```bash
# Vào network (áp cho tất cả VM trong network)
openstack network set --qos-policy web-policy private

# Vào port (override policy của network)
openstack port set --qos-policy web-policy <port-id>

# Vào FIP
openstack floating ip set --qos-policy web-policy <fip-id>

# Bỏ policy
openstack network unset --qos-policy private
openstack port unset --qos-policy <port-id>
```

### 1.6 Lab QoS

```bash
# ==== Trên CONTROLLER ====
source /root/admin-openrc

# 1. Tạo policy giới hạn 10 Mbps
openstack network qos policy create lab-bw
openstack network qos rule create --type bandwidth-limit \
  --max-kbps 10000 --max-burst-kbits 1000 --egress \
  lab-bw

# 2. Tạo VM (giả định có net-a, subnet-a)
openstack server create vm-fast --image cirros --flavor m1.tiny \
  --network net-a --key-name mykey --wait

openstack server create vm-slow --image cirros --flavor m1.tiny \
  --network net-a --key-name mykey --wait

# 3. Apply QoS vào vm-slow port
SLOW_PORT=$(openstack port list --server vm-slow -f value -c ID)
openstack port set --qos-policy lab-bw $SLOW_PORT

# 4. Test bằng iperf3 (cài trong VM nếu chưa có)
VM_FAST_IP=$(openstack server show vm-fast -f value -c addresses | awk -F'=' '{print $2}')
VM_SLOW_IP=$(openstack server show vm-slow -f value -c addresses | awk -F'=' '{print $2}')

# Cấp FIP để SSH
FIP_F=$(openstack floating ip create public -f value -c floating_ip_address)
FIP_S=$(openstack floating ip create public -f value -c floating_ip_address)
openstack server add floating ip vm-fast $FIP_F
openstack server add floating ip vm-slow $FIP_S

# Trên vm-fast: chạy iperf3 server
ssh -i ~/.ssh/id_rsa cirros@$FIP_F "iperf3 -s -D" 2>/dev/null || \
  echo "Cirros không có iperf3 - dùng image Ubuntu/Debian thật cho lab QoS"

# Hoặc dùng nc + dd cho Cirros
# Server (vm-fast):
ssh -i ~/.ssh/id_rsa cirros@$FIP_F "nc -l -p 5001 > /dev/null &"

# Client (vm-slow): gửi 100MB qua
ssh -i ~/.ssh/id_rsa cirros@$FIP_S \
  "dd if=/dev/zero bs=1M count=100 2>&1 | nc $VM_FAST_IP 5001"

# Đo bandwidth — phải bị giới hạn ~10 Mbps (1.25 MB/s)

# 5. So sánh với vm khác không có QoS
# Test tương tự nhưng từ vm-fast (không có QoS) → vm-slow
# Tốc độ sẽ cao hơn nhiều

# 6. Verify cài đặt thực tế
# Legacy ML2/OVS:
ssh compute01 "sudo ovs-vsctl --columns=name,ingress_policing_rate,ingress_policing_burst list Interface"

# OVN:
sudo ovn-nbctl list QoS

# 7. Cleanup
openstack server remove floating ip vm-fast $FIP_F
openstack server remove floating ip vm-slow $FIP_S
openstack floating ip delete $FIP_F $FIP_S
openstack server delete vm-fast vm-slow
openstack network qos policy delete lab-bw
```

### 1.7 QoS Best Practices

- **Đặt QoS ở network level** cho default tier, override ở port level cho exception.
- **Min Bandwidth chỉ dùng cho NFV/critical workload** — vì nó claim BW ở Placement, có thể fail VM scheduling.
- **DSCP marking** cần switch vật lý hỗ trợ QoS để có tác dụng end-to-end.
- **Ingress bandwidth** không phải lúc nào cũng hoạt động tốt — test trước.
- **Theo dõi**: `ovn-nbctl list QoS` hoặc `ovs-vsctl list Interface` để verify rule có thật.

## Phần 2 — Trunk Port

### 2.1 Khái niệm

**Trunk Port** cho phép **1 vNIC (parent port)** mang nhiều VLAN cùng lúc, mỗi VLAN tương ứng với 1 network khác nhau (gọi là **subport**).

```
              ┌───────────────────────┐
              │       VM eth0         │
              │  (parent port)        │
              └──┬──────────┬─────────┘
                 │          │
        VLAN 100 │          │ VLAN 200
                 ▼          ▼
            net-vlan100  net-vlan200
```

VM thấy:
- `eth0` → parent port (network A)
- `eth0.100` (VLAN sub-interface) → subport vào network B
- `eth0.200` → subport vào network C

### 2.2 Khi nào cần Trunk

| Use case | Lý do |
|---|---|
| **Kubernetes on OpenStack** | K8s worker cần nhiều network (cluster, service, storage) qua 1 vNIC |
| **VNF / Network appliances** | Firewall/Router ảo cần trunk để xử lý nhiều VLAN khác nhau |
| **Multi-tier app trong 1 VM** | App vừa nói chuyện front-end vừa với back-end network |

### 2.3 Bật Trunk

```ini
# /etc/neutron/neutron.conf
[DEFAULT]
service_plugins = ovn-router,trunk,qos,...
```

Restart `neutron-server`. Cả ML2/OVS và ML2/OVN đều hỗ trợ.

### 2.4 Tạo Trunk

```bash
# Giả sử có 3 network: net-mgmt (parent), net-vlan100, net-vlan200

# 1. Tạo parent port
openstack port create --network net-mgmt parent-port

# 2. Tạo subports
openstack port create --network net-vlan100 sub-port-100
openstack port create --network net-vlan200 sub-port-200

# 3. Tạo trunk gắn parent
openstack network trunk create my-trunk --parent-port parent-port

# 4. Thêm subport
openstack network trunk set my-trunk \
  --subport port=sub-port-100,segmentation-type=vlan,segmentation-id=100

openstack network trunk set my-trunk \
  --subport port=sub-port-200,segmentation-type=vlan,segmentation-id=200

# 5. Liệt kê
openstack network trunk list
openstack network trunk show my-trunk

# 6. Tạo VM dùng parent port
openstack server create k8s-node \
  --image ubuntu --flavor m1.small \
  --port parent-port \
  --key-name mykey --wait

# Inside VM, cấu hình VLAN sub-interface:
# sudo ip link add link eth0 name eth0.100 type vlan id 100
# sudo dhclient eth0.100
# sudo ip link add link eth0 name eth0.200 type vlan id 200
# sudo dhclient eth0.200
```

### 2.5 Lab Trunk End-to-End

```bash
# ==== Trên CONTROLLER ====
source /root/admin-openrc

# Yêu cầu: image có VLAN support (Ubuntu/CentOS, Cirros không có vlan tools)
# Dùng Ubuntu Cloud Image
wget https://cloud-images.ubuntu.com/jammy/current/jammy-server-cloudimg-amd64.img -O /tmp/ubuntu.img
openstack image create ubuntu-22 --disk-format qcow2 --container-format bare \
  --public --file /tmp/ubuntu.img

# Flavor lớn hơn cho Ubuntu
openstack flavor create --ram 1024 --disk 5 --vcpus 1 m1.small || true

# 1. Tạo 3 network
openstack network create net-mgmt
openstack subnet create mgmt-subnet --network net-mgmt \
  --subnet-range 192.168.30.0/24 --dns-nameserver 8.8.8.8

openstack network create net-app-a
openstack subnet create app-a-subnet --network net-app-a \
  --subnet-range 192.168.31.0/24 --dns-nameserver 8.8.8.8

openstack network create net-app-b
openstack subnet create app-b-subnet --network net-app-b \
  --subnet-range 192.168.32.0/24 --dns-nameserver 8.8.8.8

# 2. Tạo ports
openstack port create --network net-mgmt  parent-port
openstack port create --network net-app-a sub-a
openstack port create --network net-app-b sub-b

# 3. Tạo trunk
openstack network trunk create trunk1 --parent-port parent-port
openstack network trunk set trunk1 \
  --subport port=sub-a,segmentation-type=vlan,segmentation-id=100
openstack network trunk set trunk1 \
  --subport port=sub-b,segmentation-type=vlan,segmentation-id=200

# 4. Verify
openstack network trunk show trunk1

# 5. Tạo VM dùng parent port
openstack server create trunk-vm \
  --image ubuntu-22 --flavor m1.small \
  --port parent-port \
  --key-name mykey \
  --user-data <(cat <<'EOF'
#cloud-config
runcmd:
  - apt-get update
  - apt-get install -y vlan
  - modprobe 8021q
  - ip link add link ens3 name ens3.100 type vlan id 100
  - ip link add link ens3 name ens3.200 type vlan id 200
  - ip link set ens3.100 up
  - ip link set ens3.200 up
  - dhclient ens3.100
  - dhclient ens3.200
EOF
) \
  --wait

# 6. Gắn FIP, SSH vào và verify
FIP=$(openstack floating ip create public -f value -c floating_ip_address)
openstack server add floating ip trunk-vm $FIP

# Đợi cloud-init xong (1-2 phút)
sleep 90

ssh -i ~/.ssh/id_rsa ubuntu@$FIP "ip addr | grep -E 'ens3|inet '"
# Phải thấy:
# ens3      → IP từ net-mgmt (192.168.30.x)
# ens3.100  → IP từ net-app-a (192.168.31.x)
# ens3.200  → IP từ net-app-b (192.168.32.x)

# 7. Test ping từ trong VM
ssh -i ~/.ssh/id_rsa ubuntu@$FIP \
  "ping -c 2 -I ens3.100 192.168.31.1; ping -c 2 -I ens3.200 192.168.32.1"

# 8. Cleanup
openstack server remove floating ip trunk-vm $FIP
openstack floating ip delete $FIP
openstack server delete trunk-vm

# Phải xóa subport khỏi trunk trước khi xóa trunk
openstack network trunk unset trunk1 --subport sub-a
openstack network trunk unset trunk1 --subport sub-b
openstack network trunk delete trunk1

openstack port delete parent-port sub-a sub-b
openstack subnet delete mgmt-subnet app-a-subnet app-b-subnet
openstack network delete net-mgmt net-app-a net-app-b
```

### 2.6 Verify Trunk thực tế

```bash
# Trên compute, xem tap interface của parent port
ssh compute01 sudo virsh dumpxml <instance> | grep tap

# OVS bridge có patch ports cho mỗi subport
ssh compute01 sudo ovs-vsctl show | grep -A3 tap

# OVN: list trunk
sudo ovn-nbctl list Logical_Switch_Port | grep -i trunk
sudo ovn-nbctl find Logical_Switch_Port type=virtual
```

### 2.7 Trunk Best Practices

- **VLAN ID**: subport ID không cần trùng với VLAN ID thật của underlying network — đây là VLAN tag **trong** parent vNIC.
- **MTU**: subport có MTU bằng `parent - 4 bytes` (cho 802.1Q header). Khai báo MTU đúng cho network của subport.
- **Số subport**: lý thuyết 4094 trên 1 trunk nhưng nên giới hạn ≤16 thực tế.
- **Image OS**: phải hỗ trợ VLAN (Ubuntu/CentOS đều OK, một số image cũ thiếu module `8021q`).
- **K8s deployment**: nhiều CNI plugin (kuryr, neutron-cni) hỗ trợ trunk natively.

### 2.8 Trunk vs Multiple vNIC

| Tiêu chí | Trunk Port | Multiple vNIC |
|---|---|---|
| Số interface trong VM | 1 (+ sub) | Nhiều |
| Bandwidth | Share parent | Riêng từng vNIC |
| Phù hợp | K8s, VNF, app multi-net | App đơn giản, 2-3 network |
| Setup phức tạp | Cao (cần cấu hình VLAN trong VM) | Thấp |
| Hot-add network | Dễ (thêm subport) | Khó (cần reboot/hot-plug) |

## Phần 3 — Kết hợp QoS + Trunk

Có thể apply QoS riêng cho từng subport — ví dụ K8s cluster có 3 mạng (mgmt, pod, storage) và muốn limit storage chỉ 100 Mbps:

```bash
# QoS policy cho storage subport
openstack network qos policy create storage-bw
openstack network qos rule create --type bandwidth-limit \
  --max-kbps 100000 --egress storage-bw

# Apply lên subport
openstack port set --qos-policy storage-bw sub-storage

# Parent port có QoS riêng cho mgmt
openstack network qos policy create mgmt-bw
openstack network qos rule create --type bandwidth-limit \
  --max-kbps 10000 --egress mgmt-bw
openstack port set --qos-policy mgmt-bw parent-port
```

## 4. Troubleshooting

### 4.1 QoS không có tác dụng

```bash
# 1. Policy đã apply?
openstack port show <port> -c qos_policy_id

# 2. Backend support?
openstack network qos rule type list

# 3. Extension đã bật?
grep extensions /etc/neutron/plugins/ml2/openvswitch_agent.ini
grep extension_drivers /etc/neutron/plugins/ml2/ml2_conf.ini

# 4. Verify trên OVS
ssh compute01 sudo ovs-vsctl --columns=name,ingress_policing_rate list Interface

# 5. Verify trên OVN
sudo ovn-nbctl list QoS
sudo ovn-nbctl find QoS external_ids:neutron:qos_policy_id=<policy-id>
```

### 4.2 Trunk: subport không nhận IP

```bash
# 1. Verify subport bind
openstack network trunk show <trunk> | grep sub_ports

# 2. Check VLAN module trong VM
ssh ubuntu@<vm> "lsmod | grep 8021q"

# 3. Bridge tag thực sự
ssh compute01 sudo ovs-ofctl dump-flows br-int | grep vlan

# 4. DHCP request trên sub-interface
ssh ubuntu@<vm> "sudo tcpdump -i ens3.100 -nn port 67 or port 68"

# 5. Bắt ở compute
ssh compute01 sudo tcpdump -i tap<xxxx> -e -nn 'vlan 100'
# `-e` để hiện VLAN tag
```
