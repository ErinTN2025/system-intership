# Tìm hiểu và cài đặt WebVirtCloud quản lý hạ tầng KVM trên Ubuntu 24.04
## 1. Khái niệm
- Là web-based KVM management tool, mã nguồn mở, viết bằng Django(Python).(WebVirtCloud=Django app)
- Nó là một lớp quản lý KVM qua libvirt nhưng hiển thị dưới dạng giao diện web.
- Có thể coi như một giải pháp thay thế **Virt-Manager GUI**, nhưng chạy qua trình duyệt.

## 2. Chức năng chính
- **Quản lý host**: kết nối đến 1 hoặc nhiều kvm host qua SSH
- **Quản lý VM**:
  - Tạo, start, stop, restart, delete VM.
  - Snapshot, clone, migrate
  - Console VNC/SPICE ngay trên web.
- **Quản lý Storage**:
  - Upload ISO, qcow2.
  - Tạo disk mới, attack/detach disk
- **Quản lý network**:
  - Tạo và cấu hình bridge, NAT, isolated network
- **Multi-user:**
  - Có quản lý user & role (admin, user)
  - Hữu ích khi nhiều người cùng sử dụng KVM

### 3.5 Cài đặt WebVirtCloud

```bash
# Dùng git để tải mã nguồn WebVirtCloud vào folder /srv
cd /srv
git clone https://github.com/retspen/webvirtcloud.git

# Di chuyển vào thư mục webvirtcloud
cd webvirtcloud

# Tạo file settings.py từ file mẫu
cp webvirtcloud/settings.py.template webvirtcloud/settings.py

# Tạo SECRET_KEY bằng script có sẵn
SECRET=$(python3 conf/runit/secret_generator.py)

# Thêm secret vừa tạo vào file settings.py
sed -i "s|SECRET_KEY = \"\"|SECRET_KEY = \"$SECRET\"|" webvirtcloud/settings.py

# Mở file settings.py để chỉnh sửa
vim webvirtcloud/settings.py
# Thêm hostname hệ thống vào danh sách CSRF_TRUSTED_ORIGINS
CSRF_TRUSTED_ORIGINS = ['http://192.168.133.133','http://webvirtcloud.test',]

# Thêm dòng sau vào file /etc/hosts
192.168.133.133 webvirtcloud.test

# Sao chép file cấu hình nginx và supervisor
cp conf/nginx/webvirtcloud.conf /etc/nginx/conf.d
cp conf/supervisor/webvirtcloud.conf /etc/supervisor/conf.d

# Mở file cấu hình Nginx để chỉnh sửa
vim /etc/nginx/conf.d/webvirtcloud.conf
# Cập nhật giá trị server_name và cấu hình file log
server {
    listen 80;

    server_name webvirtcloud.test;
    access_log /var/log/nginx/webvirtcloud-access_log;
    error_log /var/log/nginx/webvirtcloud-error_log;

# Tạo môi trường ảo cho WebVirtCloud
python3 -m venv venv

# Kích hoạt môi trường ảo và cài đặt yêu cầu
source venv/bin/activate

# Cài đặt các gói Python trong môi trường ảo
pip3 install -r conf/requirements.txt

# Thực hiện migrate cơ sở dữ liệu và tạo static files
pip3 install setuptools
python3 manage.py migrate
python3 manage.py collectstatic --noinput

# Cấp quyền sở hữu cho user web server
chown -R www-data:www-data /srv/webvirtcloud

# Khởi động lại Nginx và Supervisor
systemctl enable nginx supervisor
systemctl restart nginx supervisor

# Kiểm tra các dịch vụ được quản lý bởi supervisor
supervisorctl status
# Xác nhận dịch vụ đang chạy
ss -tunelp | grep 8000
```

### 3.6 Thiết lập các kết nối
- Khi chúng ta hoàn thành các bước trên, nó có hiển thị 1 cái lỗi:
```bash
Failed to open cgroup2 by ID
```
![altimage](../images/Screenshot_56.png)
- Không phải lỗi chết chương trình
- Chỉ là warning của ss khi: hệ thống dùng cgroup v1/v2 không đồng nhất hoặc user không có đủ quyền đọc một số cgroup
- Chỉ truy cập được từ localhost, Không truy cập được từ bên ngoài

Để mở kết nối: 
- Cách 1 bạn có thể dùng 
```bash
listen_tcp = 1
auth_tcp = "none"
listen_addr = "0.0.0.0"
``` 
và 
```bash
LIBVIRTD_ARGS="--listen"
```
- **Cách này gần như đã “deprecated” trên Ubuntu mới (22.04+ và đặc biệt 24.04)**
- Ubuntu mới dùng:
  - systemd socket activation
  - modular daemons (virtqemud, virtnetworkd...)

Điều đó dễ dẫn đến bạn bị lỗi liên tục.
- Dùng SSH trực tiếp(khuyến nghị)
```bash
systemctl status ssh
```
```bash
apt install -y openssh-server
systemctl enable --now ssh
```
Test
```bash
ssh root@IP_KVM
```
- login được là xong.

Bạn nên xóa file mặc định của nginx hoặc đổi port để tránh xung đột:
```bash
rm /etc/nginx/sites-enabled/default
systemctl restart nginx
```

### 3.7 Sửa default network
- Khi bạn reset libvirt bằng lệnh:
```bash
sudo systemctl revert libvirtd
sudo systemctl enable --now libvirtd.socket
sudo systemctl daemon-reexec
sudo systemctl daemon-reload
sudo systemctl restart libvirtd
```
Nó sẽ tạo lại 1 default network mới 
- Bạn nên xóa và tạo lại 1 cái default mới không lỗi để tất cả dịch vụ đều chạy được:
```bash
ip link delete virbr0
virsh net-destroy default
virsh net-undefine default
virsh net-define /usr/share/libvirt/networks/default.xml
virsh net-start default
virsh net-autostart default
virsh net-list --all
```

### 5.1 Tạo ssh keypair
- Xác định ai là chủ webvirtcloud:
```bash
ps aux | grep -E "webvirt|gunicorn|django"
```
- Xóa SSH key cũ do ta không biết nó có tự gen ra không:
```bash
sudo -u www-data ssh-keygen -R 192.168.70.122
```
- Tạo lại known_hosts 
```bash
sudo -u www-data ssh root@192.168.70.122
```
- user `www-data` chưa có file known_hosts (thậm chí chưa có .ssh)
```bash
sudo mkdir -p /var/www/.ssh
sudo chown www-data:www-data /var/www/.ssh
sudo chmod 700 /var/www/.ssh
```
- Tạo `known_hosts` bằng cách scan key
```bash
sudo -u www-data ssh-keyscan 192.168.70.122 >> /var/www/.ssh/known_hosts
```
- Fix quyền file
```bash
sudo chown www-data:www-data /var/www/.ssh/known_hosts
sudo chmod 644 /var/www/.ssh/known_hosts
```
- Test
```bash
sudo -u www-data ssh root@192.168.70.122
```
WebVirtCloud SSH được tới server rồi. Nhưng KHÔNG login được do WebVirtCloud dùng SSH kiểu: key-based authentication
- Tạo SSH key cho `www-data`
```bash
sudo -u www-data ssh-keygen -t rsa -b 4096
```
- Copy sang KVM01
```bash
sudo -u www-data ssh-copy-id root@192.168.70.122
```
- Test lại
```bash
sudo -u www-data ssh root@192.168.70.122
```
- Hoàn thành
